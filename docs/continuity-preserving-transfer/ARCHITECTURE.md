# Continuity-Preserving Transfer Protocols — Architecture

## Overview

This document defines the architecture for maintaining unbroken subjective experience during the transition from biological to synthetic substrate. The core strategy is **Gradual Neuronal Replacement (GNR)** — replacing biological neurons with functional synthetic equivalents incrementally while the subject remains conscious, avoiding the teleporter problem entirely.

---

## Core Concepts

### Operational Definition of Experiential Continuity

Experiential continuity is defined as the unbroken persistence of integrated information processing across time. Formally:

- **Continuity Metric (Ψ)**: A composite score derived from:
  1. **Φ (Integrated Information)**: Per IIT — the quantity of integrated information generated by the system must remain above a subject-specific baseline threshold (Φ_min) at every measurement interval
  2. **Causal Continuity (CC)**: Each time-step's neural state must be causally descended from the prior time-step's state — no state is injected from an external copy
  3. **Experiential Binding (EB)**: The unified field of experience (as measured by global workspace accessibility) must not fragment into disconnected sub-experiences

- **Continuity is preserved** when Ψ(t) ≥ Ψ_threshold for all t during transfer
- **Continuity is violated** when Ψ(t) drops below Ψ_threshold at any measurement point

### The Teleporter Problem Resolution

The protocol resolves the teleporter problem through **causal substrate continuity**:

1. At no point does a complete copy of the mind exist in a separate substrate
2. Each replacement unit inherits causal state from its biological predecessor in situ
3. The biological neuron is not destroyed and then re-created — it is functionally subsumed by a synthetic unit that has already integrated into the live network
4. The subject never loses consciousness; the replacement occurs beneath an active experiential stream

This is formally distinct from copy-and-destroy because:
- **No duplication**: There is never a moment where two complete copies exist
- **Causal inheritance**: Each synthetic unit receives its initial state from the living biological unit it replaces, through direct physical coupling
- **Continuity of substrate**: The physical system is always one connected entity — parts change, but the whole persists

## System Architecture

### 1. Replacement Unit: Synthetic Neuron Equivalent (SNE)

The atomic unit of replacement. Each SNE must:

| Property | Requirement |
|---|---|
| **Functional equivalence** | Match the I/O transfer function of the biological neuron it replaces (from 0.2.2.1 emulation model) |
| **Physical interface** | Form direct synaptic connections with surrounding biological neurons |
| **State absorption** | Read and internalize the current dynamic state of the biological neuron before takeover |
| **Bidirectional handoff** | Support both bio→synthetic and synthetic→bio transition (for rollback) |
| **Telemetry** | Report local Φ contribution, firing patterns, and connection health to the monitoring system |

### 2. Replacement Protocol Engine (RPE)

Orchestrates the overall transfer process.

#### Interfaces

```
RPE.planTransfer(brainModel: EmulationModel) → TransferPlan
RPE.executeStep(plan: TransferPlan, stepIndex: number) → StepResult
RPE.pause() → PauseState
RPE.rollback(toStep: number) → RollbackResult
RPE.abort() → AbortResult
```

#### Transfer Plan Structure

```
TransferPlan {
  subject: SubjectProfile
  totalUnits: number                    // total neurons to replace
  replacementOrder: ReplacementUnit[]   // ordered sequence
  batchSize: number                     // units per step (1–100)
  pacingInterval: Duration              // minimum time between steps
  continuityThreshold: Ψ_threshold      // minimum acceptable Ψ
  rollbackCapacity: number              // how many steps back we can reverse
}
```

#### Replacement Ordering Constraints

The order of neuronal replacement is critical. Constraints:

1. **Periphery-first**: Begin with sensory/motor cortex neurons (less central to unified experience), progress toward association cortex and thalamic nuclei last
2. **Cluster coherence**: Replace neurons in local clusters to minimize interface boundaries between bio and synthetic tissue at any given time
3. **Avoid critical bottlenecks**: Never replace more than 10% of any single thalamocortical loop's neurons in one step
4. **Connectivity-aware**: Prefer replacing neurons whose immediate neighbors are already synthetic (reducing bio-synthetic interface complexity)

#### Pacing

- **Minimum interval**: Subject-specific, determined by stabilization time of Ψ after each replacement step
- **Adaptive pacing**: If Ψ dips after a step, increase interval; if Ψ remains stable, interval may be decreased
- **Typical estimate**: Hours to days per batch, full transfer over weeks to months

### 3. Real-Time Continuity Monitor (RTCM)

Continuously measures experiential continuity during transfer.

#### Interfaces

```
RTCM.startMonitoring(subject: SubjectProfile, baseline: ContinuityBaseline) → MonitorSession
RTCM.getCurrentΨ() → Ψ_value
RTCM.getComponentMetrics() → { Φ: number, CC: number, EB: number }
RTCM.checkThreshold() → { safe: boolean, margin: number }
RTCM.onThresholdBreach(callback: (alert: ContinuityAlert) → void)
RTCM.getHistory() → TimeseriesData<Ψ>
```

#### Measurement Sources

| Metric | Source | Sampling Rate |
|---|---|---|
| Φ (Integrated Information) | Global network analysis of bio+synthetic tissue | Every replacement step + continuous background |
| Causal Continuity | State-chain verification in SNE telemetry | Continuous |
| Experiential Binding | Global workspace accessibility probes | Every 100ms during active replacement |
| Subject Report | Verbal/neural self-report of experiential continuity | Periodic (every step minimum) |

#### Alert Levels

- **GREEN**: Ψ ≥ 1.5 × Ψ_threshold — safe to proceed
- **YELLOW**: Ψ_threshold ≤ Ψ < 1.5 × Ψ_threshold — proceed with caution, increase pacing interval
- **RED**: Ψ < Ψ_threshold — HALT immediately, initiate rollback

### 4. Rollback Engine

Reverses partial transfer if continuity is threatened.

#### Interfaces

```
RollbackEngine.canRollback(steps: number) → boolean
RollbackEngine.executeRollback(steps: number) → RollbackResult
RollbackEngine.getReversibleSteps() → number
```

#### Rollback Mechanism

1. Each SNE retains the biological neuron's state at time of replacement
2. The biological neuron is not destroyed — it is deactivated and maintained in viable standby during a grace period
3. Rollback reactivates biological neurons and deactivates SNEs in reverse order
4. Grace period (biological viability window): SNEs must maintain bio neurons in standby for a configurable period after replacement (minimum 72 hours)
5. Beyond the grace period, rollback requires synthetic→bio regeneration (slower, higher risk)

### 5. Subject Continuity Confirmation (SCC)

The subject's own report of unbroken experience.

#### Protocol

1. **Pre-transfer baseline**: Establish subject's normal experiential self-report patterns
2. **During transfer**: At each step, subject is asked structured questions about their experiential stream (e.g., "Has there been any gap, discontinuity, or alteration in your experience?")
3. **Post-transfer**: Extended interview and longitudinal follow-up
4. **Cross-validation**: Subject reports are compared against RTCM objective metrics — discrepancies trigger investigation
