# 0.7.3.2.3.2.3 Sign and Store Regenerated Manifest [PLAN]

Create a cryptographic signature for the new manifest and store it together with the previous version for audit comparison.

**Description**
Use GPG (or equivalent) to sign the generated manifest, verify the signature, and place both the manifest and its signature in the versioned artifacts directory.

**Acceptance Criteria**
1. A detached signature file `manifest-<timestamp>.json.sig` is produced using a trusted GPG key.  
2. The signature verification succeeds (`gpg --verify`).  
3. Both the manifest and its signature are stored under `artifacts/manifest/` with timestamped filenames.  
4. The new files are logged in `logs/manifest-signing.log`.  
5. The previous manifest version remains unchanged for side‑by‑side audit.
