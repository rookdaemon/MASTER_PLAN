# 0.7.3.2.4.3.2 Upload Archive to Repository [PLAN]

**Description:**  
Calculate the SHA‑256 checksum of the archive and upload the archive (and checksum) to the configured artifact repository via its REST API.

**Acceptance Criteria**
1. SHA‑256 checksum of `dist/plan-guardian-0.7.3.${DATE}.tar.gz` is written to `dist/plan-guardian-0.7.3.${DATE}.sha256`.
2. The repository endpoint, credentials, and target repository are read from configuration (`config/repo.yaml`).
3. Upload of the archive succeeds (HTTP 2xx) and the response JSON contains a permanent URL; this URL is logged.
4. The checksum file is also uploaded and its URL logged.
5. Any HTTP error or checksum mismatch aborts the step, logs the error to `logs/release-0.7.3.2.log`, and marks the task as failed.
