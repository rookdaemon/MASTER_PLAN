# 0.7.3.1.1.2.1.3 Save Parsed Provenance to YAML [PLAN]

Take the validated deliverable list and serialize it to `tmp/parsed-provenance.yaml` in a clear, human‑readable YAML format.

**Acceptance Criteria**
1. Output file `tmp/parsed-provenance.yaml` is created (overwrites any existing file).
2. YAML representation includes `id`, `type`, `artifact_path`, and a `valid` flag for each entry.
3. The file is syntactically correct YAML – can be parsed back without error.
4. A log entry “Parsed provenance saved to tmp/parsed-provenance.yaml (N entries)” is emitted.
5. Downstream tasks can read the file; its path is documented in the parent plan’s description.
