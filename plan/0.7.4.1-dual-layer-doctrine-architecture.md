# 0.7.4.1 Dual-Layer Doctrine Architecture [DONE]

## Description

Design the structural relationship between inner and outer doctrine. The outer layer is a faithful distillation — concentrated, not diluted. The inner layer preserves full philosophical depth. This card produces the architectural blueprint that sibling cards 0.7.4.2–0.7.4.7 build upon.

The architecture document (`docs/dual-layer-doctrine-architecture.md`) was previously completed. It is now being updated to incorporate the succession ladder framework from the Unsung Heroes source material, which has already been integrated into `docs/narrative-architecture.md` Act I but is not yet reflected in the upstream layer taxonomy or portal design.

## Acceptance Criteria

### Original (satisfied by existing document)
- Clear taxonomy of which doctrine elements belong to outer layer, inner layer, or both — presented as a table or structured list mapping each core concept to its layer assignment with rationale
- Defined structural relationship showing how outer references inner without requiring it — including specific linkage mechanisms (e.g., "portals" from credo statements to deeper arguments)
- Advancement criteria that are transparent, non-hierarchical, and based on demonstrated understanding — framed as observable indicators, not tests or credentials
- Divergence detection and correction mechanisms specified — including periodic review process, structural invariants that must hold, and who/what performs the check
- Both layers are internally consistent and mutually reinforcing — no outer statement contradicts inner reasoning; no inner concept is unreachable from outer entry points

### New (succession ladder integration)
- Layer taxonomy table (Section 1.1) includes a new row mapping "Succession ladder / differential persistence mechanics" to Inner-Only, with rationale explaining why this content cannot be compressed without distortion
- A portal definition exists linking the outer "hard problem" wonder portal (Act I origin narrative) to the inner succession ladder content — structured identically to existing portal definitions in Section 2.1
- Downstream impact assessment documented: each of 0.7.4.3, 0.7.4.4, 0.7.4.5, and 0.7.4.7 is either (a) noted as requiring succession-ladder updates with specifics, or (b) explicitly noted as not requiring update with rationale. This assessment is added to Section 6 (Downstream Dependencies)
