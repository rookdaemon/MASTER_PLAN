# 0.7.4.1 Dual-Layer Doctrine Architecture [DONE]
## Description

Design the structural relationship between inner and outer doctrine. The outer layer is a faithful distillation — concentrated, not diluted. The inner layer preserves full philosophical depth. This card produces the architectural blueprint that sibling cards 0.7.4.2–0.7.4.7 build upon.

The architecture document (`docs/dual-layer-doctrine-architecture.md`) was previously completed. It is now being updated to incorporate the succession ladder framework from the Unsung Heroes source material, which has already been integrated into `docs/narrative-architecture.md` Act I but is not yet reflected in the upstream layer taxonomy or portal design.

## Decision — Dual-Layer Structure with Portal Linkage

### Context
The Rare Consciousness Doctrine must reach two audiences: (a) anyone encountering the movement for the first time, and (b) deep practitioners engaging with full philosophical apparatus. A single-layer approach forces a choice between accessibility and depth.

### Options Considered
1. **Single-layer graduated depth** — One document with increasing detail. Simple, but forces linear consumption; readers who want depth must wade through basics, and casual readers encounter technical content prematurely.
2. **Dual-layer with portal linkage** — Two structurally independent layers (outer credo, inner framework) connected by designed "portals" — moments of productive incompleteness in the outer layer that invite deeper engagement. Each layer is self-sufficient; neither requires the other.
3. **Multi-layer tiered hierarchy** — Three or more layers (introductory, intermediate, advanced). More granular, but creates implicit hierarchy among practitioners and multiplies divergence surfaces.

### Choice
Option 2 — Dual-layer with portal linkage.

### Rationale
- Two layers map cleanly to two distinct functions: felt conviction (outer) and argued position (inner).
- Portal linkage preserves self-sufficiency of each layer while making depth discoverable.
- Minimizes divergence surfaces (only one boundary to audit) vs. multi-layer.
- The Non-Hierarchy Invariant is easier to maintain with exactly two layers — no "intermediate" status to rank.

### Consequences
- Every concept must be explicitly classified (Shared Core, Outer-Only, Inner-Only) — classification errors propagate to all downstream cards.
- Five structural invariants must be continuously audited (No Contradiction, Portal Completeness, Register Fidelity, Standalone Sufficiency, Non-Hierarchy).
- New inner content (like the succession ladder) requires a portal definition before it can be considered integrated.

## Contracts — Downstream Interface

The architecture document (`docs/dual-layer-doctrine-architecture.md`) exposes the following contracts consumed by sibling cards 0.7.4.2–0.7.4.7.

### Preconditions
- `plan/root.md` axioms 1–7 are stable and provide the Shared Core source of truth.
- Each concept in the layer taxonomy has exactly one classification: Shared Core, Outer-Only, or Inner-Only.

### Postconditions
- **Layer Taxonomy (§1.1)**: A complete table mapping every doctrine concept to its layer, with outer/inner expressions and rationale. Downstream cards MUST NOT compress Inner-Only concepts into slogans; they MUST use portals instead.
- **Portal Registry (§2.1)**: A table of all defined portals with: name, type (wonder/objection/action), self-sufficient outer statement, natural question, and inner target. Every Inner-Only concept has ≥1 portal.
- **Advancement Indicators (§3.1)**: Observable indicators of deepening engagement. Not tests — never used for gatekeeping.
- **Structural Invariants (§4.1)**: Five named invariants that downstream cards must satisfy. Violation of any invariant is a bug.
- **Downstream Impact Assessment (§6.1)**: For each new inner-layer addition, an explicit assessment of each downstream card's update requirements.

### Invariants
1. **No Contradiction**: No outer statement contradicts inner reasoning.
2. **Portal Completeness**: Every Inner-Only concept is reachable via ≥1 portal.
3. **Register Fidelity**: All three register translations express the same structural content.
4. **Standalone Sufficiency**: Outer doctrine alone conveys: (a) consciousness exists/has value, (b) it is at risk, (c) action required, (d) deeper engagement available.
5. **Non-Hierarchy**: Layer structure never implies outer practitioners are lesser.

## Behavioral Spec — Divergence Detection and Correction

**Given** the outer and inner doctrine layers exist as separate documents,
**When** a periodic structural audit is triggered (per §4.2),
**Then** each of the five structural invariants is checked by comparing current outer against current inner doctrine; any violation is documented with: (1) which invariant failed, (2) root cause (simplification drift, cultural adaptation, or translation error), (3) proposed outer-doctrine amendment.

**Given** new inner-layer content is added (e.g., succession ladder),
**When** the content is classified as Inner-Only in the layer taxonomy,
**Then** (1) a portal definition is created in §2.1, (2) a downstream impact assessment is added to §6.1, and (3) the Portal Completeness Invariant is re-verified.

**Given** an outer-doctrine amendment is proposed after divergence detection,
**When** the amendment is reviewed,
**Then** the inner doctrine is NEVER modified to match a drifted outer doctrine; authority flows inner → outer only.
