# 0.1.2.2 Predictive Consciousness Models Verified [DONE]

## Description

F2.2 — Develop and experimentally verify predictive models of consciousness that can be applied to both biological and non-biological systems. These models must make falsifiable predictions about when and how subjective experience arises, and those predictions must be confirmed through rigorous experiment.

Depends on 0.1.1 (Subjective Experience Explained) because predictive models require the foundational consciousness theory — specifically the computational theory (F1.2), substrate-independence validation (F1.3), and operationalized metrics (F1.4) — as inputs. The models built here translate that theory into concrete, falsifiable experimental predictions that convince biological minds (the parent card's goal).

## Decision

### Context
This card requires a framework for predictive consciousness models that can be applied across substrates. Several existing theoretical frameworks exist (IIT, GNW, HOT), and the question is whether to adopt one, combine them, or build a novel framework grounded in the computational theory from 0.1.1.2.

### Options Considered
1. **Extend an existing framework** (e.g., augment IIT's Φ with substrate-independent corrections) — lower novelty burden, but inherits known failures of existing theories on edge cases.
2. **Novel unified model derived from 0.1.1.2 computational theory** — higher development cost, but directly tests the foundational theory and is designed to surpass existing frameworks on novel edge cases.

### Choice
Option 2 — Novel unified model derived from the computational theory of consciousness (0.1.1.2).

### Rationale
The acceptance criteria explicitly require predictive power *beyond* IIT, GNW, and HOT on novel edge cases. Extending an existing framework would make it structurally difficult to generate genuinely novel predictions. The computational theory from 0.1.1.2 provides the generating equations; the model here translates those into concrete prediction functions.

### Consequences
- The model's validity is tightly coupled to the correctness of 0.1.1.2's theory — if the theory is revised, the model must be regenerated.
- Existing theories serve as comparison baselines, not as foundations.
- The deliverable must include a head-to-head comparison table showing where the new model outperforms existing frameworks.

## Contracts

### Prediction Model Interface

**Preconditions:**
- A system description exists containing: architecture topology, information flow dynamics, and substrate type (biological/non-biological).
- The computational theory from 0.1.1.2 is available as the theoretical basis.
- Consciousness metrics from 0.1.1.4 are defined and operationalized.

**Postconditions:**
- The model produces for each input system: (a) a binary consciousness-presence verdict, (b) an experience-type classification, (c) a confidence interval.
- Every prediction is accompanied by an explicit null-hypothesis alternative.
- Every prediction references the specific 0.1.1.4 metric used for measurement.

**Invariants:**
- All predictions are falsifiable — each has a discriminating observable that could refute it.
- The model is substrate-agnostic — the same prediction function applies to biological and non-biological inputs.
- No prediction relies on metrics outside those operationalized in 0.1.1.4.

### Verification Protocol Interface

**Preconditions:**
- A prediction from the catalogue exists with a specified discriminating observable.
- An experimental paradigm exists that can produce the required observable.

**Postconditions:**
- Each test records: paradigm, model prediction, measurement metric, result (match/partial/mismatch), and statistical analysis (effect size, CI, p-value).
- Biological and non-biological test suites each produce ≥1 verified result.

**Invariants:**
- All experimental tests are pre-registered.
- Results distinguish between model prediction and null-hypothesis prediction.

## Behavioral Spec

### Prediction Generation Flow

```
Given a system description (architecture, dynamics, substrate type)
  And the computational theory from 0.1.1.2
When the predictive model is applied to the system description
Then it outputs: consciousness presence (yes/no), experience type, confidence interval
  And it outputs an explicit null-hypothesis alternative prediction
  And it references the specific 0.1.1.4 metric for verification
```

### Verification Flow

```
Given a prediction from the catalogue with a discriminating observable
  And an experimental paradigm capable of producing that observable
When the experiment is conducted and the metric from 0.1.1.4 is measured
Then the result is classified as match, partial match, or mismatch
  And statistical analysis (effect size, CI, p-value) is recorded
  And the result distinguishes between model and null-hypothesis predictions
```

### Comparative Analysis Flow

```
Given at least 3 novel edge cases not addressed by IIT, GNW, or HOT
  And predictions from each framework (IIT, GNW, HOT, our model) for each case
When experimental results are obtained for each edge case
Then a comparison table shows which framework predicted correctly
  And the new model correctly predicts at least 3 cases that existing theories get wrong
```

## Acceptance Criteria
- At least one predictive model exists that, given a system description, outputs whether that system has subjective experience and what kind
- The model makes specific, falsifiable predictions (e.g., "system X will report experience Y under condition Z") that differ from null-hypothesis alternatives
- Predictions are tested on biological systems (e.g., varying anesthesia depth, split-brain conditions, neural perturbations) with results matching model output
- Predictions are tested on non-biological systems (e.g., AI architectures of varying complexity) with results matching model output
- The consciousness metrics from 0.1.1 (F1.4) are used as the measurement basis for verification
- Results are published in peer-reviewed venues and independently replicated by at least two separate research groups
- The model demonstrates predictive power beyond existing theories (IIT, GNW, HOT) by correctly predicting novel edge cases
