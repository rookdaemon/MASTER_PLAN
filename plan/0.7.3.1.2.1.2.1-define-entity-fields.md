# 0.7.3.1.2.1.2.1 Define Entity Fields [PLAN]

Create YAML definitions for each of the four core entities (`requirement`, `design-decision`, `code`, `test`) specifying required metadata.

**Acceptance Criteria**
1. Each entity definition includes the fields `id` (string, UUID‑style example), `title` (string), `source-path` (string, file path), and an optional `description` (string).  
2. Data types for every field are documented in the schema file.  
3. Example values are provided for each field to illustrate correct usage.  
4. The schema file `schema/crossref.yaml` validates against a JSON‑Schema that enforces the presence and types of these fields.  
5. Running `scripts/validate-schema.py` reports no missing or mismatched fields for any entity.
