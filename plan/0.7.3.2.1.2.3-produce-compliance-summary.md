# 0.7.3.2.1.2.3 Produce Compliance Summary Table [PLAN]

Aggregate results from 0.7.3.2.1.2.2 into a concise summary table for stakeholder review, and log any deviations with corrective recommendations.

**Acceptance Criteria**
1. A summary table (CSV and Markdown) lists each artifact type, total count, number compliant, number non‑compliant, and percentage compliance.
2. All identified deviations are detailed in a separate “Issues” sheet with file name, description, and suggested remediation.
3. The summary is stored at `output/0.7.3.2.1.2/template-compliance-summary.md`.
4. The summary table is reviewed and approved by the QA lead (signature captured in the document header).
5. The final summary is linked from the parent task’s documentation index.
