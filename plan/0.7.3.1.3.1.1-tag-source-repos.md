# 0.7.3.1.3.1.1 Tag Source Repos [PLAN]

Identify every source repository needed for the delivery package, create a release tag that follows the project’s versioning scheme, and verify that the tag points to the correct commit.

### Acceptance Criteria
1. A complete inventory of required repositories is documented.  
2. Each repository has a new tag (e.g., `vX.Y.Z`) created on the release commit.  
3. Tags are pushed to the central remote without errors.  
4. Tag hashes are recorded in a manifest file included in the delivery package.  
5. All tagged repositories are cloned as read‑only copies for archiving.
