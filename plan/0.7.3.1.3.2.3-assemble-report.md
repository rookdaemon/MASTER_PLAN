# 0.7.3.1.3.2.3 Assemble verification report [PLAN]

## Description
Combine the results of the ownership line check, the docs linter, and the git‑status output (from 0.7.3.1.3.1) into a single markdown verification report.

## Acceptance Criteria
1. Read `artifacts/ownership-check.json` and confirm `"found": true`.  
2. Read `artifacts/docs-lint-output.txt` and confirm it is empty (no errors).  
3. Read `artifacts/git-status.txt` produced by subtask 0.7.3.1.3.1.  
4. Generate `artifacts/verification-report.md` with:
   - `# Verification Report – 0.7.3.1.3.2` as the top heading.  
   - A section stating the ownership line was found.  
   - A section confirming the linter passed with status 0.  
   - A section containing the git‑status summary.  
   - A final line `**Overall status = SUCCESS**`.  
5. Print the absolute path to `artifacts/verification-report.md` on stdout.
