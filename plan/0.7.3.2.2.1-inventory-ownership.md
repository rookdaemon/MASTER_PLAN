# 0.7.3.2.2.1 Inventory Source Directories & Ownership [PLAN]

**Description**  
Catalogue every top‑level directory under `src/` and assign at least one owning team or individual. Capture this information in a structured table that will later be merged into the codebase map.

**Acceptance Criteria**
1. All immediate sub‑directories of `src/` are listed (e.g., `src/auth`, `src/payment`, …).  
2. Each directory has an `owner` field populated with a team name or individual email.  
3. The inventory is saved as `artifacts/codebase-inventory.yaml`.  
4. The YAML file conforms to the schema `{ path: string, owner: string[] }`.  
5. The inventory is reviewed and approved by the designated module owners via a PR comment.
