# 0.7.3.2.1.1.1 Locate Artifacts [PLAN]

Search the traceability package directory produced in **0.7.3.1** and identify the presence of the four required artifact types:

* Requirements matrix  
* Test‑case mapping  
* Risk traceability matrix  
* Compliance checklists  

**Acceptance Criteria**
1. A script or manual procedure lists all files in the top‑level package directory.
2. The list contains at least one file that belongs to each of the four required artifact categories.
3. Paths to the identified files are recorded in a temporary JSON file (`found‑artifacts.json`) for downstream steps.
4. The procedure fails with a clear error if any required category is missing.
5. The temporary JSON file is committed to the repository for audit purposes.
