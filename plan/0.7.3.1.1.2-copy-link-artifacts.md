# 0.7.3.1.1.2 Populate Artifact Directories [PLAN]

Collect and place every required provenance artifact into its corresponding sub‑folder.

**Acceptance Criteria**

1. All requirement documents are copied or symlinked into `artifacts/requirements/`.  
2. All design artifacts are copied or symlinked into `artifacts/design/`.  
3. All code link files (e.g., `source-links.yaml`, CI pipeline configs) are placed in `artifacts/code/`.  
4. All test reports, coverage data, etc., are placed in `artifacts/tests/`.  
5. Each file added is tracked by git; `git status` shows no untracked files inside the artifact tree.  
6. A verification script `verify_artifacts.sh` reports “PASS” when run, confirming that each expected category contains at least one file.
