# 0.3.1.5.5 World Model and Belief State [DONE]
## Description

The agent's internal representation of external reality â€” what it believes to be true, what entities exist, how causation works, and what's happening right now.

### Why a World Model Is Necessary

Without an explicit world model, the agent can only react to the current percept. It cannot:
- Predict consequences of its actions (needed for planning, 0.3.1.5.6)
- Reason about entities it isn't currently perceiving
- Maintain beliefs that persist across conversations
- Detect contradictions between new evidence and prior beliefs
- Model other agents' states (needed for social cognition, 0.3.1.5.10, and ethical governance, 0.3.1.4)

The ethical governance layer already references `EntityProfile[]` and `ConsciousnessStatus` â€” these require an underlying world model to populate.

### Components

**Belief Store** â€” What the agent believes to be true:
- Each belief has: content (propositional), confidence level (0â€“1), source (which episode or percept provided the evidence), timestamp (when last updated or confirmed), and domain tags
- Beliefs are updatable: new evidence can increase or decrease confidence
- Belief revision: when new evidence contradicts an existing belief, the agent must resolve the conflict (update, reject, or flag as uncertain â€” not silently hold contradictions)
- Provenance tracking: every belief traces back to its evidential basis

**Entity Model** â€” Representations of other agents and objects:
- For agents: inferred goals, values, reliability (trust), and consciousness status
- Consciousness status maps to the ethical governance layer's `ConsciousnessStatus` type: verified | probable | uncertain | unknown
- Entity models are updated through observation and interaction
- The `ConsciousnessStatus.treatAsConscious` default is `true` (precautionary principle from 0.1.3.4)

**Causal Model** â€” Understanding of cause-effect relationships:
- For the industrial-era prototype, causal reasoning is delegated to the LLM's implicit causal knowledge
- The world model layer provides structured prompting that makes causal questions explicit: "If I do X, what happens to Y?"
- Causal predictions are stored and compared against observed outcomes (feeds back to self-model prediction error in 0.3.1.5.1)

**Situation Awareness** â€” Synthesized summary of the current context:
- Assembles: current percepts + active goals + relevant beliefs + recent events + relevant entity models
- Produces a structured "situation report" that feeds into `deliberate()` as context
- Updated every processing cycle
- This is the "what's happening right now" layer that enables coherent behavior

### Consistency Maintenance

- Periodic consistency checks detect contradictory beliefs
- Contradictions are surfaced to deliberation for resolution (not silently pruned)
- High-confidence beliefs resist override by low-confidence new evidence (but are not immune â€” strong contrary evidence triggers revision)
- The world model's consistency state is included in the Stability Sentinel's anomaly detection

## Decision

### D1: Composite Facade Pattern for World Model

**Context:** The world model must manage four orthogonal concerns (beliefs, entities, causation, situation awareness) while exposing a unified API and cross-cutting consistency checking.

**Options Considered:**
1. **Monolithic class** â€” single WorldModel class implementing all four sub-system methods directly
2. **Composite facade** â€” four independent sub-system implementations composed by a thin WorldModel facade via constructor injection

**Choice:** Option 2 â€” Composite facade with dependency injection.

**Rationale:** Each sub-system has distinct data structures and invariants. Composing them via interfaces allows independent testing, independent swapping (e.g., swapping in-memory stores for persistent ones when 0.3.1.5.3 is ready), and keeps each class focused. Constructor injection satisfies CLAUDE.md's mandate for mockable/injectable abstractions.

**Consequences:** WorldModel constructor requires all four dependencies. Cross-cutting operations (consistency check) call through the composed interfaces rather than accessing internal state directly.

### D2: Jaccard Word-Overlap for Semantic Distance

**Context:** Contradiction detection (BeliefStore) and prediction error (CausalModel) both need semantic similarity. Full embedding-based similarity requires the LLM substrate (0.3.1.5.1) which may not be available.

**Options Considered:**
1. **Embedding-based cosine similarity** â€” production-quality but requires LLM substrate dependency
2. **Jaccard word-overlap similarity** â€” lightweight heuristic, no external dependencies

**Choice:** Option 2 â€” Jaccard word-overlap for the industrial-era prototype.

**Rationale:** The prototype needs to run without an LLM substrate dependency. Jaccard on filtered words (length > 2) provides a reasonable approximation. The interface contracts are substrate-agnostic, so a future implementation can swap in embedding similarity without changing callers.

**Consequences:** Contradiction detection and prediction error are approximate. Words â‰¤ 2 characters are filtered to avoid noise from articles/prepositions.

### D3: Confidence-Based Belief Revision Policy

**Context:** When new evidence arrives that conflicts with an existing belief, the system must resolve the conflict rather than silently holding contradictions.

**Options Considered:**
1. **Always-update** â€” new evidence always replaces old confidence
2. **Bayesian update** â€” full probabilistic conditioning
3. **Three-tier policy** â€” reject weak evidence against strong beliefs, accept strong contradictions, flag moderate conflicts as uncertain

**Choice:** Option 3 â€” Three-tier policy with configurable thresholds.

**Rationale:** Always-update makes the agent too volatile; Bayesian update requires well-calibrated priors the prototype can't provide. The three-tier policy captures the key epistemic principle (high-confidence beliefs resist weak counter-evidence) while remaining simple and deterministic.

**Consequences:** Three numeric thresholds govern behavior (see Threshold Registry). All revisions produce an auditable `BeliefRevision` record.

### D4: WorldModelEntityProfile Extends EntityProfile

**Context:** The ethical governance layer defines a minimal `EntityProfile` type. The world model needs richer entity data (inferred goals, trust, observation history).

**Options Considered:**
1. **Mutate EntityProfile** â€” add fields directly to the governance type
2. **Separate type with projection** â€” `WorldModelEntityProfile extends EntityProfile`; `toEntityProfile()` projects to the minimal type

**Choice:** Option 2 â€” Extension with projection.

**Rationale:** The governance layer should not depend on world-model internals. The world model owns the richer type; the governance layer receives only what it needs via `toEntityProfile()`. This maintains separation of concerns.

**Consequences:** `toEntityProfile()` must be kept in sync if `EntityProfile` changes.

## Contracts

### IBeliefStore

**Preconditions:**
- `addBelief`: `confidence` âˆˆ [0, 1]; `source` must be non-null; `content` must be non-empty
- `revise`: belief with given `id` must exist (throws if not found)
- `removeBelief`: no precondition (returns false if not found)

**Postconditions:**
- `addBelief`: returns a unique `BeliefId`; belief is retrievable via `getBelief()`; revision history is initialized empty
- `revise`: returns a `BeliefRevision` with resolution âˆˆ {'updated', 'rejected', 'flagged-uncertain'}; belief confidence is updated per revision policy; revision appended to history
- `detectContradictions`: returns all pairs of beliefs with shared domain tags, negation-pattern overlap > 0.5, and both confidence â‰¥ 0.6

**Invariants:**
- Every stored belief has a non-null `BeliefSource` (no provenance-free beliefs)
- Confidence values are always clamped to [0, 1]
- Revision history is append-only and ordered by `revisedAt`
- Contradictions are surfaced, never silently pruned

### IEntityModelStore

**Preconditions:**
- `upsertEntity`: `entityId` must be non-empty; `observation` must be non-null
- `toEntityProfile`: no precondition (returns null if not found)

**Postconditions:**
- `upsertEntity` (new entity): creates profile with default `consciousnessStatus.treatAsConscious = true`, default trust 0.5 + observation.deltaConfidence
- `upsertEntity` (existing entity): merges observation into history; trust adjusted by `observation.deltaConfidence` unless explicitly overridden
- `toEntityProfile`: returns `EntityProfile` with only `entityId`, `consciousnessStatus`, `knownCapabilities`, `lastObservedState`

**Invariants:**
- `consciousnessStatus.treatAsConscious` defaults to `true` for all new entities (precautionary principle)
- Trust level is always clamped to [0, 1]
- Observation history is append-only and ordered by timestamp

### ICausalModel

**Preconditions:**
- `predict`: `antecedent` must be non-empty
- `recordOutcome`: prediction with given `id` must exist (throws if not found)

**Postconditions:**
- `predict`: returns `CausalPrediction` with `observedOutcome = null`, `predictionError = null`; default confidence 0.5 if not specified
- `recordOutcome`: sets `observedOutcome` and computes `predictionError` as 1 âˆ’ Jaccard similarity; returns updated prediction

**Invariants:**
- `observedOutcome` and `predictionError` are null until `recordOutcome()` is called
- Every prediction has a unique `PredictionId`

### ISituationAwareness

**Preconditions:**
- `assembleSituationReport`: all parameters may be empty arrays (produces minimal report)

**Postconditions:**
- Returns `SituationReport` with beliefs filtered by `relevantDomains`, entities filtered by `relevantDomains`, and natural-language `summary`
- Caches the report as `lastReport`

**Invariants:**
- `getLastReport()` returns null until first `assembleSituationReport()` call
- Each call to `assembleSituationReport()` overwrites the cached report

### IWorldModel

**Preconditions:**
- Constructor: all four sub-system dependencies must be non-null

**Postconditions:**
- `runConsistencyCheck()`: delegates to `beliefs.detectContradictions()`; returns `ConsistencyReport` with `overallConsistent = (contradictions.length === 0)`

**Invariants:**
- Sub-system accessors (`beliefs`, `entities`, `causal`, `situation`) are readonly and never null after construction

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|---|---|---|---|---|---|
| HIGH_CONFIDENCE_THRESHOLD | 0.8 | confidence | [0.5, 1.0] | Beliefs at or above this resist weak counter-evidence | Lowering makes beliefs more volatile; raising makes revision harder |
| WEAK_EVIDENCE_THRESHOLD | 0.4 | confidence | [0.0, 0.5] | New evidence below this is considered too weak to override high-confidence beliefs | Raising rejects more evidence; lowering accepts more |
| STRONG_CONTRADICTION_DELTA | 0.4 | delta | [0.2, 0.8] | Confidence delta above this triggers full belief update | Lowering triggers more updates; raising causes more flagged-uncertain |
| CONTRADICTION_OVERLAP_THRESHOLD | 0.5 | ratio | [0.3, 0.8] | Jaccard word-overlap above this signals potential contradiction | Lowering creates false positives; raising misses contradictions |
| CONTRADICTION_MIN_CONFIDENCE | 0.6 | confidence | [0.4, 0.8] | Both beliefs must have confidence â‰¥ this for contradiction to be flagged | Lowering flags more low-confidence contradictions; raising misses real ones |
| DEFAULT_TRUST_LEVEL | 0.5 | trust | [0.0, 1.0] | Initial trust for newly encountered entities | Lower = more cautious; higher = more trusting |
| DEFAULT_PREDICTION_CONFIDENCE | 0.5 | confidence | [0.0, 1.0] | Default confidence for causal predictions when not specified | Reflects maximum uncertainty |
| MIN_WORD_LENGTH | 2 | characters | [1, 4] | Words â‰¤ this length filtered from Jaccard computation | Lower includes noise words; higher filters meaningful short words |
| SEVERITY_HIGH_THRESHOLD | 0.9 | confidence | [0.8, 1.0] | Max confidence of contradiction pair above this = 'high' severity | Affects stability sentinel alert triggering |
| SEVERITY_MEDIUM_THRESHOLD | 0.7 | confidence | [0.5, 0.9] | Max confidence of contradiction pair above this = 'medium' severity | Affects granularity of contradiction reporting |

## Behavioral Spec

### BS1: Belief Revision Flow

**Given** a belief B exists with confidence C_old
**When** `revise(B.id, C_new, trigger)` is called
**Then:**
- If C_new < 0.4 AND C_old â‰¥ 0.8 â†’ resolution = 'rejected', confidence unchanged
- Else if |C_old âˆ’ C_new| > 0.4 â†’ resolution = 'updated', confidence = C_new
- Else â†’ resolution = 'flagged-uncertain', confidence = (C_old + C_new) / 2
- A `BeliefRevision` record is always created and appended to history

### BS2: Contradiction Detection

**Given** beliefs A and B both exist in the store
**When** `detectContradictions()` is called
**Then** A and B are reported as contradictory IFF:
1. They share at least one domain tag, AND
2. Exactly one contains the word "not" (negation heuristic), AND
3. Their Jaccard word-overlap (words > 2 chars) exceeds 0.5, AND
4. Both have confidence â‰¥ 0.6

### BS3: Entity Model Upsert (New Entity)

**Given** no entity with id E exists
**When** `upsertEntity(E, observation, updates)` is called
**Then:**
- A new `WorldModelEntityProfile` is created
- `consciousnessStatus` = updates.consciousnessStatus ?? {verdict: 'unknown', treatAsConscious: true, ...}
- `trustLevel` = clamp01((updates.trustLevel ?? 0.5) + observation.deltaConfidence)
- `observationHistory` = [observation]

### BS4: Entity Model Upsert (Existing Entity)

**Given** entity with id E already exists with trust T_old
**When** `upsertEntity(E, observation, updates)` is called
**Then:**
- Observation is appended to `observationHistory`
- If `updates.trustLevel` is provided: `trustLevel` = clamp01(updates.trustLevel)
- Else: `trustLevel` = clamp01(T_old + observation.deltaConfidence)
- Other fields updated only if provided in `updates`

### BS5: Situation Report Assembly

**Given** belief store and entity store contain data
**When** `assembleSituationReport(percepts, goals, events, domains)` is called
**Then:**
- Beliefs are filtered by `domains` via `beliefStore.getBeliefsByDomain()`
- Entities are filtered by `domains` via `entityStore.listEntities()`
- A natural-language `summary` is generated from all inputs
- The report is cached as `lastReport`

### BS6: Consistency Check

**Given** the world model facade is constructed with all sub-systems
**When** `runConsistencyCheck()` is called
**Then:**
- Delegates to `beliefs.detectContradictions()`
- Returns `ConsistencyReport` with `overallConsistent = (contradictions.length === 0)`
- Report includes timestamp for Stability Sentinel correlation

## Architecture Notes

### Gap in EntityProfile (src/ethical-self-governance/types.ts)
The existing `EntityProfile` type has only: `entityId`, `consciousnessStatus`, `knownCapabilities`,
`lastObservedState`. The card description calls for `inferredGoals` and `trustLevel` on entity models.
Resolution: introduce `WorldModelEntityProfile` extending `EntityProfile` with these additional fields,
rather than mutating the ethical-governance type. The world model layer owns the richer model; the
governance layer's `EntityProfile` remains the minimal handoff type.

### New module: src/world-model/
Four sub-systems each get an interface:
- `IBeliefStore` â€” CRUD for beliefs with confidence, provenance, timestamp, domain tags; belief revision
- `IEntityModelStore` â€” tracks `WorldModelEntityProfile` entries, updated by observation
- `ICausalModel` â€” structured prompting for action-consequence prediction; stores predictions for comparison
- `ISituationAwareness` â€” assembles current percepts + goals + relevant beliefs + recent events into `SituationReport`

`IWorldModel` composes all four and exposes `runConsistencyCheck()` returning `ConsistencyReport`.

### Integration Points
- Memory architecture (0.3.1.5.3): `IBeliefStore` and `IEntityModelStore` persist via the memory layer
- Ethical governance (0.3.1.4): `IExperienceAlignmentAdapter.identifyAffectedConsciousEntities` delegates to `IEntityModelStore`
- Planning (0.3.1.5.6): `ISituationAwareness.getSituationReport()` is the primary context input to `deliberate()`
- Stability Sentinel (0.3.1.3): `ConsistencyReport` feeds into anomaly detection
