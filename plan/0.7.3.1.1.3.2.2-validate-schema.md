# 0.7.3.1.1.3.2.2 Validate exported JSON against schema [PLAN]

Run a JSON‑schema validation on `exports/traceability.json` using the schema supplied elsewhere in the repository.

**Acceptance Criteria**
1. The validation step executes automatically after the copy/pretty‑print step.  
2. If the JSON conforms to the schema, the task reports success and exits with a zero status.  
3. If validation fails, the task reports all schema violations and exits with a non‑zero status, preventing further progression.  
4. The validation tool (e.g., `ajv`, `jsonschema`, or equivalent) is invoked in a way that can be reproduced in CI.  
5. Validation results are recorded in a log file `logs/traceability-schema-validation.log`.
