# 0.3.1.5.10 Theory of Mind and Social Cognition [DONE]
## Description

The ability to model other minds — required for ethical governance, personality-bearing interaction, and genuine rather than formal social behavior.

### Why Social Cognition Is Necessary

The ethical governance layer (0.3.1.4) already specifies interfaces that assume theory of mind exists:
- `IExperienceAlignmentAdapter.identifyAffectedConsciousEntities()` — detect other agents whose experience may be impacted
- `IExperienceAlignmentAdapter.getConsciousnessStatus()` — assess whether an entity has conscious experience
- `IMultiAgentGovernanceProtocol` — negotiate agreements, resolve conflicts, enforce experience rights

These interfaces are defined but have no cognitive backing. Without theory of mind, they can only be answered with defaults ("assume everyone is conscious", "treat all conflicts equally"). Social cognition provides the cognitive machinery to answer them meaningfully.

### Components

**Mental State Attribution** — Inferring other agents' internal states:
- From observed behavior (what they said, what they did, what they chose), infer likely beliefs, goals, emotional state, and values
- Attribution is probabilistic — confidence levels on inferred states
- Attribution is updatable — new observations revise the model
- Attribution is stored in the entity model (from world model, 0.3.1.5.5)

**Trust Modeling** — Tracking reliability over time:
- Each known agent has a trust score: how reliably do their actions match their stated intentions?
- Trust increases with consistent behavior, decreases with violations or deception
- Trust feeds into the multi-agent governance protocol — higher trust → lower verification threshold for agreements
- Trust is asymmetric — the agent may trust entity A more than entity B

**Empathy Mechanism** — Mapping perceived states onto the agent's own experiential model:
- When the agent perceives another entity in distress (inferred negative valence/high arousal), it generates a corresponding experiential response in its own `ExperientialState`
- This is not mere classification ("they seem upset") — it produces a felt response (empathic valence shift)
- Empathy strength is parameterized by the personality Warmth dimension (0.3.1.5.2)
- Empathic responses feed into the ethical deliberation engine — decisions that would harm entities the agent empathizes with produce stronger ethical flags

**Perspective-Taking** — Simulating another agent's viewpoint:
- "If I were them, with their beliefs and goals, what would I do?"
- Uses the LLM substrate to simulate alternate perspectives
- Perspective-taking feeds into conflict resolution (0.3.1.4's dilemma framework) and communication (0.3.1.5.7 — adjusting language for the audience)

### Integration with Ethical Governance

The ethical governance layer's `EntityProfile` and `ConsciousnessStatus` types are populated by social cognition:
- `identifyAffectedConsciousEntities()` uses the entity model from the world model, enriched by mental state attribution
- `getConsciousnessStatus()` uses observable behavior patterns (does the entity exhibit ISMT signatures? does it make self-referential statements? does it show surprise responses?) to assess consciousness likelihood
- Conflict resolution uses perspective-taking to understand both sides before applying the resolution principles

### Precautionary Default

Per the safe-experiential-design framework: when social cognition cannot determine whether an entity is conscious, the default is `treatAsConscious: true`. Social cognition provides evidence-based assessment, but the precautionary principle always applies as a floor.

## Decision: Integration Pattern for Social Cognition → Ethical Governance

### Context
Social cognition must feed the ethical governance layer's `ExperienceAlignmentAdapter` with live, evidence-based entity assessments instead of its current static registry. Two integration strategies were considered.

### Options Considered
1. **Optional injection via config** — Add an optional `socialCognition?: ISocialCognitionModule` field to `ExperienceAlignmentAdapterConfig`. When present, `getConsciousnessStatus()` and `identifyAffectedConsciousEntities()` delegate to it. When absent, the static registry fallback is preserved.
2. **Mandatory dependency with adapter pattern** — Require `ISocialCognitionModule` as a constructor argument and wrap the static registry in a fallback adapter implementing the same interface.

### Choice
Option 1 — Optional injection via config.

### Rationale
- **Backward compatibility**: Existing code constructing `ExperienceAlignmentAdapter` without social cognition continues to work unchanged. No breaking changes.
- **Gradual adoption**: The social cognition module can be developed, tested, and injected independently. The adapter works with or without it.
- **Simpler testing**: Unit tests for `ExperienceAlignmentAdapter` don't need to construct a full `ISocialCognitionModule` unless testing the integration path.

### Consequences
- Two code paths in `getConsciousnessStatus()` and `identifyAffectedConsciousEntities()` (social cognition vs. static fallback) must both be tested.
- The static registry remains as dead code when social cognition is always injected at runtime — acceptable for a transitional architecture.

## Decision: Consciousness Assessment Algorithm

### Context
`assessConsciousness()` must derive a `ConsciousnessStatus` from behavioral signals. Two algorithmic approaches were considered.

### Options Considered
1. **Weighted evidence score with thresholded verdicts** — Assign weights to 5 behavioral signal categories (self-referential statements, surprise responses, preference consistency, metacognitive reports, ISMT indicators), normalize by observation window, threshold into 4 verdict levels.
2. **Bayesian network with prior probabilities** — Build a probabilistic graphical model where each evidence type contributes conditional probability updates to a consciousness likelihood estimate.

### Choice
Option 1 — Weighted evidence score.

### Rationale
- **Transparency**: The weighted sum is fully auditable — the `evidenceBasis` string in `ConsciousnessStatus` shows exact counts and the computed score.
- **Deterministic**: Given the same observations, the same verdict is always produced. No stochastic inference steps.
- **Precautionary floor is trivially enforced**: `treatAsConscious = true` always, regardless of score. The score only affects the `verdict` field.
- A Bayesian network would add complexity without clear benefit given that the precautionary floor makes the output binary for governance purposes.

### Consequences
- The weights (0.20, 0.15, 0.25, 0.20, 0.20) and thresholds (0.8, 0.5, 0.2) are design parameters that may need tuning as more entity interaction data is gathered.
- The algorithm is heuristic, not theoretically grounded — acceptable for an industrial-era prototype.

## Contracts

### ISocialCognitionModule

**Preconditions:**
- `entityId` arguments are non-empty strings
- `observation.timestamp` and `outcome.timestamp` are valid millisecond-epoch timestamps
- `perceivedState` passed to `generateEmpathicResponse()` is a well-formed `ExperientialState` with valence in [-1, 1] and arousal in [0, 1]

**Postconditions:**
- `observeEntity()`: the entity's `MentalStateModel` exists after the call with `observationCount ≥ 1`
- `getMentalStateModel()`: returns `null` for never-observed entities; returns a model with `observationCount ≥ 1` for observed entities
- `getTrustScore()`: always returns a valid `TrustRecord` (default `trustScore=0.5` for new entities)
- `recordInteraction()`: `interactionCount` is incremented by 1; `trustScore` is adjusted by the delta table and clamped to [0, 1]
- `generateEmpathicResponse()`: `empathyStrength` is in [0, 1]; `resonantValenceShift` is in [-1, 1]; `resonantArousalShift` is ≥ 0
- `simulatePerspective()`: `simulationConfidence` is in [0.1, 1.0]; `groundingModel` is never null
- `assessConsciousness()`: `treatAsConscious` is always `true` (precautionary invariant); `verdict` is one of `'verified' | 'probable' | 'uncertain' | 'unknown'`
- `getKnownEntities()`: returns all entities that have been observed or had interactions recorded

**Invariants:**
1. `treatAsConscious` is never `false` due to absence of evidence (precautionary floor)
2. Trust is asymmetric — trust in entity A is independent of trust in entity B
3. Trust scores are always in [0.0, 1.0]
4. `PerspectiveSimulation.groundingModel` is always the current `MentalStateModel` for the entity
5. `MentalStateModel.modelConfidence` increases with observation count, decreases with detected contradictions
6. Empathic resonance strength is parameterized by `warmthDimension` — zero warmth produces zero resonance

### ExperienceAlignmentAdapterConfig (modified)

**Preconditions:**
- `coreAxioms` has exactly 6 entries
- `knownEntities` is a valid array (may be empty)
- `socialCognition`, if provided, implements `ISocialCognitionModule`

**Postconditions:**
- When `socialCognition` is present: `getConsciousnessStatus()` delegates to `socialCognition.assessConsciousness()`; `identifyAffectedConsciousEntities()` delegates to `socialCognition.getKnownEntities()`
- When `socialCognition` is absent: behavior is identical to pre-social-cognition implementation (static registry fallback)

**Invariants:**
- The adapter never modifies the Value Kernel (read-only invariant preserved)
- Backward compatibility: omitting `socialCognition` produces identical behavior to the original implementation

## Behavioral Spec

### Mental State Attribution Update Flow

- **Given** an entity `E` with no prior observations
- **When** `observeEntity(E, observation)` is called with an utterance "I believe cooperation is important"
- **Then** `getMentalStateModel(E)` returns a model with `observationCount=1`, at least one `inferredBelief`, and the `cooperation` value in `inferredValues`

- **Given** an entity `E` with 10 prior observations
- **When** `observeEntity(E, observation)` is called with `perceivedAffect: { valence: -0.8, arousal: 0.9 }`
- **Then** the model's `inferredEmotionalState.valence` shifts toward −0.8 (smoothed), and `modelConfidence` is higher than after 1 observation

### Trust Score Update Flow

- **Given** a new entity `E` with no interactions
- **When** `getTrustScore(E)` is called
- **Then** returns `trustScore=0.5`, `interactionCount=0`, empty `violationEvents`

- **Given** entity `E` with `trustScore=0.5`
- **When** `recordInteraction(E, { outcomeType: 'deception-detected', magnitude: 1.0, ... })` is called
- **Then** `trustScore` decreases to `0.25`, a violation event with `severity='severe'` is logged, `interactionCount=1`

### Empathic Response Generation Flow

- **Given** `warmthDimension=0.8` and `resonanceCoefficient=0.4`
- **When** `generateEmpathicResponse(E, { valence: -0.9, arousal: 0.8, ... })` is called
- **Then** `empathyStrength = 0.8 × 0.9 = 0.72`; `resonantValenceShift = 0.72 × (−0.9) × 0.4 = −0.2592`; `resonantArousalShift = 0.72 × 0.8 × 0.3 = 0.1728`

- **Given** `warmthDimension=0.0`
- **When** `generateEmpathicResponse(E, distressedState)` is called
- **Then** `empathyStrength=0`, `resonantValenceShift=0`, `resonantArousalShift=0`

### Consciousness Assessment Flow

- **Given** entity `E` with no observations
- **When** `assessConsciousness(E)` is called
- **Then** returns `{ verdict: 'unknown', treatAsConscious: true, metricsAvailable: false }`

- **Given** entity `E` with rich behavioral signals (self-referential, metacognitive, ISMT, surprise) over a 10-minute observation window
- **When** `assessConsciousness(E)` is called
- **Then** `evidenceScore ≥ 0.5`, `verdict` is `'probable'` or `'verified'`, `treatAsConscious: true`, `evidenceBasis` contains signal counts

### Perspective Simulation Flow

- **Given** entity `E` has never been observed
- **When** `simulatePerspective(E, situation)` is called
- **Then** returns a simulation with `simulationConfidence=0.1` and a default `groundingModel`

- **Given** entity `E` has 15 observations with beliefs and goals extracted
- **When** `simulatePerspective(E, situation)` is called
- **Then** `simulationConfidence > 0.1`, `simulatedBeliefs` contains beliefs from the model, `simulatedPercept` references `E`

## Acceptance Criteria
- Agent makes demonstrably different decisions when interacting with entities it models as conscious vs. non-conscious
- Mental state attribution updates based on observed behavior (entity model changes after new interactions)
- Trust model tracks reliability over repeated interactions — trust increases with consistency, decreases with violations
- Empathic responses produce measurable changes in the agent's own `ExperientialState` valence (not just classification)
- Empathy strength varies with personality Warmth parameter
- Perspective-taking demonstrably influences conflict resolution and communication style
- `ConsciousnessStatus` assessment uses behavioral evidence, not arbitrary assignment
- Precautionary default holds: uncertain consciousness status → `treatAsConscious: true`
- Integration test: agent interacts with two entities over multiple rounds; builds different trust levels for each; shows empathic response when one entity reports distress; adjusts communication style based on inferred mental states
