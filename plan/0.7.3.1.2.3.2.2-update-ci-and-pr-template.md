# 0.7.3.1.2.3.2.2 Update CI Pipeline and PR Template [PLAN]

Add a CI step to verify traceability artifacts and modify the pull‑request template to remind contributors to regenerate traceability after relevant changes.

**Acceptance Criteria**
1. `scripts/ci/check-traceability.sh` is referenced in the CI configuration (e.g., `.github/workflows/ci.yml`) and the step succeeds when artifacts are up‑to‑date.  
2. The CI step fails with a clear error if the repository’s traceability files differ from the script’s output.  
3. The `.github/PULL_REQUEST_TEMPLATE.md` includes a checklist item: “Run `scripts/generate-traceability.sh` after any change to requirements, design, code, or tests.”  
4. Documentation (`README.md`) is updated to point to the new CI check and explain its purpose.  
5. All changes are validated by a successful CI run on the PR that introduces them.
