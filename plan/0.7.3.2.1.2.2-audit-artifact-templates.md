# 0.7.3.2.1.2.2 Audit Artifact Templates and Versions [PLAN]

Using the inventory from 0.7.3.2.1.2.1, programmatically inspect every artifact in the traceability package to verify:
* the file adheres to its declared template schema (column headers, required fields, data formats)
* the version identifier in the filename matches the version declared inside the document header
* the version number meets or exceeds the minimum required version from the standards.

**Acceptance Criteria**
1. All artifacts are processed without runtime errors; a log of processed file paths is produced.
2. Any artifact that fails schema validation is recorded with file name, offending fields, and error description.
3. Any artifact with a mismatched or insufficient version identifier is recorded with expected vs. actual version.
4. A machine‑readable compliance JSON (`output/0.7.3.2.1.2/compliance-results.json`) lists each artifact and its pass/fail status.
5. The audit script (`scripts/audit_templates_versions.py`) is checked into the repository and passes its own unit tests.
