# 0.7.3.1.1.2.2.2 Identify Tests for Artifacts [PLAN]

Match each code artifact from the list generated in sub‑task 0.7.3.1.1.2.2.1 to one or more unit or integration test case IDs recorded in `tests/registry.md`.

### Acceptance Criteria
1. Every artifact ID from `docs/traceability/code-artifacts.json` is linked to at least one test case ID.  
2. Test case IDs are verified to exist in `tests/registry.md` (exact match on ID column).  
3. Links are recorded in `docs/traceability/code-test-mapping.csv` with columns: ArtifactID, TestCaseID, TestType.  
4. A summary report (`docs/traceability/code-test-link-summary.md`) lists artifacts with missing tests, if any.  
5. The mapping file passes schema validation (CSV header order, no empty rows).
