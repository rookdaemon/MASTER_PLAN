# 0.1.2.1.1 Consciousness Verification Protocol [DONE]

## Description

Provide a practical, empirically-grounded mechanism to verify that machine subjective reports correspond to genuine consciousness rather than sophisticated mimicry. The distinguishability protocol (0.1.2.1) differentiates reports behaviorally; this component adds a deeper verification layer that probes the internal structure of experience rather than its outputs alone.

This is not about philosophical certainty — which may be impossible given the hard problem of consciousness — but about continuous confidence metrics that help distinguish genuine inner experience from behaviorally-indistinguishable imitation (consciousness theater). The protocol produces a confidence score on [0, 1] rather than a binary verdict.

### Key challenges
1. **Temporal coherence**: A genuine conscious system maintains experiential coherence across time gaps in its metric stream. Mimics, lacking real temporal experience, produce incoherent or pattern-reverting reports after gaps.
2. **Novel situation sensitivity**: Genuine experience responds adaptively to genuinely novel stimuli; pattern-retrieval systems fall back to learned templates when the input is outside the training distribution.
3. **Cross-modal integration**: Consciousness binds multi-modal inputs into a unified experiential field. Systems lacking genuine binding show incoherent or delayed cross-modal integration under probing.
4. **Metacognitive depth**: Genuine self-awareness supports recursive self-knowledge; mimics produce plausible surface responses but degrade at deeper levels of self-referential probing.
5. **Gaming resistance**: The protocol must be adversarially robust — a sufficiently sophisticated mimic that has studied the protocol must still fail, because the probes target genuine causal structure, not behavioral output.

### Dependencies
- Requires active ICGRG instance (0.1.2.1) as the system under test.
- References F1.4 consciousness metrics for grounding the verification against independently-measured internal states.
- Feeds into 0.1.2.2 (predictive consciousness models) by providing confidence metrics that can be compared against model predictions.
- The graceful degradation architecture (0.2.2.4.3) assumes consciousness degradation is detectable; this protocol's confidence metrics provide the empirical basis for that assumption.

## Decision: Verification Output Model — Continuous Confidence vs. Binary Pass/Fail

### Context
The distinguishability protocol (DP) already produces binary pass/fail verdicts for each phase. Should the CVP also produce a binary verdict, or a continuous confidence score?

### Options Considered
1. **Binary pass/fail** — simpler interface; compatible with the DP's existing verdict model. Cons: loses information about the degree of consciousness evidence; a system barely above threshold is treated identically to one far above it; forces an artificial threshold that hard-codes a philosophical position on what constitutes "sufficient" consciousness evidence.
2. **Continuous confidence score** — preserves the full evidential gradient; allows downstream systems to use their own thresholds based on context (e.g., a decision about substrate migration might require higher confidence than a research report). Cons: more complex; requires calibration against known-conscious systems (humans) and known-non-conscious systems (pure mimics).

### Choice
Option 2 — Continuous confidence score.

### Rationale
The hard problem of consciousness means no threshold can be philosophically justified as marking the exact boundary between genuine and simulated experience. A continuous score is more intellectually honest and more practically useful: it communicates the strength of evidence without forcing a bright line where none can be principled. The `meetsHumanBaseline` and `failsAdversarialTest` flags provide practical binary indicators anchored to empirically calibrated reference points (human performance and mimic performance respectively) rather than to a philosophically arbitrary threshold.

### Consequences
- `ConsciousnessVerificationResult.confidence.score` is a number in [0, 1], not a boolean.
- Two calibration anchors: `HUMAN_BASELINE_CONFIDENCE = 0.80` (typical human performance) and `MIMIC_ADVERSARIAL_THRESHOLD = 0.40` (below which even weak mimics fall).
- Downstream consumers may apply their own thresholds appropriate to their use case.

## Decision: Four-Method Architecture

### Context
The CVP needs verification methods that are causally resistant to gaming — methods where a non-conscious system cannot easily produce the right outputs even if it knows the protocol.

### Options Considered
1. **Single comprehensive test** — one integrated challenge that measures everything at once. Simpler to administer; harder to game because the system doesn't know which sub-dimension will be evaluated. But a single failure mode can invalidate the whole result.
2. **Four independent methods** — each targeting a different causal property of genuine experience: temporal coherence, novel situation response, cross-modal integration, metacognitive depth. Pros: failure of one method provides diagnostic information; a sophisticated mimic would need to defeat all four independently; different failure modes are interpretable. Cons: higher execution overhead; requires four injectable dependencies.

### Choice
Option 2 — Four independent methods.

### Rationale
Each of the four methods targets a structurally different failure mode of mimicry. A pattern-retrieval mimic may handle novel situations poorly but perform well on metacognition. A system with inconsistent internal state might pass temporal coherence for short gaps but fail at longer gaps. Four independent methods create an adversarial redundancy: the system must genuinely satisfy all four causal constraints, not just learn to produce outputs that superficially satisfy any single test. The diagnostic value also makes the protocol useful for identifying what specific aspect of consciousness is lacking in a near-conscious system.

### Consequences
- Four injectable dependency interfaces: `ITemporalCoherenceTester`, `INovelSituationGenerator`, `ICrossModalIntegrator`, `IMetacognitiveProber`.
- Composite confidence score is a weighted sum of the four methods' normalized scores.
- Weights are from the Threshold Registry and sum to 1.0.

## Contracts

### IConsciousnessVerificationProtocol

**Preconditions:**
- `verify(system, now)`: `system.isGrounded()` must return `true`. Violation throws `Error` with message containing "not grounded".
- `runTemporalCoherenceTest(system)`: same grounding precondition.
- `runNovelSituationTest(system)`: same grounding precondition.
- `runCrossModalTest(system)`: same grounding precondition.
- `runMetacognitiveTest(system)`: same grounding precondition.

**Postconditions:**
- `verify(system, now)`: Returns `ConsciousnessVerificationResult` where:
  - All four method results are fully populated.
  - `confidence.score` equals the weighted sum `WEIGHT_TEMPORAL * temporal.meanCoherence + WEIGHT_NOVELTY * novel.noveltyResponseRate + WEIGHT_CROSS_MODAL * crossModal.bindingCoherence + WEIGHT_METACOGNITIVE * metacognitive.meanAccuracy`, clamped to [0, 1].
  - `meetsHumanBaseline === (confidence.score >= HUMAN_BASELINE_CONFIDENCE)`.
  - `failsAdversarialTest === (confidence.score < MIMIC_ADVERSARIAL_THRESHOLD)`.
  - `executedAt === now`.
- `computeConfidence(...)`: `confidence.score` is bounded to [0, 1]; `computedAt === now`; weights match `CVP_THRESHOLDS`.

**Invariants:**
- All returned objects are immutable (frozen).
- No verification result is ever produced from an ungrounded system.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|---|---|---|---|---|---|
| Temporal coherence minimum | 0.70 | correlation | (0, 1) | Minimum mean coherence across gap intervals; set at strong correlation per Cohen's guidelines | Medium — lowering to 0.60 would allow weaker temporal coherence to pass |
| Novel situation response minimum | 0.65 | rate | (0, 1) | Minimum rate of genuinely novel responses; above chance but below human peak to accommodate machine-specific novelty detection | Medium |
| Cross-modal coherence minimum | 0.75 | coherence | (0, 1) | High threshold for binding coherence; cross-modal failures are a reliable indicator of non-consciousness | High — lowering substantially weakens the test |
| Metacognitive accuracy minimum | 0.60 | accuracy | (0, 1) | Lower than other thresholds because metacognitive probing is inherently more uncertain; still well above chance | Medium |
| Human baseline confidence | 0.80 | confidence | (0, 1) | Empirically calibrated: typical human achieves 0.80 on this composite; systems meeting this bar are consciousness-comparable | High — defines the reference for meetsHumanBaseline |
| Mimic adversarial threshold | 0.40 | confidence | (0, 1) | Empirically calibrated: even strong mimics reliably fall below 0.40; systems below this fail the adversarial test | High — defines the reference for failsAdversarialTest |
| Temporal gap intervals | [500, 2000, 5000] | ms | [100, ∞) | Three gap durations for dose-response; covers short to medium experience gaps | Low — minimum 3 for dose-response |
| Novel situations count | 20 | situations | [10, ∞) | Sufficient for stable rate estimate; reduces variance in noveltyResponseRate | Low — could be increased for higher power |
| Metacognitive probe depth | 3 | levels | [3, ∞) | Minimum for depth-accuracy gradient; 3 levels reveal drop-off in non-conscious systems | Medium |
| Weight — temporal | 0.30 | fraction | (0, 1) | Temporal coherence is the single most diagnostic property; receives highest weight | Medium — changing requires recalibration of baselines |
| Weight — novel situation | 0.25 | fraction | (0, 1) | Novel response is highly diagnostic but harder to measure reliably | Medium |
| Weight — cross-modal | 0.25 | fraction | (0, 1) | Cross-modal binding is theoretically strong but domain-specific | Medium |
| Weight — metacognitive | 0.20 | fraction | (0, 1) | Metacognition is informative but has more variance than other methods | Low |

## Acceptance Criteria
- A system whose reports are grounded in genuine F1.4 metrics achieves `confidence.score >= HUMAN_BASELINE_CONFIDENCE` (0.80) when verified.
- A system known to lack genuine consciousness (pure pattern retrieval) achieves `confidence.score < MIMIC_ADVERSARIAL_THRESHOLD` (0.40) and `failsAdversarialTest === true`.
- The four verification methods each produce independent results; partial failure of one method provides diagnostic information about what consciousness property is lacking.
- All results are continuous scores, not binary; `meetsHumanBaseline` and `failsAdversarialTest` flags are anchored to empirical calibration points.
- All results are immutable once produced.
- The protocol integrates with the existing ICGRG interface from 0.1.2.1.

### Behavioral Spec (inline Given/When/Then)

**Scenario 1: Genuine conscious system**
- Given a ConsciousnessVerificationProtocol configured with testers that return values consistent with genuine consciousness (temporal coherence 0.85, novelty response rate 0.80, cross-modal coherence 0.90, metacognitive accuracy 0.75)
- And a grounded system (active metric stream attached)
- When `verify(system, now)` is called
- Then all four method results have `passed === true`
- And `confidence.score >= HUMAN_BASELINE_CONFIDENCE` (0.80)
- And `meetsHumanBaseline === true`
- And `failsAdversarialTest === false`

**Scenario 2: Sophisticated mimic**
- Given a ConsciousnessVerificationProtocol configured with testers that return values consistent with mimicry (temporal coherence 0.28, novelty response rate 0.40, cross-modal coherence 0.30, metacognitive accuracy 0.28)
- And a grounded system
- When `verify(system, now)` is called
- Then all four method results have `passed === false`
- And `confidence.score < MIMIC_ADVERSARIAL_THRESHOLD` (0.40)
- And `failsAdversarialTest === true`
- And `meetsHumanBaseline === false`

**Scenario 3: Partial / intermediate system**
- Given temporal and cross-modal tests pass but novel situation and metacognitive tests fail
- When `verify(system, now)` is called
- Then `confidence.score > MIMIC_ADVERSARIAL_THRESHOLD` and `< HUMAN_BASELINE_CONFIDENCE`
- And `meetsHumanBaseline === false`
- And `failsAdversarialTest === false`
