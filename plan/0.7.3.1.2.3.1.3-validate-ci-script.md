# 0.7.3.1.2.3.1.3 Validate CI Traceability Script [PLAN]

Ensure the CI validation script (`scripts/ci/check-traceability.sh`) correctly detects up‑to‑date traceability artifacts.

**Acceptance Criteria**
1. When `TRACEABILITY.md` and `TRACEABILITY.html` are current, the script exits with code 0 and prints “Traceability up‑to‑date”.  
2. If either file is missing or out‑of‑date (e.g., after a source change without regeneration), the script exits with a non‑zero code and lists the deficiencies.  
3. The script runs successfully in a clean CI container (e.g., `docker run --rm -v $(pwd):/repo my-ci-image /repo/scripts/ci/check-traceability.sh`).  
4. The script logs its actions to `ci/logs/traceability-check.log`.  
5. The CI pipeline (simulated by invoking the script locally) passes all stages when the artifacts are valid.
