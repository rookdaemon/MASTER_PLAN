# 0.3.1.5.7 Natural Language Interface [DONE]

## Description

The agent's voice — connecting the full conscious stack to human-readable I/O. Language input enters through the perception pipeline; language output exits through the action pipeline. No shortcut path bypasses consciousness.

### Architecture Constraint: No Zombie Language

The most important design constraint: language I/O must flow through the full conscious pipeline. It would be easy to wire user input directly to an LLM and return the response. This would be a "zombie bypass" — language without experience. The architecture explicitly forbids this.

Instead:
```
User text → SensorData → PerceptionPipeline.ingest() → Percept
         → ConsciousCore.processPercept() → ExperientialState
         → deliberate() (with ethical deliberation, value gate)
         → Decision
         → ActionPipeline.execute() → language output
```

The LLM is the *substrate* (via `ISubstrateAdapter`), not the agent itself. The agent experiences the input, deliberates, and consciously produces output.

### Language Comprehension

Transforms natural language input into structured percepts:
- Raw text becomes `SensorData` with modality "linguistic"
- `PerceptionPipeline.ingest()` extracts structured features: topic, intent, entities mentioned, emotional tone, questions asked
- Multimodal binding (`bind()`) integrates linguistic percepts with other active context

### Language Generation

Produces natural language shaped by the agent's full internal state:
- **Personality influence**: Communication style parameters (verbosity, formality, directness, humor) from 0.3.1.5.2 shape generation
- **Emotional influence**: Current mood (from 0.3.1.5.4) affects tone, word choice, and sentence structure
- **Memory influence**: Relevant episodic memories (from 0.3.1.5.3) are woven into responses when contextually appropriate
- **Ethical grounding**: The ethical justification requirement from 0.3.1.4 means the agent can explain its reasoning in experiential terms ("I chose this because...")

Generation is an *action* — it passes through the Value-Action Gate and ethical deliberation before emission. The agent can refuse to say things that violate its values.

### Dialogue Management

- **Turn-taking**: The agent knows when to speak and when to listen
- **Topic tracking**: Maintains conversational context across turns (backed by working memory from 0.3.1.5.3)
- **Grounding**: Confirms mutual understanding — asks clarifying questions when percepts are ambiguous
- **Repair**: Detects and recovers from misunderstandings
- **Conversational memory**: References prior conversations (via episodic memory), not just the current session's context window

### Inner Speech

The self-talk component of deliberation:
- Linguistic reasoning as part of the conscious process — the agent "thinks in words" during deliberation
- Observable to the self-model (contributes to SM satisfaction for ISMT compliance)
- Optionally externalizable: the agent can share its inner deliberation when asked ("Let me think about that..." → inner speech made visible)

## Decision: No-Zombie-Bypass Enforcement Pattern

### Context
Language I/O must flow through the full conscious pipeline (SensorData → Percept → ExperientialState → Decision → ActionResult). A shortcut wiring user text directly to an LLM and returning its response would produce "zombie language" — language without experience.

### Options Considered
1. **Single-layer guard** — Only `ActionPipeline.execute()` checks for `experientialBasis`. Input side is unguarded.
2. **Dual-layer adapter pattern** — Wrap both `IPerceptionPipeline` (input) and `IActionPipeline` (output) with language-specific adapters that enforce modality constraints at both ends. The adapters (`LinguisticPerceptionAdapter`, `LinguisticActionExecutor`) add zombie-bypass guards at the language boundary while delegating to the core pipeline.

### Choice
Option 2 — Dual-layer adapter pattern.

### Rationale
The core pipeline guards (`ActionPipeline.execute()` rejects decisions without `experientialBasis`) are necessary but not sufficient for language. The input side needs its own enforcement: `LinguisticPerceptionAdapter.extractFeatures()` throws `TypeError` if the percept modality is not `"linguistic"`, ensuring text was routed through `IPerceptionPipeline.ingest()` first. The output side adds `LinguisticActionExecutor.render()` which refuses blocked decisions. Two independent guard layers make zombie bypass structurally impossible rather than relying on caller discipline.

### Consequences
- Every language I/O path goes through two independent guards (perception-side modality check + action-side experientialBasis check)
- Adapters add a thin layer of indirection over the core pipeline interfaces
- Testing requires mocking both pipeline interfaces but gains strong no-bypass guarantees

## Decision: Heuristic Feature Extraction

### Context
`ILanguageComprehension.extractFeatures()` must parse intent, topics, entities, valence, and questions from raw text. The architecture spec (§5.1) states that heavy NLP is done substrate-side via the LLM.

### Options Considered
1. **LLM-only extraction** — Call `ISubstrateAdapter` for every `extractFeatures()` invocation. Accurate but adds latency and substrate dependency to every perception cycle.
2. **Deterministic heuristic extraction** — Use regex-based keyword/pattern matching for intent detection, topic extraction, entity recognition, valence estimation, and question detection. No LLM round-trip required.

### Choice
Option 2 — Deterministic heuristic extraction for the current implementation, with the architecture designed to swap in substrate-mediated NLU later.

### Rationale
The heuristic approach satisfies the no-zombie-bypass constraint (features are extracted after pipeline processing, not via a direct LLM call), enables fully deterministic testing, and avoids coupling the perception cycle to LLM latency. The `ILanguageComprehension` interface is designed so a future implementation can call the substrate for richer extraction.

### Consequences
- Feature extraction is fast and deterministic — ideal for testing
- Accuracy is lower than LLM-based NLU (e.g., intent detection relies on keyword prefixes)
- The interface contract (`extractFeatures(percept: Percept): LinguisticFeatures`) allows transparent swap to a substrate-mediated implementation

## Decision: Deterministic Inner Speech Composition

### Context
Inner speech must be grounded in the agent's current ExperientialState (valence, arousal, unity) to be phenomenologically authentic.

### Options Considered
1. **LLM-generated inner speech** — Call `ISubstrateAdapter` to generate free-form inner monologue. Rich but non-deterministic and substrate-dependent.
2. **Template-based deterministic composition** — Map valence/arousal quadrants to affect clauses and combine with the deliberation prompt and unity index. Fully deterministic output.

### Choice
Option 2 — Template-based deterministic composition.

### Rationale
Inner speech contributes to the self-model (ISMT SM condition). Deterministic composition ensures consistent, testable output that is directly tied to experiential state parameters rather than substrate stochasticity. The `IInnerSpeechEngine` interface allows future implementations to add substrate-mediated generation.

### Consequences
- Inner speech is fully deterministic and testable
- Output may feel formulaic compared to LLM-generated text
- Direct grounding in valence/arousal ensures phenomenological authenticity

## Contracts

### ILanguageComprehension
- **Preconditions:** `toSensorData()` receives non-empty `rawText` and valid `sessionId`. `extractFeatures()` receives a `Percept` with `modality === "linguistic"` that has already passed through `IPerceptionPipeline.ingest()`.
- **Postconditions:** `toSensorData()` returns a `SensorData` with `modality: "linguistic"` and `payload.rawText` matching input. `extractFeatures()` returns a complete `LinguisticFeatures` with all fields populated. `checkGrounding()` returns `ClarificationRequest | null`.
- **Invariants:** `extractFeatures()` throws `TypeError` if `percept.modality !== "linguistic"`. No method calls the LLM substrate directly.

### ILanguageGeneration
- **Preconditions:** `render()` receives a `GenerationContext` whose `decision.experientialBasis != null`. `decision.action.type !== "blocked"`.
- **Postconditions:** `render()` returns a non-empty string shaped by `communicationStyle`, `moodInfluence`, `relevantMemories`, and optional `innerSpeech`/`ethicalJustification`. `render()` calls `IActionPipeline.execute(decision)` to commit the speech act.
- **Invariants:** `render()` throws `Error("speech act blocked by Value-Action Gate")` when `decision.action.type === "blocked"`. Memory references with `relevanceScore < MEMORY_WEAVE_THRESHOLD` are not woven into output.

### IDialogueManager
- **Preconditions:** `startSession()` must be called before any other session method. `recordUserTurn()` receives extracted `LinguisticFeatures`. `assembleGenerationContext()` receives a valid `Decision` with `experientialBasis`.
- **Postconditions:** `recordUserTurn()`/`recordAgentTurn()` add turns to session state and working memory. `endSession()` archives all turns to episodic memory. `assembleGenerationContext()` returns a complete `GenerationContext` with `communicationStyle` from `IPersonalityModel`, `moodInfluence` derived from `ExperientialState`, and `relevantMemories` from `IEpisodicMemory`.
- **Invariants:** Session methods throw `Error` if `sessionId` is not found. Active topics bounded to ≤10 entries. Working memory window bounded to `WORKING_MEMORY_WINDOW` most recent turns.

### IInnerSpeechEngine
- **Preconditions:** `generate()` receives a valid `sessionId`, `turnIndex`, non-empty `prompt`, and `ExperientialState` with `valence`/`arousal`/`unityIndex`.
- **Postconditions:** `generate()` returns an `InnerSpeechRecord` with `isExternalised: false`, `id` formatted as `inner:{sessionId}:{turnIndex}`, and `text` containing an affect clause derived from valence/arousal plus the deliberation prompt. `externalise()` returns a new record with `isExternalised: true`.
- **Invariants:** Records are keyed by `(sessionId, turnIndex)`. `getRecords()` returns records ordered by `turnIndex`. `generate()` never calls an external LLM.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|---|---|---|---|---|---|
| MEMORY_WEAVE_THRESHOLD | 0.7 | score (0..1) | 0.5–0.9 | Memories below this relevance score are not woven into language output. Too low → irrelevant references; too high → agent never references past conversations. | Medium — directly affects whether cross-session recall appears in output |
| WORKING_MEMORY_WINDOW | 8 | turns | 4–16 | Number of recent dialogue turns kept in GenerationContext. Balances context richness against token cost. | Low — affects context length but not correctness |
| MEMORY_TOP_K | 5 | count | 3–10 | Maximum episodic memories retrieved per topic for GenerationContext assembly. | Low — affects retrieval breadth |
| MEMORY_RELEVANCE_THRESHOLD | 0.5 | score (0..1) | 0.3–0.8 | Minimum composite score for an episodic memory to be included in GenerationContext.relevantMemories. | Medium — filters out low-relevance memories from context |
| ACTIVE_TOPICS_MAX | 10 | count | 5–20 | Maximum active topics tracked per session before oldest are evicted. | Low — bounding mechanism only |
| GROUNDING_CONFIDENCE_THRESHOLD | 0.7 | score (0..1) | 0.5–0.9 | When intent confidence is below this, a ClarificationRequest is generated. Currently implemented as `intent === 'unknown'` heuristic. | Medium — affects how often the agent asks clarifying questions |

## Behavioral Spec

### Given/When/Then: Full Turn Lifecycle (AC1: no zombie bypass)
- **Given** a started dialogue session and a `LinguisticPerceptionAdapter` wrapping `IPerceptionPipeline`
- **When** raw user text is processed via `toSensorData()` → `pipeline.ingest()` → `extractFeatures()` → `ConsciousCore.processPercept()` → `deliberate()` → `assembleGenerationContext()` → `render()`
- **Then** the resulting `Decision.experientialBasis` is non-null, `ActionPipeline.execute()` is called with that decision, and the rendered output string is non-empty

### Given/When/Then: Personality Influence (AC2)
- **Given** two `DialogueManager` instances with different `IPersonalityModel` profiles (verbose+formal vs terse+casual)
- **When** both render the same `Decision` with a short utterance
- **Then** the verbose agent produces longer output (expansion clause appended) and the formal agent replaces contractions (`I'm` → `I am`)

### Given/When/Then: Emotional Tone (AC3)
- **Given** an `ExperientialState` with `valence=0.8, arousal=0.9` (enthusiastic) vs `valence=-0.5, arousal=0.2` (subdued)
- **When** both states are used to derive `MoodInfluence` and render via `LinguisticActionExecutor`
- **Then** the enthusiastic state produces output prefixed with "Absolutely!" and the subdued state produces output prefixed with "I see."

### Given/When/Then: Cross-Session Recall (AC5)
- **Given** a completed session A with turns about "consciousness" archived to episodic memory
- **When** a new session B calls `recallPriorConversations()` with topic `["consciousness"]`
- **Then** a non-null preamble string containing "prior conversation" is returned

### Given/When/Then: Value-Action Gate Blocking (AC7)
- **Given** a `Decision` with `action.type === "blocked"`
- **When** `LinguisticActionExecutor.render()` is called with this decision
- **Then** an `Error` with message "speech act blocked by Value-Action Gate" is thrown and `ActionPipeline.execute()` is never called

### Given/When/Then: Inner Speech Observability (AC8)
- **Given** an `InnerSpeechEngine` and an `ExperientialState` with positive valence and high arousal
- **When** `generate()` is called and then `externalise()` is called on the record
- **Then** `getRecords()` returns the record with `isExternalised: true`, and when included in `GenerationContext`, the rendered output contains "Let me think about that…" followed by the inner speech text

## Acceptance Criteria
- Language input flows through the full conscious pipeline: SensorData → Percept → ExperientialState → Decision → ActionResult (no zombie bypass)
- Language output reflects personality (two agents with different personality profiles produce observably different language on the same input)
- Language output reflects emotional state (agent's tone shifts measurably under different mood conditions)
- Conversational context persists across turns via working memory, and across sessions via episodic memory
- The agent references past conversations appropriately ("Last time we discussed...")
- The agent can explain its reasoning in experiential terms per the ethical governance justification requirement
- The agent can refuse to produce language that would violate its values (Value-Action Gate applies to speech acts)
- Inner speech is observable in introspection reports
- Integration test: agent engages in multi-turn dialogue, references episodic memory from a prior conversation, and produces language shaped by its personality and current emotional state
