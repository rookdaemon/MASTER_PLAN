# 0.7.3.3.2.2.1 Generate Evaluation Matrix [PLAN]

## Description
Create a YAML matrix that lists every top‑level (H2) section from the outline (produced in 0.7.3.3.2.1) and scores it on visibility, relevance, and expected disruption (1‑5 scale).

## Acceptance Criteria
1. `artifacts/0.7.3.3.2-candidate-evaluation.yaml` exists and contains an entry for each H2 section.
2. Each entry includes three numeric scores: `visibility`, `relevance`, `disruption`.
3. Scores are documented with a brief rationale (≤ 30 words per criterion).
4. The matrix file is added to version control under `artifacts/`.
5. The file follows the project's YAML schema (validated with `yamllint`).
