# 0.7.3.1.1.2.2 Format Summary [PLAN]
## Description
Transform the raw occurrence data from the previous subtask into the final markdown snippet. Include a one‑sentence overview of the codebase map’s layout and a placeholder line if the entry is missing.

## Acceptance Criteria
1. The snippet begins with a single sentence describing the map’s structure (e.g., “The codebase map uses heading levels for modules and bullet points for files”).  
2. For each occurrence, a line reads “Found at line X (or anchor Y)”.  
3. If the entry is absent, a line reads “Entry not found – insertion placeholder needed”.  
4. The snippet is valid markdown and ready to be copied into the map.  
5. No external files are altered.
