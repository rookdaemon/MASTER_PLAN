# 0.7.3.1.3.3.2 Run Extraction Test [PLAN]

Execute the verification script in a clean CI‑style environment to ensure:
- The zip archive extracts without errors.
- Temporary directories are cleaned up after the run.
- The script’s exit code and log output meet expectations for a known‑good archive.

**Acceptance Criteria**
1. Test runs in a Docker container (e.g., `python:3.11-slim` or `alpine`) with no pre‑existing files.  
2. Extraction completes within a reasonable time (< 30 seconds for a typical package).  
3. No residual files remain after the test (cleanup verified).  
4. Successful run yields exit code 0 and a log containing “Verification succeeded”.  
5. Failure scenario (e.g., deliberately corrupt zip) produces a non‑zero exit code and a log entry describing the error.
