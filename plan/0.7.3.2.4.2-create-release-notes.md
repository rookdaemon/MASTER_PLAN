# 0.7.3.2.4.2 Create Release Notes [PLAN]

**Description:**  
Compose the release notes document, embed version metadata, high‑level changes, links to the release archive, and cross‑references to related tasks.

**Acceptance Criteria**
1. A markdown file `release-notes/plan-guardian-0.7.3.${DATE}.md` is generated, where `${DATE}` is `YYYYMMDD` of the release.
2. The notes contain:
   * Version number `0.7.3.${DATE}`.
   * Release date (ISO‑8601).
   * A concise “High‑level Changes” section summarizing major improvements.
   * A hyperlink to the archive `plan-guardian-0.7.3.${DATE}.tar.gz` once it is uploaded.
   * References to tasks `0.7.3.1`, `0.7.3.2‑*` (wildcard for all sibling subtasks).
3. The release notes are programmatically posted to the central knowledge‑base via its API; the API response (including the permanent URL) is recorded in `logs/release-0.7.3.2.log`.
4. The `logs/release-0.7.3.2.log` file is attached to the knowledge‑base entry.
5. Failure to post results in a non‑zero exit code and an error entry in the log.
