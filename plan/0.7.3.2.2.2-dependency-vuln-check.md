# 0.7.3.2.2.2 Dependency Vulnerability Check [PLAN]

Perform automated vulnerability scanning of all third‑party dependencies used by the package.

**Acceptance Criteria**
1. Dependency list is extracted from lock files (e.g., `requirements.txt`, `package-lock.json`, `go.mod`).
2. Scanning is executed with at least two vulnerability databases (e.g., Snyk and OSV).
3. All discovered CVEs are recorded with CVE ID, severity, and affected version.
4. Any vulnerability rated Critical or High is flagged for immediate remediation.
5. Results are exported to `artifacts/dependency-vuln-report.json`.
