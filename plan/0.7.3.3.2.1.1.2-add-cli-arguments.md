# 0.7.3.3.2.1.1.2 Add CLI Arguments [PLAN]

## Description
Wrap the core function from Subtask 0.7.3.3.2.1.1.1 in a command‑line interface using `argparse`:
* `--input` / `-i` for the markdown source path.
* `--output` / `-o` for the YAML destination path.
* The script should call the core extractor, serialize the list to YAML (using `yaml.safe_dump`), and write the file.

## Acceptance Criteria
1. The script can be executed as `./scripts/extract_section_outline.py -i <md> -o <yaml>`.
2. It validates that both arguments are provided; missing arguments cause a helpful `argparse` error message.
3. The generated YAML exactly matches the required schema: a list of objects each containing `heading`, `text`, and `line`.
4. The script exits with status 0 on success and non‑zero on failure (e.g., file not found).
5. The file is made executable (`chmod +x`) and placed under `scripts/`.
