# 0.7.3.2.2.2.2 Generate YAML Dependency Map [PLAN]

**Description**  
Consume the JSON output from the parser, translate raw import strings into full dotted module IDs defined in the traceability map, and write `artifacts/module-dependencies.yaml` adhering to the schema `{ module: string, depends_on: string[] }`.

**Acceptance Criteria**
1. A conversion step resolves each imported path to a dotted ID (e.g., `src/utils/logger.ts` → `0.7.3.2.2.2.X`).  
2. The resulting data structure matches the schema exactly for every module.  
3. The YAML file is written to `artifacts/module-dependencies.yaml` with proper indentation and UTF‑8 encoding.  
4. A unit test verifies that the file can be parsed back into the expected JavaScript object.  
5. The step runs automatically after the parser script via an npm `postprocess` script.
