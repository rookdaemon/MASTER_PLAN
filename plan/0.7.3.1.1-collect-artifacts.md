# 0.7.3.1.1 Collect Provenance Artifacts [PLAN]

Gather every provenance artifact required for the traceability package.

* Requirements documentation (user stories, functional specs).  
* Design decisions (architecture diagrams, RFCs, design logs).  
* Code links (source‑file references, commit hashes, CI pipelines).  
* Test artifacts (unit‑test reports, integration‑test results, coverage data).

**Acceptance Criteria**

1. All three child tasks are completed.  
2. The folder `traceability/plan-guardian/artifacts/` contains the required sub‑folders.  
3. All artifacts are present in the appropriate sub‑folders.  
4. `traceability/plan-guardian/artifacts/INDEX.md` correctly lists every artifact with relative paths and the latest commit hash.  
5. A final checklist confirms no missing references.
