# 0.7.3.1.1 Collect Provenance Artifacts [PLAN]

Gather every provenance artifact required for the traceability package:

* Requirements documentation (user stories, functional specs).  
* Design decisions (architecture diagrams, RFCs, design logs).  
* Code links (source‑file references, commit hashes, CI pipelines).  
* Test artifacts (unit‑test reports, integration‑test results, coverage data).

**Acceptance Criteria**

1. A folder `traceability/plan-guardian/artifacts/` exists with sub‑folders `requirements/`, `design/`, `code/`, `tests/`.  
2. Every artifact from the current release is copied or linked into the appropriate sub‑folder.  
3. A master index file `traceability/plan-guardian/artifacts/INDEX.md` lists all collected items with relative paths.  
4. All files are version‑controlled (git) and the latest commit hash is recorded in `INDEX.md`.  
5. No missing references: a checklist verifies that each requirement, design decision, code module, and test case appears at least once.
