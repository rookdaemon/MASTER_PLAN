# 0.7.3.1.1.2.1.2 Collect Code Artifacts [PLAN]

For each requirement ID, locate all concrete code artifacts (file paths, classes, functions) that implement it, using static analysis and existing documentation.

### Acceptance Criteria
1. A searchable index (e.g., `docs/traceability/code-index.json`) maps requirement IDs to an array of code artifact descriptors (`{path, type, identifier}`).  
2. Every requirement ID from `req-ids.csv` has at least one associated artifact; missing links are logged for manual review.  
3. No artifact is linked to a requirement that does not exist in `req-ids.csv`.  
4. The collection process is automated via a script (`scripts/collect-code-artifacts.py`) that can be re‑run on demand.  
5. The lead developer validates a random sample of 10 mappings and approves the script output.
