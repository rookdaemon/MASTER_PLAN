# 0.7.3.2.2.3.1 Generate Merged Codebase Map [PLAN]

**Description**  
Consume `artifacts/ownership-inventory.yaml` and `artifacts/module-dependencies.yaml` and produce a combined, machine‑readable `artifacts/codebase-map.yaml` where each module entry contains: `path`, `owners`, and a `depends-on` list.

**Acceptance Criteria**
1. `artifacts/codebase-map.yaml` is created and committed.  
2. Every module from both source artifacts appears exactly once.  
3. For each module the `owners` field matches the ownership inventory.  
4. The `depends-on` list mirrors the extracted dependency data.  
5. The generation script is idempotent and runs in CI without errors.
