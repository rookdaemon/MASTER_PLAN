# 0.7.3.1.1.1.3 Generate Provenance Output [PLAN]

Merge the plan‑file inventory and artifact‑hash data, produce the canonical provenance JSON file, commit it, and create the accompanying documentation.

**Acceptance Criteria**
1. A driver script `scripts/build_provenance.py` that reads `temp/plan_files.json` and `temp/artifact_hashes.json`, merges them into a single ordered array, and writes `artifacts/provenance/provenance.json`.
2. The script adds a top‑level field `generated_at` (ISO‑8601 timestamp) and `generated_by` (`plan‑guardian`).
3. After execution, the file `artifacts/provenance/provenance.json` is added, staged, and committed with a message `"chore: add provenance inventory"` (use `git add` and `git commit` via subprocess).
4. `docs/provenance.md` is created/updated to include:
   * a short description of the provenance file purpose,
   * instructions on how to regenerate it (`scripts/build_provenance.py`),
   * a link to the JSON file.
5. End‑to‑end test `tests/test_provenance_full_cycle.py` runs the three scripts in order and asserts that the final JSON contains at least 5 entries (plan files + artifacts) and that the Git commit exists.
