# 0.7.3.1.1.3.1.3 Export markdown – write file, ensure idempotency [PLAN]

Finalize the script to:

1. Write the markdown table to `exports/traceability.md`.
2. Overwrite any existing file safely (write to a temporary file then rename).
3. Ensure that repeated executions produce identical file content (hash‑based comparison optional).
4. Log a concise success message with the output path.

**Acceptance Criteria**
- After execution, `exports/traceability.md` exists and matches the expected content exactly.
- Running the script a second time does not change the file (checksum before/after is identical).
- The script exits with status 0 on success and non‑zero on error.
- The script can be invoked as `python tools/export_traceability_md.py` (no flags required for the final run).
- CI step can verify idempotency by comparing file hashes on two consecutive runs.
