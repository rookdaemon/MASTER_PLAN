# 0.7.3.1.1.1.2 Implement Provenance Generator [PLAN]

Write a script (`scripts/generate-provenance.js`) that scans the repository for deliverables and produces `provenance.json` conforming to the schema.

**Acceptance Criteria**
1. The script discovers all files in `artifacts/` and `plan/` that are marked as deliverables (e.g., via a `deliverable: true` front‑matter flag or a naming convention).
2. For each deliverable, the script records a unique `id` (e.g., `deliverable-<hash>`), its `type` (e.g., `artifact`, `plan`), and an array `artifactPaths` with relative paths.
3. The output `provenance.json` validates against `schemas/provenance-schema.json`.
4. The script is idempotent: running it multiple times rewrites the same file without duplicating entries.
5. A `package.json` script entry `npm run provenance` invokes the generator, and running it commits the file with the message “Add provenance data for deliverables”.
