# 0.7.3.1.2.1.1.3 Setup CI Hook for Schema Presence [PLAN]

Add a CI step (or pre‑commit hook) that ensures `crossref-schema.yaml` is present on every push and that it passes syntax validation.

**Acceptance Criteria**
1. CI workflow includes a step that checks `[ -f traceability/schema/crossref-schema.yaml ]` and fails if the file is missing.  
2. The same step also runs the yamllint check from task 0.7.3.1.2.1.1.2 to enforce validity.  
3. The CI pipeline reports success only when the file exists and passes linting.  
4. The pre‑commit hook (if used) runs the same checks locally before allowing a commit.  
5. Documentation is updated to describe the new CI/check behavior for future contributors.
