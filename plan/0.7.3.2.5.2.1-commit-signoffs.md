# 0.7.3.2.5.2.1 Commit Sign‑offs to Repository [PLAN]

Create a commit that adds every collected sign‑off PDF/e‑signature to the repository under `signoffs/0.7.3.2/` using the naming convention `<reviewer-id>_signoff.pdf`.

**Acceptance Criteria**
1. All sign‑off files received from reviewers are present in the local `signoffs/0.7.3.2/` folder.  
2. Each file follows the naming pattern `<reviewer-id>_signoff.pdf`.  
3. A Git commit is made with a message that includes the review ID (e.g., “Add sign‑offs for review 0.7.3.2”).  
4. The commit is pushed to the main branch and CI verifies no merge conflicts.  
5. The commit SHA is recorded in the project log for traceability.
