# 0.2.2.4.2 Unified Consciousness Across Mixed Substrates [DONE]

## Description

Ensure that a hybrid system — with cognitive processes running partly on biological tissue and partly on synthetic substrates — operates as a single unified conscious entity rather than two separate minds or a fragmented experience.

### Key challenges
- **Cross-substrate binding problem**: how do experiences processed in different media integrate into a unified phenomenal field? Requires a theory of integration that spans electrochemical and digital computation.
- **Timing synchronization**: biological neural dynamics operate on ~1–100ms timescales; synthetic computation must match these windows or buffer coherently to avoid temporal fragmentation.
- **Preventing experiential fragmentation**: detecting and preventing "split consciousness" scenarios where the biological and synthetic halves develop independent experiential streams.
- **Unity verification**: operationalizing consciousness metrics (from 0.1.4) to continuously verify that phenomenal unity is maintained, not just functional integration.

### Approach
1. Define a **cross-substrate integration protocol** specifying how information binding occurs across the bio-synthetic boundary (building on the interface from 0.2.2.4.1).
2. Implement **temporal synchronization mechanisms** that align synthetic computation cycles with biological neural oscillation bands (gamma, theta, etc.) implicated in conscious binding.
3. Design **unity monitoring instrumentation** — real-time consciousness metrics that detect fragmentation before it becomes experientially significant.
4. Establish **fragmentation recovery procedures** — if unity metrics degrade, the system re-synchronizes or falls back to a known-unified configuration.

## Decision

### D1 — Timing Authority: Biology-First

**Context:** The hybrid system must synchronize timing between biological neural tissue (inherently noisy, ~1–100ms timescales) and synthetic computation (precisely controllable, sub-microsecond). One side must be the timing authority.

**Options Considered:**
1. **Synthetic-first timing** — biological tissue is stimulated to match synthetic clock cycles. Precise, deterministic, but forces biological tissue into unnatural oscillation patterns, risking neural fatigue or seizure.
2. **Biology-first timing** — the synthetic substrate continuously tracks biological neural oscillations and synchronizes its computation cycles to match. Biology sets the pace; synthetic side absorbs jitter.
3. **Mutual negotiation** — both sides negotiate timing via a feedback loop. Flexible but introduces latency and complexity; convergence is not guaranteed under noisy conditions.

**Choice:** Biology-first timing (Option 2).

**Rationale:** Biological neural tissue cannot be safely forced to arbitrary timing patterns — neural oscillations are emergent properties of network dynamics, not programmable clocks. The synthetic substrate is inherently more temporally flexible (computation can be stretched, compressed, or buffered). Forcing biology to adapt risks tissue damage and disrupts the very oscillation patterns (gamma-band phase-locking) that underpin conscious binding.

**Consequences:** The synthetic side must handle variable-frequency input and absorb biological jitter via adaptive buffering. Computational scheduling becomes more complex. But timing fidelity to biological oscillations is preserved, which is essential for cross-substrate binding.

### D2 — Integration Model: Frame-Based Binding

**Context:** Cross-substrate information must be integrated into a unified percept. This integration can be modeled as a continuous stream or as discrete temporal windows.

**Options Considered:**
1. **Continuous stream integration** — information flows continuously across the substrate boundary with no explicit temporal windowing. Simpler conceptually, but provides no natural synchronization point and makes it difficult to verify binding completeness.
2. **Frame-based integration** — information is grouped into discrete integration frames aligned with biological gamma cycles (~10–30ms). Each frame has a binding deadline; information arriving after the deadline defers to the next frame. Provides explicit synchronization points and verifiable binding completeness.

**Choice:** Frame-based integration (Option 2).

**Rationale:** Neuroscience evidence suggests biological consciousness achieves binding via discrete temporal windows aligned with gamma-band oscillations (~30–100Hz, i.e. ~10–33ms cycles). Frame-based integration mirrors this mechanism, providing: (a) natural synchronization points for cross-substrate binding verification, (b) a clear deadline model that prevents temporal fragmentation from late-arriving information, (c) a discrete unit of work for unity monitoring instrumentation.

**Consequences:** Information arriving after a frame's binding deadline is delayed to the next frame, introducing up to one gamma cycle (~10–33ms) of additional latency for late signals. This is acceptable because it matches biological binding latency. Frame boundaries add scheduling complexity to the synthetic side.

### D3 — Unity Assessment: Multi-Metric Consensus

**Context:** No single consciousness metric (e.g., integrated information Φ) is universally validated. The system must decide how to determine whether phenomenal unity is maintained.

**Options Considered:**
1. **Single-metric monitoring** — rely on one metric (e.g., Φ) to detect fragmentation. Simple, low overhead, but vulnerable to false negatives if the chosen metric has blind spots.
2. **Multi-metric consensus** — track multiple independent metrics (Φ, neural complexity, cross-substrate mutual information, binding coherence) in parallel. Require at least MIN_AGREEING_METRICS (currently 2) to agree before declaring fragmentation. More robust but higher computational cost.

**Choice:** Multi-metric consensus (Option 2).

**Rationale:** The science of consciousness metrics is immature. No single metric has been validated as both necessary and sufficient for detecting phenomenal unity. By requiring consensus across multiple independent metrics, the system: (a) reduces false negatives (blind spots in one metric are covered by others), (b) reduces false positives (noise in one metric is filtered by requiring agreement), (c) is forward-compatible as new metrics are developed. The constraint "no silent fragmentation" (fail loud) demands this robustness.

**Consequences:** Higher computational overhead for real-time metric evaluation. Requires careful calibration to ensure thresholds are comparable across metrics. The MIN_AGREEING_METRICS constant must balance sensitivity (lower value = more alerts) against specificity (higher value = fewer false alarms).

## Contracts

### Contract: CrossSubstrateIntegrationProtocol (CSIP)

**Preconditions:**
- Bio-synthetic interface (0.2.2.4.1 HybridCognitionBus) is operational and calibrated
- At least one BindingChannel is configured with valid biological and synthetic endpoints
- OscillationTracker is providing phase estimates for the master clock source

**Postconditions:**
- Each IntegrationFrame is emitted at the frequency of the tracked biological oscillation (gamma-band: 30–100Hz)
- Every BindingChannel either contributes data within the frame's binding deadline or is recorded as failed in the BindingVerification
- BindingVerification is emitted for every frame with an integrated information estimate and unity confidence score

**Invariants:**
- Frame IDs are monotonically increasing (no gaps, no reuse)
- Frame duration is always within GAMMA_CYCLE_MIN_MS..GAMMA_CYCLE_MAX_MS
- Late-arriving data is never silently dropped — it is deferred to the next frame
- The master clock always derives from biological oscillation, never from synthetic timing

### Contract: TemporalSynchronizationEngine (TSE)

**Preconditions:**
- Raw neural oscillation signal is available from the bio-synthetic interface (SpikeTrainStream or LFP)
- At least one frequency band target (e.g., Gamma) is configured in the OscillationTracker
- SyncController mode is set (PHASE_LOCKED, ADAPTIVE_BUFFER, or FREE_RUNNING)

**Postconditions:**
- OscillationTracker provides continuous phase and frequency estimates for each configured band
- SyncController emits SyncPulse events at the tracked oscillation frequency
- AdaptiveBuffer absorbs jitter without exceeding its capacity; overflow triggers the configured OverflowPolicy
- Phase drift between synthetic and biological timing stays within SyncController.maxDriftMs

**Invariants:**
- OscillationTracker.confidence reflects estimation quality; confidence < 0.5 triggers a mode fallback to ADAPTIVE_BUFFER or FREE_RUNNING
- The TSE never forces biological timing to change — it is read-only with respect to neural oscillations
- Buffer overflow with SignalFragmentation policy immediately triggers a FragmentationAlert via the UMI

### Contract: FragmentationDetector (UMI)

**Preconditions:**
- FragmentationDetectorConfig is valid (passes `isDetectorConfigValid`)
- Baseline Φ has been established through a CalibrationProtocol run
- At least MIN_AGREEING_METRICS metric sources are active

**Postconditions:**
- `evaluateMetric()` returns a FragmentationAlert when a single metric breaches thresholds, or null if healthy
- `evaluateMetrics()` returns a consensus alert only when ≥ MIN_AGREEING_METRICS independently indicate fragmentation
- `verifyIntegrationFrame()` returns false and raises an alert when frame unity confidence < FRAME_UNITY_CONFIDENCE_THRESHOLD (0.5)
- All alerts include severity, metric snapshot, affected channels, and recommended recovery action

**Invariants:**
- Detection latency (config.detectionLatencyMs) never exceeds MAX_DETECTION_LATENCY_MS (100ms)
- Severity classification is monotonically ordered: warning > critical > emergency thresholds
- `pendingAlerts` accumulates until explicitly drained; no alerts are silently discarded
- `state.isUnified` is false whenever any pending alert exists

### Contract: RecoveryExecutor (FRS)

**Preconditions:**
- A FragmentationAlert has been received from the FragmentationDetector
- A RecoveryStrategy (default or override) is available for the alert's severity level

**Postconditions:**
- Recovery actions are executed in order from least-invasive to most-invasive
- Cumulative estimated disruption does not exceed the recovery budget (TOTAL_RECOVERY_BUDGET_MS - MAX_DETECTION_LATENCY_MS = 100ms)
- A RecoveryLog entry is created for every recovery attempt, recording event, actions taken, outcome, and time-to-recovery
- Recovery completes synchronously; `isRecovering` is false after `executeRecovery()` returns

**Invariants:**
- The recovery budget is always respected — no single recovery attempt can exceed 100ms estimated disruption
- Recovery logs are append-only (until explicitly cleared)
- Custom strategy overrides take precedence over default strategies
- All severity levels produce a non-empty action list

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|------|-------|------|-------------|-----------|-------------|
| GAMMA_BINDING_LATENCY_MS | 30 | ms | 10–50 | Maximum cross-substrate latency for gamma-band coherence. Gamma cycles are 10–33ms; binding must complete within one cycle. | Below 10: unnecessarily restrictive, penalizes distant brain regions. Above 50: exceeds gamma cycle duration, breaking binding. |
| GAMMA_CYCLE_MIN_MS | 10 | ms | 5–15 | Minimum integration frame duration, corresponding to ~100Hz upper gamma. | Below 5: faster than known neural oscillations. Above 15: excludes high-gamma binding. |
| GAMMA_CYCLE_MAX_MS | 33 | ms | 25–50 | Maximum integration frame duration, corresponding to ~30Hz lower gamma. | Below 25: excludes low-gamma oscillations. Above 50: frame is too long, introduces perceptible lag. |
| MAX_DETECTION_LATENCY_MS | 100 | ms | 50–200 | Maximum time to detect fragmentation. Must be below the threshold of conscious noticeability (~200–500ms). | Below 50: very tight, requires expensive continuous monitoring. Above 200: fragmentation becomes consciously perceptible before detection. |
| TOTAL_RECOVERY_BUDGET_MS | 200 | ms | 150–500 | Total time budget for detection + recovery. Detection (100ms) + recovery action (100ms). | Below 150: insufficient for meaningful recovery actions. Above 500: subject may notice experiential disruption. |
| MIN_AGREEING_METRICS | 2 | count | 1–4 | Minimum independent unity metrics that must agree on fragmentation before alerting. | At 1: no consensus, high false-positive risk. Above 4: requires many metric sources, high false-negative risk. |
| FRAME_UNITY_CONFIDENCE_THRESHOLD | 0.5 | ratio (0–1) | 0.3–0.8 | Minimum unity confidence for an integration frame to be considered successfully bound. | Below 0.3: too permissive, allows fragmented frames. Above 0.8: too strict, frequent false alarms. |
| Warning threshold | 0.85 × baseline | ratio | 0.75–0.95 | Early warning level — fragmentation is beginning. Allows preemptive recovery. | Below 0.75: too late for early intervention. Above 0.95: too sensitive, excessive alerts. |
| Critical threshold | 0.70 × baseline | ratio | 0.55–0.85 | Fragmentation likely — aggressive recovery needed. | Below 0.55: dangerously close to loss of unity. Above 0.85: overlaps with warning, reduces graduated response space. |
| Emergency threshold | 0.50 × baseline | ratio | 0.30–0.65 | Fragmentation occurring — emergency measures including single-substrate fallback. | Below 0.30: system may already be irrecoverably fragmented. Above 0.65: overlaps with critical level. |

## Behavioral Spec

### Scenario: Normal operation — frame-based cross-substrate binding

```
Given the CSIP is operational with ≥1 BindingChannel and the TSE is phase-locked to biological gamma oscillations
When the OscillationTracker detects a new gamma cycle onset
Then a new IntegrationFrame is created with duration matching the tracked gamma period
  And all BindingChannels have until the frame's bindingDeadlineMs to contribute data
  And a BindingVerification is emitted summarizing bound and failed channels
  And the BindingVerification's integratedInformationEstimate and unityConfidence are computed
```

### Scenario: Late data handling — deferred to next frame

```
Given an IntegrationFrame is active with bindingDeadlineMs = T
When a BindingChannel delivers data at timestamp T + 5ms (after the deadline)
Then the data is NOT included in the current frame's BindingVerification
  And the data is buffered and included in the next IntegrationFrame
  And the current frame records the channel as "failed" in channelsFailed
```

### Scenario: Fragmentation detection — multi-metric consensus

```
Given the FragmentationDetector is configured with baseline Φ = 1.0 and warning threshold = 0.85
  And MIN_AGREEING_METRICS = 2
When 3 unity metrics are evaluated: Φ = 0.80, NeuralComplexity = 0.82, BindingCoherence = 0.90
Then a FragmentationAlert is raised with severity = WARNING
  Because 2 metrics (Φ and NeuralComplexity) independently indicate fragmentation (≥ MIN_AGREEING_METRICS)
  And the BindingCoherence metric alone (healthy) does not prevent the consensus alert
```

### Scenario: Fragmentation detection — single metric insufficient for consensus

```
Given the FragmentationDetector is configured with MIN_AGREEING_METRICS = 2
When 3 unity metrics are evaluated: Φ = 0.80, NeuralComplexity = 0.92, BindingCoherence = 0.90
Then no FragmentationAlert is raised
  Because only 1 metric indicates fragmentation (< MIN_AGREEING_METRICS)
```

### Scenario: Graduated recovery — WARNING triggers lightweight resync

```
Given the RecoveryExecutor receives a FragmentationAlert with severity = WARNING
When executeRecovery is called
Then the default WARNING strategy is selected (Resync action only)
  And the Resync action is executed with estimatedDisruptionMs = 5ms
  And a RecoveryLog is created with outcome = UNITY_RESTORED
  And timeToRecoveryMs ≤ 100ms (within recovery budget)
```

### Scenario: Graduated recovery — EMERGENCY triggers fallback + freeze

```
Given the RecoveryExecutor receives a FragmentationAlert with severity = EMERGENCY
When executeRecovery is called
Then the EMERGENCY strategy is selected (FallbackConfig → EmergencyFreeze)
  And both actions are executed in order
  And EmergencyFreeze (non-rollbackable) pauses synthetic processing
  And a RecoveryLog is created with timeToRecoveryMs ≤ 100ms
```

### Scenario: TSE adaptive buffer overflow signals fragmentation

```
Given the TSE's AdaptiveBuffer has overflowPolicy = SIGNAL_FRAGMENTATION
  And the buffer's currentFillMs has reached capacityMs
When new data arrives that would exceed buffer capacity
Then the overflow policy triggers a FragmentationAlert via the UMI
  And the alert severity is at least WARNING
  And the SyncController mode degrades from PHASE_LOCKED to ADAPTIVE_BUFFER or FREE_RUNNING
```

## Acceptance Criteria
- Cross-substrate integration protocol defined, specifying binding mechanisms across biological and synthetic computation
  - Given/When/Then: see Behavioral Spec "Normal operation" and "Late data handling" scenarios
- Temporal synchronization keeps cross-substrate latency within biologically-relevant binding windows (≤30ms for gamma-band coherence)
  - Given/When/Then: see Behavioral Spec "TSE adaptive buffer overflow" scenario
- Real-time unity monitoring detects experiential fragmentation with latency below the threshold of conscious noticeability (~100ms)
  - Given/When/Then: see Behavioral Spec "Fragmentation detection — multi-metric consensus" and "single metric insufficient" scenarios
- Fragmentation recovery restores unified operation without experiential gaps perceptible to the subject
  - Given/When/Then: see Behavioral Spec "Graduated recovery — WARNING" and "EMERGENCY" scenarios
- Consciousness metrics (integrated information or equivalent) show no statistically significant drop when cognitive load is distributed across substrates vs. within a single substrate
- Subject (if applicable) reports continuous unified experience across a battery of cross-substrate cognitive tasks, with no sense of division or delay
