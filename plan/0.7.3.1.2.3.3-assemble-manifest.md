# 0.7.3.1.2.3.3 Assemble Manifest [PLAN]

Combine the TOC, per‑file tables (derived from `tmp/metadata.json`), and a summary section into `TRACEABILITY-MANIFEST.md`.

**Acceptance Criteria**
1. The manifest file is created at the repository root with the exact name `TRACEABILITY-MANIFEST.md`.  
2. For each file in `inventory.txt` a Markdown table row is added containing all required fields (filename, size, hash, etc.).  
3. A summary section reports the total number of files and the cumulative size in bytes and in a human‑readable format (e.g., KB/MB).  
4. The final document passes the linting step that checks Markdown syntax and validates that all internal links (including TOC anchors) are correct.  
5. The assembled manifest matches a provided golden file for the reference data set (binary diff yields no differences).
