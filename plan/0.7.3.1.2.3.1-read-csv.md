# 0.7.3.1.2.3.1 Read CSV [PLAN]

Parse `file‑metadata.csv` and produce an in‑memory representation (e.g., list of dictionaries) containing all required fields for each file listed in `inventory.txt`.

**Acceptance Criteria**
1. The CSV is read without errors on the supplied test data set.  
2. Every row is validated to contain all required columns (filename, size, hash, etc.).  
3. Rows for files not present in `inventory.txt` are ignored; missing rows for inventory entries cause a validation error.  
4. The resulting data structure is serialized to `tmp/metadata.json` for downstream steps.  
5. Unit tests verify handling of empty lines, quoted fields, and non‑ASCII characters.
