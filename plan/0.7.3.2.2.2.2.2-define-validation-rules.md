# 0.7.3.2.2.2.2.2 Define Validation Rules [PLAN]
## Description
For each field in the draft schema, specify precise validation constraints:
- Regex patterns for strings  
- Minimum/maximum values for numbers  
- Enumerated allowed values for enums  
- Length limits, format checks (e.g., email, UUID)  

Update the JSON Schema with these constraints.

## Acceptance Criteria
1. All fields have concrete validation rules attached in `schema-with-rules.json`.  
2. Regex patterns are tested against at least two positive and two negative examples per field.  
3. Enumerations include a documented business justification for each allowed value.  
4. Length and range limits are justified and recorded in accompanying markdown.  
5. The updated schema validates successfully against a set of synthetic test instances.
