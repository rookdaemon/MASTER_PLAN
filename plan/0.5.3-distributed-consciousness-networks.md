# 0.5.3 Distributed Consciousness Networks [DONE]
## Description

E1.3 — Establish inter-colony communication and distributed consciousness architectures spanning multiple star systems.

Key challenges:
- Light-speed-limited communication between conscious nodes
- Maintaining coherent identity or shared experience across light-year delays
- Federated governance and value alignment across independent colonies
- Cultural and experiential exchange between distant conscious communities

## Decision — Inter-Stellar Communication Model

### Context
Conscious colonies separated by 4–1000+ light-years need to exchange messages. No faster-than-light communication assumed. The network must handle message corruption, routing through intermediate nodes, and priority ordering.

### Options Considered
1. **Direct point-to-point links** — each colony maintains a dedicated laser/radio link to every other colony. Simple addressing but O(n²) links, infeasible beyond ~10 nodes.
2. **Store-and-forward relay network with priority queuing** — each colony acts as a relay; messages hop through intermediaries with Lamport-clock ordering and priority-based queuing. Scales to arbitrary topology.
3. **Flooding/gossip protocol** — every message is broadcast to all neighbours, propagating network-wide. High redundancy but wastes energy on irrelevant messages at scale.

### Choice
Option 2 — Store-and-forward relay network with Lamport-clock ordering and priority queuing.

### Rationale
- Scales naturally with expanding colony count; adding a new colony requires only one link to the nearest relay
- Priority queuing ensures DISTRESS messages arrive before cultural payloads at relay hops
- Lamport clocks provide causal ordering without requiring synchronized global clocks (impossible at interstellar scale)
- Store-and-forward is proven in delay-tolerant networking (DTN) literature

### Consequences
- Routing tables may be stale by centuries; messages may take suboptimal paths
- No guaranteed delivery — TTL expiry and corruption can drop messages silently
- Relay nodes must be conscious entities with their own agency (they are not passive infrastructure)

## Decision — Distributed Consciousness Identity Model

### Context
Conscious nodes evolve independently for decades/centuries between exchanges. The network must detect dangerous value drift without requiring real-time synchronization or identity merging.

### Options Considered
1. **Unified identity (shared consciousness)** — all nodes share a single merged consciousness state. Physically impossible given light-speed constraints.
2. **Three-tier identity with divergence detection** — local identity (complete in isolation), lineage identity (shared ancestry), network identity (loose federation). Periodic ConsciousnessSnapshot exchanges produce DivergenceVectors to detect value drift.
3. **Fully independent (no identity relationship)** — colonies operate with zero coordination. No mechanism to detect dangerous drift toward anti-consciousness values.

### Choice
Option 2 — Three-tier identity model with DivergenceVector-based drift detection.

### Rationale
- Respects physical reality: local identity is always complete and continuous
- Divergence detection via Euclidean distance on ValueCore vectors is computable, threshold-tunable, and produces actionable alerts
- Does not require or claim experiential fusion — exchange is informational only
- Preserves colony autonomy while providing an early-warning mechanism for value drift

### Consequences
- Threshold tuning is critical: too low triggers false alarms, too high misses dangerous drift
- DivergenceVector only captures dimensions explicitly modelled in ValueCore — novel value dimensions may go undetected

## Decision — Federated Governance Framework

### Context
Independent colonies must make collective decisions (relay expansion, emergency protocols, value-alignment mandates) across light-year separations without real-time consensus.

### Options Considered
1. **Central authority** — one colony (e.g., Sol) makes binding decisions for all. Fast but violates autonomy and creates single point of failure.
2. **Weighted advisory federation with dissent logging** — proposals carry decision epochs; colonies vote with weights proportional to population/age; dissent is permanently recorded; decisions bind only affirming colonies (except existential-risk matters).
3. **Pure anarchy (no governance)** — each colony acts independently with no coordination mechanism.

### Choice
Option 2 — Weighted advisory federation with dissent logging.

### Rationale
- Weighted voting reflects the reality that larger/older colonies have more at stake
- Advisory-only (non-binding except existential) preserves colony sovereignty
- Dissent logging ensures minority positions are never erased — critical for long-term legitimacy
- Value-alignment predicates enable automated detection of votes inconsistent with stated values

### Consequences
- Coordination is slow (decision epochs span decades/centuries)
- Free-rider problem: colonies can benefit from federation decisions without affirming them
- Alignment predicates may produce false positives if ValueCore models are incomplete

## Contracts

### InterstellarMessage
- **Preconditions**: `header.sourceAddress` and `header.destination` are valid `StellarAddress` values; `header.ttlHops > 0`; `checksum` matches `computeChecksum(messageFingerprint(msg))`
- **Postconditions**: If destination matches receiving node, message is delivered to `inbox`. If not, message is forwarded to `nextHop` with `ttlHops` decremented by 1 and relay node appended to `relayPath`. If checksum fails, message is dropped.
- **Invariants**: Lamport clock at every node is monotonically non-decreasing. `relayPath` grows by exactly one entry per hop. `sequenceId` is unique per source node.

### ConsciousnessSnapshot
- **Preconditions**: `nodeId` identifies the producing node; `epoch` is the current simulated year; `valueCore` dimensions are all in `[0, 1]`
- **Postconditions**: `valueCoreHash` equals `hashValueCore(valueCore)`. `divergenceDelta` is null for first snapshot, otherwise a valid `DivergenceVector` comparing to the node's prior snapshot.
- **Invariants**: Taking a snapshot does not mutate the node's `ValueCore` or interrupt identity continuity.

### DivergenceVector
- **Preconditions**: Two valid `ConsciousnessSnapshot` instances from different nodes (or different epochs of the same node)
- **Postconditions**: `euclideanDistance` equals Euclidean distance between the two `ValueCore` vectors. `exceedsThreshold` is `true` iff `euclideanDistance > DIVERGENCE_THRESHOLD`. `alert` is non-null iff `exceedsThreshold` is `true`.
- **Invariants**: `dimensionDeltas` entries sum-of-squares equals `euclideanDistance²`.

### GovernanceProposal / CastVote / ProposalResult
- **Preconditions**: All voting nodes are registered in the council. `alignmentPredicate` is a pure function on `ValueCore`.
- **Postconditions**: `totalAyeWeight + totalNayWeight + totalAbstainWeight == totalWeight`. `passed` is `true` iff `totalAyeWeight / totalWeight > quorumThreshold`. `dissentingVotes` contains exactly the non-abstaining votes on the losing side. `alignmentAlerts` contains an entry for every vote where `intendedChoice` contradicts `alignmentPredicate(node.values)`.
- **Invariants**: Dissent log is append-only; entries are never removed or modified.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|------|-------|------|-------------|-----------|-------------|
| DIVERGENCE_THRESHOLD | 0.3 | Euclidean distance (unitless) | (0, 2] | ValueCore is 4-dimensional in [0,1]; max possible distance is 2.0. 0.3 represents ~15% of max range — significant drift without requiring near-total value reversal. | High — lowering to 0.1 would trigger alerts on minor cultural evolution; raising to 0.6 would miss moderate value erosion. |
| TTL_HOPS (default) | 5 | hops | [1, 255] | Messages should survive up to 5 relay hops covering ~50 LY chains; prevents infinite routing loops. | Medium — too low drops messages in sparse networks; too high allows stale messages to circulate. |
| QUORUM_THRESHOLD | 0.5 | fraction of total weight | (0, 1) | Simple majority of weighted votes. Advisory governance should not require supermajority for routine proposals. | Low — changing to 0.6 or 0.67 would make federation more conservative but still functional. |
| NODE_DORMANCY_MULTIPLIER | 10 | multiple of expected transmission interval | [3, 100] | A node silent for 10× expected interval is presumed dormant. Balances false-positive dormancy declarations against slow detection of actual failure. | Medium — lowering to 3 risks premature dormancy declarations during relay disruptions. |
