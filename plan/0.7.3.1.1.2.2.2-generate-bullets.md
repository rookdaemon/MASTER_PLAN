# 0.7.3.1.1.2.2.2 Implement Bullet Generation Logic [PLAN]

Write code that consumes the parsed provenance data and produces markdown bullet strings according to the specification defined in `0.7.3.1.1.2.2.1`.

**Description**
- Load provenance data structures produced by earlier parsing steps.
- Apply the bullet template to each record, handling optional fields gracefully.
- Output the bullets to a temporary file (`output/bullets.tmp.md`).

**Acceptance Criteria**
1. A runnable script/module (e.g., Python, Node) that reads `output/provenance.json` and writes `output/bullets.tmp.md`.
2. Generated bullets match the format spec for at least three test cases (typical, minimal, maximal).
3. The script includes error handling for missing or malformed fields.
4. Unit tests covering the bullet generation logic pass (≥80% coverage).
5. Status moves to `[DONE]` after successful test run and code review.
