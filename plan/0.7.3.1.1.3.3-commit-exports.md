# 0.7.3.1.1.3.3 Commit traceability exports to repository [PLAN]

Add `exports/traceability.md` and `exports/traceability.json` to version control in a single commit.

**Acceptance Criteria**
1. Both export files exist at the expected paths before the commit step.
2. A git commit is created with the exact message: **“Export traceability package”**.
3. The commit includes only the two export files (no unrelated changes).
4. The commit operation is safe to repeat: if a commit with the same SHA already exists, the task reports success without creating a duplicate.
5. After the commit, the repository is clean and any CI checks (if present) pass.
