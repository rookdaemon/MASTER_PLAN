# 0.7.3.1.1.1.3.3 Add End‑to‑End Provenance Tests [PLAN]

Implement an automated test suite that runs the three provenance‑related scripts in order, verifies the JSON output, and ensures the Git commit was made.

**Acceptance Criteria**
1. Test file `tests/test_provenance_full_cycle.py` invokes (a) the plan‑file collection script, (b) the artifact‑hash collection script, and (c) `scripts/build_provenance.py`.
2. After execution, the test asserts that `artifacts/provenance/provenance.json` exists and contains at least five entries.
3. The test uses `git log` or `git rev‑parse` to confirm that a commit with the message `"chore: add provenance inventory"` is present.
4. The test cleans up any temporary files it creates and is idempotent.
5. The test suite passes on a fresh clone of the repository.
