# 0.7.3.2.4.1.2 Extract Ownership Information [PLAN]

**Description**  
For each module identified in the previous step, parse its header comments or associated metadata files to determine the responsible owner(s).

**Acceptance Criteria**
1. Ownership is read from a `@owner` JSDoc tag, a `// Owner:` comment, or a `module-meta.json` side‑car file if present.  
2. If no explicit owner is found, the module is flagged with `"owner": "UNKNOWN"` and logged for manual review.  
3. The ownership data is merged with the module list to produce `tmp/module-ownership.json`.  
4. The output conforms to the schema: `{ module: string, owners: string[] }`.  
5. Validation against the schema succeeds (no schema errors).
