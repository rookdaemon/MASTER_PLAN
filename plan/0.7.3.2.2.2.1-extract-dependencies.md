# 0.7.3.2.2.2.1 Extract Dependencies [PLAN]

Generate a complete, machine‑readable list of all third‑party dependencies from the project's lock files (`requirements.txt`, `package-lock.json`, `go.mod`, etc.).

**Acceptance Criteria**
1. Identify all supported lock file types present in the repository.
2. Parse each lock file and output a unified JSON file `artifacts/dependencies.json` containing package name, version, and source.
3. The JSON schema must include fields: `name`, `version`, `ecosystem`, `file`.
4. The extraction script runs without error on a fresh clone of the repo.
5. The script is added to `scripts/extract-dependencies.{sh,py}` and documented in `README.md`.
