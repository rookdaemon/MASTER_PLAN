# 0.7.3.1.1.1 Document Provenance of All Deliverables [PLAN]

Create a `provenance.json` file that lists every deliverable produced in this release, including its unique ID, type, and the artifact(s) that generated it.

**Acceptance Criteria**
1. `provenance.json` contains an array of objects, each with fields `id`, `type`, `artifactPaths`.
2. All artifact paths are relative and point to files inside the repository (`artifacts/...` or `plan/...`).
3. The JSON validates against the provided schema `schemas/provenance-schema.json`.
4. The file is committed with the message “Add provenance data for deliverables”.
5. Running the generation script twice does not duplicate entries (idempotent).
