# 0.7.3.2.4.2.1 Draft Sign‑off Table [PLAN]

Create a markdown table that lists each reviewer, their sign‑off status, and the date signed. Pull the data from the sign‑off records collected in **0.7.3.2.4.1**.

**Acceptance Criteria**
1. Table includes columns: Reviewer, Sign‑off Status, Date Signed.  
2. All reviewers from the sign‑off collection step appear in the table.  
3. Status values are normalized to “Approved”, “Pending”, or “Rejected”.  
4. Table is saved as `docs/signoff_table.md`.  
5. The file passes a markdown‑table lint check (no broken pipes, consistent column count).
