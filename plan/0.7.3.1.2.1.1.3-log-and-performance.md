# 0.7.3.1.2.1.1.3 Log errors and enforce performance [PLAN]

*Description*  
Create a logger that writes any fatal validation errors to `logs/parse-inventory-errors.log`. Ensure the log file is created (or appended to) atomically. After processing, verify total elapsed time does not exceed the 5‑second limit for a 10 k‑line file; if it does, emit a performance warning.

*Acceptance Criteria*
1. Errors are written to `logs/parse‑inventory‑errors.log` with timestamp, line number, and message.
2. The log file is created if missing; otherwise entries are appended.
3. After the read‑validate pipeline finishes, the elapsed wall‑clock time is measured.
4. If total time > 5 seconds for 10 k lines, a warning is logged to `logs/performance-warnings.log`.
5. No uncaught exceptions escape; the process exits cleanly with appropriate status codes (0 on success, non‑zero on fatal validation error).
