# 0.7.3.2.2.2 Extract Module Dependencies [PLAN]

**Description**  
Parse every source file in `src/` to build a `depends-on` list that records imports or calls to other modules. Represent each module as an entry with a list of sibling module IDs it depends on.

**Acceptance Criteria**
1. A parsing script (e.g., using AST tools) runs against all `*.js/ts` files under `src/`.  
2. For each module file, a `depends-on` array is generated containing the full dotted IDs of referenced modules.  
3. Output is stored as `artifacts/module-dependencies.yaml` following the schema `{ module: string, depends_on: string[] }`.  
4. The generated file passes JSON/YAML schema validation.  
5. Results are reviewed by a senior engineer for correctness.
