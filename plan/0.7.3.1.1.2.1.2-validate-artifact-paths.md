# 0.7.3.1.1.2.1.2 Validate Artifact Paths [PLAN]

For each deliverable produced by the previous step, verify that `artifact_path` points to an existing file under `artifacts/` or `plan/`.

**Acceptance Criteria**
1. Every `artifact_path` is resolved relative to the repository root.
2. If the file exists, the deliverable is marked as “valid”; otherwise a warning “Missing artifact for <id>: <artifact_path>” is logged.
3. The validation step does **not** abort the whole process; invalid entries are retained but flagged.
4. A summary log “Validation complete – X valid, Y missing” is emitted.
5. The filtered/annotated list is stored for the next step (`tmp/validated-provenance.json`).
