# 0.7.3.1.1.2.2 Map Code‑to‑Test Links [PLAN]

Identify for every code artifact from the requirement‑to‑code list the unit and/or integration tests that validate it.

### Acceptance Criteria
1. Each code artifact from the `req‑code-mapping` file has at least one associated test case ID.  
2. Test case IDs correspond to entries in the test suite registry (`tests/registry.md`).  
3. The mapping is recorded in `docs/traceability/code‑test-mapping.*` (CSV/JSON).  
4. The test‑to‑requirement relationship is cross‑checked so that every test references a valid requirement ID.  
5. The mapping receives sign‑off from QA lead.
