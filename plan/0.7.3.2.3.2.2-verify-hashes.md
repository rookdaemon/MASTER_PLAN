# 0.7.3.2.3.2.2 Verify File Hashes in Manifest [PLAN]

Validate that all file hashes recorded in the newly generated manifest match the current repository state.

**Description**
Compute SHA‑256 hashes for each repository file and compare them to the corresponding entries in the manifest produced by the previous sub‑task.

**Acceptance Criteria**
1. Every file listed in the manifest has a matching hash computed from the current checkout.  
2. No mismatched or missing hash entries are reported.  
3. A verification report `reports/hash-verification.txt` is generated summarizing the result (PASS/FAIL).  
4. The verification step exits with status 0 on success, non‑zero otherwise.  
5. The report is archived alongside the manifest for audit purposes.
