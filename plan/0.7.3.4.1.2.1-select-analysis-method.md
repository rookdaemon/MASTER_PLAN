# 0.7.3.4.1.2.1 Select Analysis Method [PLAN]

Identify and justify the root‑cause analysis technique to be used (e.g., 5 Whys, Fishbone/Ishikawa, Fault Tree).

**Acceptance Criteria**
1. Evaluate at least three candidate techniques against the project context (complexity, data availability, stakeholder familiarity).
2. Document the chosen technique with rationale in a one‑page brief.
3. Obtain sign‑off from the project lead and one stakeholder representative.
4. Create a reusable template (e.g., worksheet) for the selected technique.
5. Store the brief and template in the project repository under `analysis/`.
