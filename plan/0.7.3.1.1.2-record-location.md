# 0.7.3.1.1.2 Record Location [PLAN]
## Description
Create a concise summary of the findings from the identification step, including line numbers or textual anchors, and note a placeholder insertion point if the entry is absent. Also add a brief description of the overall format of the codebase map.

## Acceptance Criteria
1. A markdown snippet is produced that lists:  
   - “Found at line X (or anchor Y)” for each occurrence, **or** “Entry not found – insertion placeholder needed”.  
2. The snippet includes a one‑sentence overview of the map’s structure (e.g., headings, bullet style).  
3. The output is stored as part of the task’s result (no file modifications).  
4. The summary is clear enough for a downstream editor to add or update the entry without further clarification.  
5. No modifications are made to any source files.
