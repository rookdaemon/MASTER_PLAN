# 0.7.3.2.1.3.2 Assign Action Items [PLAN]

Translate each finding from the compiled report into a tracked work item, assign an owner, and set a due date in the project tracker.

**Acceptance Criteria**
1. For every finding, a corresponding ticket exists in the project tracker (`tracker/`), linked from the report.
2. Each ticket includes an owner (team member) and a concrete due date no later than the next sprint.
3. All tickets are tagged with the appropriate severity label.
4. A summary matrix (`review-output/action-items.xlsx`) is generated linking findings to tickets.
5. Stakeholders confirm that no critical findings are left unassigned.
