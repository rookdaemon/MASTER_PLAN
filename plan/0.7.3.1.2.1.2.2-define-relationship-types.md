# 0.7.3.1.2.1.2.2 Define Relationship Types [PLAN]

Specify the three relationship types (`references`, `implemented-by`, `validated-by`) including source/target entity constraints and cardinality.

**Acceptance Criteria**
1. Each relationship entry lists `source-type`, `target-type`, and `cardinality` (e.g., `many-to-many`, `one-to-many`).  
2. Allowed source/target entity combinations are documented (e.g., `design-decision` → `code` for `implemented-by`).  
3. Data types for relationship attributes are defined (all strings).  
4. Example instances for each relationship type are included in `schema/crossref.yaml`.  
5. The validation script confirms that every relationship in the schema respects the defined source/target constraints and cardinality.
