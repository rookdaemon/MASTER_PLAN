# 0.7.3.1.1.3.3.2 Add CI Validation for INDEX.md [PLAN]

Extend the CI pipeline to verify the correctness of `INDEX.md` after it is generated.

**Acceptance Criteria**
1. CI runs a verification step that parses `traceability/plan-guardian/artifacts/INDEX.md`.
2. It extracts the hash from the first line and confirms it matches `git rev-parse HEAD` (restricted to the `artifacts/` directory).
3. It reads the checklist section and ensures the line `- [x] No missing references` is present.
4. It extracts all Relative Paths listed in the embedded table and fails the job if any path does not exist on disk.
5. The CI step reports a clear pass/fail status and logs any mismatches for debugging.
