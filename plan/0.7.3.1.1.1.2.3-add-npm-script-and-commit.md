# 0.7.3.1.1.1.2.3 Add NPM Script and Commit Automation [PLAN]

Wire the generator into the project workflow.

**Acceptance Criteria**
1. `package.json` receives a script entry `"provenance": "node scripts/generate-provenance.js"`.
2. After running `npm run provenance`, a Git commit is created automatically (using `simple-git` or a shell command) with the exact message “Add provenance data for deliverables”.
3. The commit only includes `provenance.json`; no other files are staged.
4. If the repository is in a detached HEAD state or has uncommitted changes, the script aborts with a clear error message.
5. A CI step example is added to `.github/workflows/ci.yml` that runs `npm run provenance` and fails the build if validation errors occur.
