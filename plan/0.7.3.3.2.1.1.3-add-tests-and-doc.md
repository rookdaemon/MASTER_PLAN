# 0.7.3.3.2.1.1.3 Add Tests and Documentation [PLAN]

## Description
Create automated tests and a brief usage README for the extraction script.

## Acceptance Criteria
1. **Unit Tests** (`tests/test_extract_section_outline.py`) covering:
   * Correct identification of multiple H2 headings.
   * Preservation of case/punctuation.
   * Accurate line numbers.
   * Proper error handling for missing files.
2. **Integration Test** that runs the CLI on a sample markdown file and checks that the output YAML matches the expected content.
3. Add a `README.md` in `scripts/` that explains:
   * Purpose of the script.
   * Installation (`chmod +x`), usage example, and dependencies.
4. All tests pass (`pytest -q`) with 0 failures.
5. CI‑compatible (tests importable without the package being installed globally).
