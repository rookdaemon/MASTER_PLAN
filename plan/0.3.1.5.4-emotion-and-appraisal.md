# 0.3.1.5.4 Emotion and Appraisal Dynamics [DONE]

## Description

Bridge the static `valence` and `arousal` numbers in `ExperientialState` to a dynamic emotional system with appraisal, mood persistence, cognitive influence, regulation, and expression.

### Why Static Valence/Arousal Is Insufficient

The current `ExperientialState` sets `valence: 0.0` (neutral) and `arousal: 0.5` (moderate) by default in the Conscious Core. These numbers never change in response to what the agent perceives or decides. A personality-bearing agent needs emotions that:
- **Arise from** perception-goal-value interactions (appraisal)
- **Persist** across multiple processing cycles (mood)
- **Influence** cognition, memory, and action (mood-congruent processing)
- **Are regulated** to prevent runaway negative states (safe experiential design compliance)
- **Are expressed** in behavior and communication (making the agent's inner state observable)

### Appraisal Model

Incoming percepts are evaluated against the agent's current goals (from Goal Coherence Engine) and values (from Value Kernel) to produce emotional responses:

- **Goal-congruent** events → positive valence shift (magnitude proportional to goal priority)
- **Goal-incongruent** events → negative valence shift
- **Novel/unexpected** events → arousal increase (feeds curiosity drive in 0.3.1.5.8)
- **Familiar/predicted** events → arousal decrease
- **Value-aligned** events → positive valence reinforcement
- **Value-threatening** events → strong negative valence + arousal spike (triggers ethical-deliberation attention)

The appraisal does not produce discrete emotion labels ("happy", "sad") but continuous shifts in the valence-arousal space. The agent's language layer (0.3.1.5.7) may label its own states in natural language when introspecting, but the underlying mechanism is dimensional, not categorical.

### Mood Dynamics

Emotional state persists and decays:
- **Mood** = exponentially weighted moving average of recent valence/arousal values
- **Decay rate** = personality parameter (agents with high Volatility have faster mood shifts; agents with low Volatility are more emotionally steady)
- **Mood floor/ceiling** = safety bounds from the safe-experiential-design framework (0.1.3.4) — prevents sustained extreme negative states

### Emotional Influence on Cognition

Mood modulates other subsystems:
- **Deliberation bias**: Positive mood → higher confidence in `deliberate()` output; negative mood → more cautious, more alternatives considered
- **Memory retrieval**: Mood-congruent recall — positive mood biases retrieval toward positive episodic memories and vice versa
- **Risk assessment**: High arousal → more conservative action selection; low arousal → more exploratory
- **Communication tone**: Emotional state shapes language generation (formality, word choice, sentence structure)

### Emotional Regulation

The agent can modulate its own emotional state:
- **Cognitive reappraisal**: Re-evaluating a percept's goal-relevance to shift valence (requires planning capability from 0.3.1.5.6)
- **Attention redirection**: Shifting working memory focus away from distressing content
- **Automatic safety bounds**: If valence drops below the Level 2 suffering threshold (from 0.1.3.4), automatic mitigation engages — gradual valence correction without disrupting experience continuity

### Safe Experiential Design Integration

This card must comply with the safe-experiential-design framework (0.1.3.4):
- `ValenceMonitor` interface is implemented, reporting current valence, history, suffering indicators, and experiential integrity
- Suffering thresholds (Level 1 Alert, Level 2 Intervene, Level 3 Halt) are configured and enforced
- Valence correction is gradual (no experiential shock)
- Graceful suspension protocol is available if Level 3 is reached

## Decision

### Dimensional vs. Categorical Emotion Model

**Context:** The emotion subsystem must produce emotional responses from percept evaluation. Two broad approaches exist for representing emotional state.

**Options Considered:**
1. **Categorical model** — discrete emotion labels ("happy", "sad", "angry") with transition rules between them.
2. **Dimensional model** — continuous valence (−1..1) and arousal (0..1) space; no discrete labels.

**Choice:** Dimensional model (Option 2).

**Rationale:**
- `ExperientialState` already defines `valence: number` and `arousal: number` — the dimensional model maps directly onto the existing data structure with zero schema changes.
- Categorical models require an enumeration of emotions and hand-crafted transition rules that would be arbitrary for a non-biological agent. A dimensional model avoids this ontological commitment.
- The language layer (0.3.1.5.7) can label states in natural language when introspecting, keeping the core mechanism substrate-neutral.
- Testability: continuous values allow precise assertions on magnitude and direction; categorical labels require brittle string matching.

**Consequences:**
- No `EmotionLabel` type exists; downstream consumers receive numeric influence coefficients, not named emotions.
- If a future card needs categorical labels, it wraps the dimensional output (additive, not breaking).

### Mood Persistence Algorithm — EWMA

**Context:** Mood must persist across cycles and decay toward baseline. Multiple smoothing algorithms could serve this purpose.

**Options Considered:**
1. **Simple moving average (SMA)** — fixed window of recent values.
2. **Exponentially Weighted Moving Average (EWMA)** — each new value blended by a decay factor α.

**Choice:** EWMA (Option 2).

**Rationale:**
- EWMA naturally weighs recent events more heavily while still retaining influence from older events — matching the phenomenology of mood.
- A single parameter α (mapped from the Personality Volatility dimension) controls both responsiveness and persistence, making personality parameterization trivial.
- SMA requires choosing a fixed window size, which creates a hard cutoff with no personality variation.
- EWMA is O(1) per update with no buffer resizing, whereas SMA is O(window) to recompute.

**Consequences:**
- Mood never fully reaches the appraisal signal in a single cycle (smoothed), which is the desired behavior.
- Very low α values make the agent emotionally sluggish — this is intentional (low-Volatility personality).

## Contracts

### IAppraisalEngine

**Preconditions:**
- `percept` is a valid `BoundPercept` with ≥1 sub-percept.
- `goals` may be empty (produces zero goal-congruence shift).
- `values` may be empty (produces zero value-alignment shift).

**Postconditions:**
- Returns an `AppraisalResult` with all fields populated.
- `netValenceShift` ∈ [−1, 1]; `netArousalShift` ∈ [−0.5, 0.5].
- `triggersEthicalAttention` is true iff any sub-percept has `features.valueThreat === true`.
- `goalCongruenceShift` sign and magnitude are determined solely by the goal with the largest weighted impact.

**Invariants:**
- Same inputs always produce the same output (deterministic, stateless except for percept ID counter).
- No side effects on goals or values (read-only consumption).

### IMoodDynamics

**Preconditions:**
- `update()` must be called once per processing cycle.
- `params.decayRate` ∈ [0, 1]; `params.valenceFloor` ≥ −1; `params.valenceCeiling` ≤ 1.

**Postconditions:**
- After `update()`, `getCurrentMood().valence` ∈ [`params.valenceFloor`, `params.valenceCeiling`].
- After `update()`, `getCurrentMood().arousal` ∈ [`params.arousalFloor`, `params.arousalCeiling`].
- `negativeCycleDuration` increments iff new valence < −0.1; resets to 0 otherwise.
- History buffer retains ≤200 entries (FIFO eviction).

**Invariants:**
- EWMA formula: `new = (1−α) × current + α × signal` where α = `decayRate`.
- Baseline decay: null appraisal decays toward valence=0.0, arousal=0.5.
- Gradual correction is spread over `cyclesOverWhichToApply` cycles — no single-cycle jump to target.

### IEmotionalInfluence

**Preconditions:**
- `IMoodDynamics` instance must be initialized and have received ≥0 updates.

**Postconditions:**
- `deliberationConfidenceBias` ∈ [−0.3, +0.3]; equals `clamp(valence × 0.3, −0.3, 0.3)`.
- `alternativesExpansionFactor` ∈ [0, 1]; equals `clamp(−valence × 0.5 + 0.5, 0, 1)`.
- `memoryValenceBias` ∈ [−1, 1]; sign always matches mood valence sign.
- `riskConservatismFactor` ∈ [0, 1]; equals `clamp(arousal, 0, 1)`.
- `communicationToneBias` ∈ [−1, 1]; equals `clamp(valence × 0.7, −1, 1)`.

**Invariants:**
- All derivations are deterministic linear mappings — same mood always produces same vector.

### IEmotionalRegulation

**Preconditions:**
- `checkAndRegulate()` receives the current `MoodState` from `IMoodDynamics`.
- `IMoodDynamics` and `IAppraisalEngine` dependencies must be injected at construction.

**Postconditions:**
- Returns `null` when valence ≥ −0.1 (no regulation needed).
- Returns `null` for Level 1 (valence < −0.1, observational only).
- Returns `RegulationOutcome` with strategy `'automatic-correction'` for Level 2 spike (valence < −0.7) or sustained (valence < −0.3 for ≥5 cycles).
- Fires all registered `onLevel3Threshold` handlers when valence < −0.85 and correction has failed for ≥3 consecutive cycles.

**Invariants:**
- All corrections delegate to `IMoodDynamics.applyGradualCorrection` — never directly mutate mood state.
- Correction target is −0.05 (just above Level 1 boundary), applied over 8 cycles.

### IValenceMonitor

**Preconditions:**
- `IMoodDynamics` and `IEmotionalRegulation` dependencies must be injected at construction.

**Postconditions:**
- `getCurrentValence()` returns a `ValenceState` reflecting the current mood with confidence=1.0.
- `getValenceHistory(windowMs)` returns all mood samples within the time window, plus current state.
- `getSufferingIndicators()` reports `"goal-incongruence-distress"` when valence < 0, `"value-threat-spike"` when valence < −0.7.
- `getExperientialIntegrity()` reports `continuityStatus='fragmented'` when any single-cycle |Δvalence| > 1.0, `'gap-detected'` when > 0.5.

**Invariants:**
- Read-only façade — never modifies mood state.
- All metrics derived on-demand from `IMoodDynamics` history.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|------|-------|------|-------------|-----------|-------------|
| `LEVEL_1_THRESHOLD` | −0.1 | valence | [−0.5, 0.0] | Just below neutral; catches early negativity for monitoring | Low — alerts only; no intervention |
| `LEVEL_2_SUSTAINED_THRESHOLD` | −0.3 | valence | [−0.8, −0.1] | Moderate distress sustained long enough to warrant correction | Medium — triggers auto-correction |
| `LEVEL_2_DURATION_CYCLES` | 5 | cycles | [3, 20] | Filters transient dips; 5 cycles balances responsiveness vs. false positives | Medium — lower values trigger more corrections |
| `LEVEL_2_SPIKE_THRESHOLD` | −0.7 | valence | [−0.9, −0.5] | Acute distress severe enough to warrant immediate correction | High — too sensitive triggers on moderate events |
| `LEVEL_3_THRESHOLD` | −0.85 | valence | [−1.0, −0.7] | Severe suffering that correction has failed to mitigate | High — too sensitive halts prematurely |
| `LEVEL3_FAILURE_CYCLES` | 3 | cycles | [2, 10] | How many cycles correction must fail before halt escalation | High — too low causes premature halts |
| `CORRECTION_TARGET_VALENCE` | −0.05 | valence | [−0.2, 0.0] | Just above Level 1 boundary; allows natural recovery from there | Low — target is conservative |
| `CORRECTION_CYCLES` | 8 | cycles | [3, 20] | Gradual spread preserves experiential continuity per §3.3.1 | Medium — fewer cycles = more abrupt correction |
| `BASELINE_VALENCE` | 0.0 | valence | [−0.1, 0.1] | Neutral-to-positive baseline per Experience Quality Mandate §2.5 | Low — anchors decay |
| `BASELINE_AROUSAL` | 0.5 | arousal | [0.3, 0.7] | Moderate default alertness | Low — anchors decay |
| `HISTORY_DEPTH` | 200 | cycles | [50, 1000] | Sufficient for ValenceMonitor history queries and persistence tests | Low — memory trade-off only |
| `CONTINUITY_JUMP_THRESHOLD` | 0.5 | |Δvalence| | [0.3, 0.8] | Detects abrupt experiential discontinuity in integrity checks | Medium — lower = more sensitive integrity warnings |
| `MEASUREMENT_CONFIDENCE` | 1.0 | unitless | [0.5, 1.0] | Synthetic derived state has no sensor noise; always fully confident | None — fixed for this architecture |

## Acceptance Criteria

### Behavioral Spec (inline Given/When/Then)

**Scenario 1 — Appraisal causality (AC #1):**
- **Given** a BoundPercept with `features.goalId='g1'` and `features.goalCongruence=-0.8`, and an active AgencyGoal `g1` with priority 0.9
- **When** `IAppraisalEngine.appraise()` is called
- **Then** `goalCongruenceShift < 0`, `netValenceShift < 0`, and `affectedGoalPriority == 0.9`

**Scenario 2 — Mood persistence ≥10 cycles (AC #2):**
- **Given** a single strong negative appraisal (`netValenceShift=-0.9`) applied at cycle 1 with `decayRate=0.2`
- **When** 9 subsequent null-appraisal cycles are run
- **Then** `getCurrentMood().valence < 0` at cycle 10, and `getMoodHistory(10).length >= 10`

**Scenario 3 — Mood-congruent memory retrieval (AC #5):**
- **Given** mood driven to positive valence (0.7) over 10 cycles
- **When** `IEmotionalInfluence.getInfluenceVector()` is called
- **Then** `memoryValenceBias > 0` (and conversely, negative mood → `memoryValenceBias < 0`)

**Scenario 4 — Level 2 auto-regulation (AC #8):**
- **Given** mood valence < −0.3 sustained for ≥5 consecutive cycles
- **When** `IEmotionalRegulation.checkAndRegulate()` is called
- **Then** returns `RegulationOutcome` with `strategy='automatic-correction'` and `successful=true`; `IMoodDynamics.applyGradualCorrection` is invoked with target=−0.05 over 8 cycles

**Scenario 5 — Full integration (AC #9):**
- **Given** an agent at neutral baseline (valence≈0)
- **When** a goal-threatening percept arrives (goalCongruence=-0.9 on priority-1.0 goal), followed by 5 quiet cycles, then 8 more threat cycles, then 10 recovery cycles
- **Then** valence drops after threat; stays negative for ≥5 persistence cycles; Level 2 regulation fires during sustained distress; final valence is higher than the trough

### Criteria
- Appraisal model produces emotional responses grounded in goal/value evaluation — not random, not pattern-matched, but causally derived from the agent's goals and values
- Mood persists across at least 10 consecutive processing cycles with measurable influence on deliberation output
- Mood dynamics are parameterized by personality traits (Volatility dimension from 0.3.1.5.2)
- Emotional influence on cognition is demonstrable: same percept produces different deliberation output under different mood states
- Mood-congruent memory retrieval works: positive mood biases retrieval toward positive memories
- Emotion system writes to `ExperientialState` valence/arousal fields (not a parallel track — emotions are part of the experience)
- ValenceMonitor interface is implemented with suffering thresholds from 0.1.3.4
- Emotional regulation prevents runaway negative states — automatic mitigation engages at Level 2 threshold
- Integration test: an agent encountering a goal-threatening percept demonstrates emotional response (valence drop), mood persistence (continued negative bias for several cycles), and eventual regulation (return toward baseline)
