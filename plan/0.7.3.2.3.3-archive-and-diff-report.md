# 0.7.3.2.3.3 Archive Package & Produce Diff Report [PLAN]

Rebuild the distribution archive (ZIP/TAR), verify its integrity, and generate a diff report showing changes between the previous and updated manifests.

**Acceptance Criteria**
1. The archive (ZIP or TAR) is created and includes the updated code, manifest, and documentation.
2. An SHA‑256 checksum of the archive is calculated and matches the stored checksum file.
3. A diff report (`manifest-diff.txt`) highlighting added, removed, and modified entries between old and new manifests is generated.
4. Both the archive and diff report are uploaded to the designated repository location with read‑only access for reviewers.
5. The repository records (e.g., release notes) reference the new archive checksum and diff report.
