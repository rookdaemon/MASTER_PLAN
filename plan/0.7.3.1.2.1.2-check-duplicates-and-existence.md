# 0.7.3.1.2.1.2 Check Duplicates and Verify Existence [PLAN]

From the ordered `rawEntries` list, resolve each to an absolute path, detect duplicates, and confirm that every file exists on disk.

**Acceptance Criteria**
1. Each entry is converted to an absolute path using the project root as base.
2. If any duplicate absolute path is found, the task logs all duplicates and fails the build.
3. For every absolute path, `fs.access` (or equivalent) is used to confirm existence; missing files are logged and cause a fatal error.
4. A clean list `uniqueExistingPaths` (ordered, duplicate‑free) is produced for downstream steps.
5. The step completes in ≤ 10 seconds for 10 k entries.
