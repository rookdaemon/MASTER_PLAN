# 0.7.3.2.4.2.3 Produce Mapping & Validate [PLAN]

Generate the final CSV mapping of comment IDs to remediation actions and priorities, version‑control it, and obtain peer‑review validation.

**Acceptance Criteria**
1. CSV file `feedback/mapping.csv` contains columns: `CommentID,FilePath,Action,Priority`.
2. The CSV is added to the repository under `feedback/` and version‑controlled.
3. A peer reviewer signs off on at least 90 % of rows (recorded in `validation.log`).
4. Any rejected rows are flagged for re‑work and listed in `rework-list.txt`.
5. A CI step runs the validation script; the pipeline passes only if the 90 % threshold is met.
