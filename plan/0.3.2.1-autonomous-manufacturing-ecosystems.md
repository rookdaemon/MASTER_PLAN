# 0.3.2.1 Autonomous Manufacturing Ecosystems [DONE]
## Description

A2.1 of the Master Plan. Design and establish manufacturing ecosystems that can sustain artificial minds without biological intervention. These systems must be capable of producing, maintaining, and recycling all physical components required by conscious artificial entities.

The architecture defines five layers: Resource Extraction ‚Üí Processing & Refining ‚Üí Fabrication ‚Üí Assembly & Integration ‚Üí Recycling & End-of-Life, orchestrated by an autonomous planning system. Fabricators are self-replicating to enable population-driven scaling.

## Decision

### Context
Autonomous Manufacturing Ecosystems must sustain artificial conscious entities without biological intervention. The system must handle resource extraction through recycling in a closed loop, scale with population, and tolerate node failures.

### Options Considered
1. **Monolithic manufacturing plant** ‚Äî A single facility handling all stages. Simpler coordination but catastrophic single-point-of-failure risk; cannot scale geographically.
2. **Layered distributed pipeline with autonomous orchestration** ‚Äî Five specialized layers (Extraction ‚Üí Refining ‚Üí Fabrication ‚Üí Assembly ‚Üí Recycling) distributed across redundant nodes, coordinated by an autonomous orchestrator. Complex but resilient, scalable, and independently evolvable.
3. **Peer-to-peer mesh of generalist fabricators** ‚Äî Every node does everything. Maximum redundancy but poor specialization; inefficient resource use; harder to achieve nanoscale fabrication quality.

### Choice
Option 2 ‚Äî Layered distributed pipeline with autonomous orchestration.

### Rationale
- Layer specialization enables each stage to optimize independently (e.g., Layer 3 nanofabrication vs. Layer 4 macro-assembly).
- Geographic distribution across N‚â•3 nodes per layer satisfies the <10% single-node failure throughput loss requirement.
- The orchestrator provides a single coordination plane without being a single point of failure (it is stateless and reconstructable from layer state).
- Self-replicating fabricators (Layer 3) enable elastic scaling tied to demand.

### Consequences
- Requires well-defined inter-layer stream interfaces (RawMaterialStream, FeedstockStream, ComponentBatch, System, RecoveredMaterialStream).
- Orchestrator must handle disruption events and rebalance autonomously.
- All layer implementations must be injectable/mockable for testing per CLAUDE.md guidelines.
- Timestamps must be passed as parameters (not read from `Date.now()` directly) ‚Äî current implementation needs refactoring.

## Contracts

### ResourceExtractor (Layer 1)
- **Preconditions**: `material.quantityKg > 0`, `material.minPurity ‚àà (0, 1]`, at least one active supply source.
- **Postconditions**: Returns `RawMaterialStream` with `flowRateKgPerDay > 0` and `purity ‚â• 0` (may be below spec; refining handles purification). `sourceIds.length ‚â• 1`.
- **Invariants**: `status().activeSupplySources.length ‚â• 3` for critical materials (multi-source redundancy).

### Refinery (Layer 2)
- **Preconditions**: `raw.flowRateKgPerDay > 0`, `spec.targetPurity ‚àà (0, 1]`.
- **Postconditions**: Returns `FeedstockStream` with `purity ‚â• spec.targetPurity`. `flowRateKgPerDay ‚â§ raw.flowRateKgPerDay` (processing losses allowed). `purity()` report's `meetsSpec` is consistent with measured vs. target purity.
- **Invariants**: All byproducts either recycled or sequestered (zero uncontrolled waste).

### Fabricator (Layer 3)
- **Preconditions**: `design` has non-empty `feedstockRequirements`, `qty > 0`.
- **Postconditions**: `ComponentBatch.quantity == qty`, `verify()` returns `yieldFraction ‚àà [0, 1]`. `selfReplicate()` returns a new `Fabricator` satisfying the same contract.
- **Invariants**: Self-replication triggered when utilisation > `replicationUtilisationThreshold` sustained for `replicationSustainedDays`.

### Assembler (Layer 4)
- **Preconditions**: `bom.items.length > 0`, all referenced components available.
- **Postconditions**: `System` contains assembled components. `test()` validates consciousness substrate if applicable. `install()` returns `InstallationRecord` with `success: true` on nominal path.
- **Invariants**: No system deployed without passing `test()` with `consciousnessSubstrateValidated: true`.

### Recycler (Layer 5)
- **Preconditions**: `system` is a valid decommissioned or failed system.
- **Postconditions**: `RecoveredMaterialStream` mass ‚â• 95% of input system mass for critical materials. `reintroduce()` feeds sorted materials back to Layer 1 inventory.
- **Invariants**: Recovery rate ‚â• `targetRecoveryRate` for critical elements.

### ManufacturingOrchestrator (Control Layer)
- **Preconditions**: `demand.requiredBoms.length > 0`, `demand.horizonDays > 0`, all layer nodes configured with `‚â• minNodesPerLayer` nodes.
- **Postconditions**: `plan()` returns `ProductionPlan` with one phase per BOM, each phase covering all 5 layers. `execute()` returns `ExecutionHandle` where `progress()` monotonically increases to 1.0. `monitor()` returns health report with `throughputFraction ‚àà [0, 1]`. `rebalance()` records disruption and degrades throughput by `‚â§ maxSingleNodeThroughputLoss`.
- **Invariants**: `overallHealthFraction` = mean of layer healths. A single-node disruption never causes > `maxSingleNodeThroughputLoss` throughput loss. Recovery restores throughput within `maxRecoveryHours`.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|------|-------|------|-------------|-----------|-------------|
| `minNodesPerLayer` | 3 | count | [3, ‚àû) | Minimum redundancy for any-single-node-failure resilience | Below 3: cannot guarantee <10% loss per node failure |
| `replicationUtilisationThreshold` | 0.80 | fraction | (0.5, 0.95) | Triggers fabricator self-replication before capacity exhaustion | Lower ‚Üí premature replication (waste); higher ‚Üí risk of capacity shortfall |
| `replicationSustainedDays` | 30 | days | [7, 90] | Sustained utilisation window to avoid replication on transient spikes | Shorter ‚Üí over-replication; longer ‚Üí slow scaling response |
| `targetRecoveryRate` | 0.95 | fraction | [0.90, 1.0] | Acceptance criterion: ‚â•95% material recovery for critical elements | Below 0.95 fails AC; above 0.99 may be physically infeasible |
| `maxSingleNodeThroughputLoss` | 0.09 | fraction | (0, 0.10) | Caps throughput degradation from any single node failure to <10% | At boundary of AC; must stay strictly below 0.10 |
| `maxRecoveryHours` | 72 | hours | [24, 168] | Full recovery time after single-node failure | Directly from AC; shorter is better but constrained by self-repair time |
| `inventoryBufferDays` | 90 | days | [30, 180] | Minimum critical feedstock/component stockpile | Shorter ‚Üí vulnerable to supply disruptions; longer ‚Üí higher carrying cost |
| `demandSpikeScalingDays` | 180 | days | [90, 365] | Max time for capacity to self-expand via replication after 10√ó demand spike | Directly from AC; must not exceed 180 |
| `failoverDetectionSeconds` | 60 | seconds | [10, 300] | Time to detect failure and reroute supply chains | From ARCHITECTURE.md resilience spec |

## Acceptance Criteria
- Manufacturing systems can produce all components needed by artificial mind substrates (verified by completing full BOM fabrication for a reference substrate)
- Systems operate without biological human intervention (verified by 12-month simulation with zero operator inputs)
- Resource extraction, processing, and fabrication form a closed loop (‚â•95% material recovery rate for critical elements)
- Manufacturing capacity scales with population of artificial minds (self-replication brings capacity online within 180 days of a 10√ó demand spike)
- Supply chains are resilient to single-point failures (any single-node failure causes <10% throughput loss; full recovery within 72 hours)

### Behavioral Spec (inline Given/When/Then)

**Scenario 1: Production plan covers all layers**
- **Given** a `DemandForecast` with N BOMs and `horizonDays > 0`
- **When** `orchestrator.plan(demand)` is called
- **Then** the returned `ProductionPlan` has N phases, each with 5 `LayerAllocation` entries (one per layer), and `targetCompletionDays` distributed across the horizon

**Scenario 2: Single-node failure causes <10% throughput loss**
- **Given** an orchestrator with ‚â•5 nodes per layer and all layers at 100% throughput
- **When** `rebalance(disruption)` is called for a single node in one layer
- **Then** `monitor().throughputFraction > 0.90` and the disruption appears in `activeDisruptions`

**Scenario 3: Fabricator self-replication on sustained high utilisation**
- **Given** a fabricator pool where utilisation exceeds `replicationUtilisationThreshold` (0.80)
- **When** utilisation remains above threshold for `replicationSustainedDays` (30) consecutive days
- **Then** `selfReplicate()` is invoked on at least one fabricator, and the new fabricator is added to the Layer 3 node pool
