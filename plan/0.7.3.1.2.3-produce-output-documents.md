# 0.7.3.1.2.3 Produce Output Documents [PLAN]

Run the generation script to create the final TRACEABILITY markdown and HTML files, ensure they are navigable, and integrate them into the project repository.

**Acceptance Criteria**
1. `TRACEABILITY.md` contains a table of contents linking to every requirement, design decision, code file, and test result section.  
2. Backlinks are verified: each design decision section lists originating requirements, and each test section lists related code.  
3. `pandoc` (or equivalent) converts `TRACEABILITY.md` to `TRACEABILITY.html` with all links preserved and functional offline.  
4. Both files are placed under `traceability/plan-guardian/` and referenced in the repository README.  
5. A CI step (`scripts/ci/check-traceability.sh`) verifies that the generated files are up‑to‑date; the CI fails if the script needs to be re‑run.
