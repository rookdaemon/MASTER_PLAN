# 0.7.3.1.1.1.3.2 Commit JSON and Update Documentation [PLAN]

Automate Git staging/committing of the generated provenance file and create/update the documentation page.

**Acceptance Criteria**
1. After the driver script runs, `git add artifacts/provenance/provenance.json` is executed via `subprocess`.
2. A commit is created with message `"chore: add provenance inventory"`; the commit appears in `git log`.
3. `docs/provenance.md` is generated (or updated) containing:
   - a brief description of the provenance file purpose,
   - a code block showing how to run `scripts/build_provenance.py`,
   - a relative link to the JSON file.
4. The documentation file is added and committed in the same commit as the JSON file.
5. Integration test `tests/integration/test_provenance_commit_and_docs.py` asserts that the commit touches both files and that the markdown contains the expected sections.
