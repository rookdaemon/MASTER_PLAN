# 0.7.3.2.4.1.1 Export Comments Data [PLAN]

Extract all reviewer comments from the review platform via its API or export function and write them to a structured machine‑readable file (JSON preferred, CSV acceptable).

**Acceptance Criteria**
1. A script/tool authenticates to the review platform and retrieves the full comment set in one operation.  
2. The output file `feedback/comments.json` is created and contains a top‑level array of comment objects.  
3. Each comment object includes the fields: `reviewer_id`, `timestamp`, `text`, and `artifact_ref`.  
4. The script logs the total number of comments retrieved and exits with status 0 on success.  
5. The file is placed under `feedback/` and added to git staging (but not yet committed).
