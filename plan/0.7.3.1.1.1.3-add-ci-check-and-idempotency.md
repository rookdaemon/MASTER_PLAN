# 0.7.3.1.1.1.3 Add CI Check and Idempotency Assurance [PLAN]

Integrate provenance generation into the CI pipeline and verify no duplicate entries appear.

**Acceptance Criteria**
1. CI workflow (e.g., `.github/workflows/ci.yml`) runs `npm run provenance` and fails if the generated `provenance.json` does not validate against the schema.
2. A test (`tests/provenance.test.js`) asserts that running the generator twice yields identical `provenance.json` (deep equality).
3. The pipeline caches the generated file to avoid unnecessary diffs when no deliverables changed.
4. CI logs include a clear “Provenance validation passed” message on success.
5. Documentation in `CONTRIBUTING.md` explains how contributors can locally verify provenance generation before PR submission.
