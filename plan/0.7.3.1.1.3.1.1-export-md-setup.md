# 0.7.3.1.1.3.1.1 Export markdown – script scaffold and JSON loading [PLAN]

Create a new Python (or Node) script `export_traceability_md.py` that:

1. Loads `provenance.json` from the project root.
2. Validates that the JSON conforms to the expected schema (deliverable objects with `id`, `type`, `artifacts` array).
3. Exposes a `main()` function that can be invoked from the command line.

**Acceptance Criteria**
- The script exists at `tools/export_traceability_md.py`.
- Running `python tools/export_traceability_md.py --dry-run` prints a concise summary of how many deliverables were detected.
- Errors are raised with clear messages if `provenance.json` is missing or malformed.
- The script contains type‑hints / JSDoc and is lint‑compatible.
- No output files are written in this step (pure data loading).
