# 0.7.3.1.1.1.3.3 Update Docs and Enable Caching [PLAN]

Add guidance for local provenance verification and configure CI caching of the generated file.

**Acceptance Criteria**
1. `CONTRIBUTING.md` gains a section “Verifying provenance locally” with commands to run the generator and validation script.
2. CI workflow includes a caching step (`actions/cache`) that stores `provenance.json` between runs when no source files changed.
3. Cache key is based on a hash of all deliverable source files; cache is restored before the provenance step.
4. Documentation notes the cache behavior and how to force regeneration (e.g., `git commit --amend`).
5. All markdown updates pass linting and spelling checks.
