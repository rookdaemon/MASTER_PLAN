# 0.7.3.2.4.2.2 Compose Release Notes [PLAN]

Draft the release‑notes section summarizing all changes since the previous release version, using the changelog data generated in earlier review steps.

**Acceptance Criteria**
1. Release notes contain a concise summary of new features, bug fixes, and breaking changes.  
2. Every relevant issue/PR ID is referenced with a hyperlink to the repository.  
3. The notes follow the organization’s standard release‑notes template.  
4. Output file is `docs/release_notes.md`.  
5. The file passes spell‑check and markdown linting.
