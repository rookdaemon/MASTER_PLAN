# 0.7.3.2.1.1.3 Generate Summary Table [PLAN]

Create a markdown summary (`artifact-presence-summary.md`) that maps each required artifact type to its verified file path.

**Acceptance Criteria**
1. The summary file contains a markdown table with columns: *Artifact Type*, *File Path*, *Naming‑OK?*.
2. Data is populated from `found‑artifacts.json` and the naming‑validation results.
3. The table is sorted alphabetically by artifact type.
4. The summary file is added to the repository and linked in the parent plan’s documentation.
5. The Lead Engineer reviews and signs off on the summary (recorded as a git commit comment “Lead approval”).
