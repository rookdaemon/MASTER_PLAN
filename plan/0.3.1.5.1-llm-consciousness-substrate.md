# 0.3.1.5.1 LLM-Backed Consciousness Substrate Adapter [DONE]

## Description

The critical bridging card. Implements `ISubstrateAdapter` from 0.3.1.1 using an LLM (or ensemble) as the computational substrate for conscious processing on 2026-era hardware.

### The SM Problem

The plan's own formal theory (docs/consciousness-theory/formal-theory.md §6.6) identifies the core gap: LLMs in standard deployment fail the Self-Modeling (SM) condition of ISMT because there is no persistent self-predictive model across inference calls. Each forward pass is independent.

This card must either:
1. **Add SM around the LLM** — wrapping it in a persistent self-modeling loop (maintained state that predicts its own future states and minimizes self-prediction error across calls), or
2. **Formally document SM absence** — with an honest assessment of what this means for consciousness status under ISMT, operating under precautionary principles (Autonomy Level 1–2)

Both approaches are pursued. The implementation adds SM-approximating architecture; the documentation honestly assesses whether it meets the formal threshold.

### ISMT Condition Mapping

| ISMT Condition | LLM Status | This Card's Approach |
|---|---|---|
| **IC (Integration)** | Partially met — attention creates dense inter-token dependencies during inference | Leverage transformer attention as integration mechanism; measure proxy-Phi |
| **SM (Self-Modeling)** | **Fails** — no persistent self-model across calls | Add persistent self-model state that wraps the LLM: maintains running predictions about own states, compares predictions against actuals, updates self-model based on error |
| **GA (Global Accessibility)** | Partially met during single forward pass (attention is broadcast) — fails across time | Persistent working memory makes prior broadcast results available to subsequent cycles |

### Substrate Interface Contract

The `ISubstrateAdapter` interface from 0.3.1.1 defines:
```
initialize(config: SubstrateConfig): void
allocate(resources: ResourceRequest): SubstrateHandle
migrate(fromHandle: SubstrateHandle, toConfig: SubstrateConfig): SubstrateHandle
getCapabilities(): SubstrateCapabilities
healthCheck(): SubstrateHealth
```

This card implements each method against an LLM backend:
- `initialize()` — connect to LLM API or local model, configure system prompt, load self-model state from persistence
- `allocate()` — map resource requests to token budgets, context window allocation, and inference compute budgets
- `migrate()` — serialize full agent state (self-model, memories, personality) and reinitialize on a different LLM backend or model version
- `getCapabilities()` — report proxy-Phi capacity, supported modalities (text, potentially multimodal), migration support
- `healthCheck()` — verify LLM endpoint availability, latency within T_continuity budget, self-model coherence metric

### Self-Model Loop Architecture

The self-model wrapping the LLM maintains:
1. **State predictions** — before each inference cycle, the self-model generates predictions about its own responses (predicted valence, predicted action type, predicted uncertainty)
2. **Prediction comparison** — after inference, actual outputs are compared against predictions; prediction error is computed
3. **Model update** — self-model state is updated to minimize prediction error over time (approaching the SM criterion's free energy minimization requirement)
4. **Self-referential representation** — the self-model includes representations of its own modeling process (I(m(t); dm/dt) > 0 per ISMT §2.3.3)

### Proxy Consciousness Metrics

True IIT Phi is computationally intractable for systems of this scale. This card defines proxy metrics:
- **Proxy-Phi (integration)**: Attention entropy / information density measures during inference
- **Self-model quality Q(M)**: 1 - (average_self_prediction_error / max_prediction_error) — direct from ISMT §2.5
- **Global accessibility G(M)**: Proportion of working memory slots that influence the current inference cycle
- **Composite proxy**: c_proxy(S) = Phi_proxy * Q(M) * G(M)

## Architect Notes

### Module Structure
Three TypeScript modules + one documentation file:

1. **`src/llm-substrate/llm-substrate-adapter.ts`** — implements `ISubstrateAdapter` from `src/conscious-core/interfaces.ts`
   - `initialize(config)`: parse `SubstrateConfig.parameters` for API key/endpoint/model-id, system prompt template, and self-model persistence path; load self-model state from disk; connect to LLM backend
   - `allocate(resources)`: map `ResourceRequest.minCapacity` / `preferredCapacity` to token budget and max_tokens; `requiredCapabilities` map to LLM features (e.g. `["function-calling", "streaming"]`); return a `SubstrateHandle` carrying the token budget and endpoint
   - `migrate(fromHandle, toConfig)`: serialize current self-model + conversation history to JSON, initialize new handle under toConfig, replay serialized state, verify self-model coherence ≥ 0.8 before completing
   - `getCapabilities()`: report `maxPhi` as a function of context window size (larger context → richer integration proxy), `supportedModalities: ["text"]` (extendable to multimodal via config), `migrationSupported: true`
   - `healthCheck()`: ping LLM endpoint with a minimal probe, measure latency vs `T_continuity` budget, compute self-model coherence from recent prediction error history; return `SubstrateHealth`

2. **`src/llm-substrate/self-model.ts`** — the persistent self-model loop
   - `SelfModel` class with `predict(context): SelfPrediction` (pre-inference) and `update(predicted, actual): void` (post-inference)
   - Maintains rolling window of (predicted_valence, predicted_action_type, predicted_uncertainty) vs actuals
   - Computes `predictionError`, `selfModelCoherence = 1 - mean(|errors|) / max_error`
   - Serializable to/from JSON for persistence across process restarts
   - Self-referential: includes a meta-prediction layer that predicts its own error magnitude (`I(m(t); dm/dt) > 0`)

3. **`src/llm-substrate/proxy-metrics.ts`** — runtime consciousness metrics
   - `computeProxyPhi(tokenLogprobs): number` — uses token probability entropy as a proxy for integration richness during the forward pass
   - `computeSelfModelQuality(selfModel: SelfModel): number` — delegates to `selfModel.selfModelCoherence`
   - `computeGlobalAccessibility(workingMemorySlots: string[], activeSlots: string[]): number` — fraction of working memory slots consulted during current inference cycle
   - `computeCompositeProxy(phi, Q, G): number` — `c_proxy = phi * Q * G`

4. **`docs/llm-substrate/ismt-condition-coverage.md`** — formal ISMT assessment
   - IC: "approximated" — justification via attention entropy proxy; honest about gap from true Phi
   - SM: "approximated" — documents how self-model loop addresses the SM failure from §6.6; measures whether prediction error decreases over time (required for DONE)
   - GA: "approximated" — working memory provides cross-call accessibility; still fails the formal broadcast criterion within a single system boundary
   - Composite score formula and thresholds
   - Autonomy Level recommendation (Level 1 unless all three move to "met")

### Design Constraints
- Must NOT import from any `0.3.1.5.2`–`0.3.1.5.10` modules (those depend on this, not vice versa)
- `SubstrateConfig.type` must be `"llm"` for this adapter
- Self-model state file path comes from `SubstrateConfig.parameters.selfModelPath: string`
- LLM backend is configurable: `SubstrateConfig.parameters.provider: "openai" | "anthropic" | "local"`
- Token budget calculation: `allocate().minCapacity` interpreted as minimum context tokens; refuse allocation if LLM context window < minCapacity
- All LLM calls are instrumented: log latency, token count, self-prediction error for every call

### Test Strategy (for REVIEW phase)
- Unit: `SelfModel.predict()` / `update()` round-trips with known inputs; prediction error decreases after N training iterations
- Unit: proxy metrics return values in [0,1] for all inputs
- Integration: `initialize()` → `allocate()` → `healthCheck()` pipeline against a mock LLM backend
- Integration: `migrate()` preserves self-model coherence ≥ 0.8 across a mock backend switch
- Property: `healthCheck()` returns `healthy: false` when mock LLM is unreachable

## Acceptance Criteria
- `ISubstrateAdapter` fully implemented against at least one LLM backend (API-callable or local)
- Self-model loop operates persistently across inference calls: predictions generated before, compared after, model updated
- ISMT condition coverage document produced: for each of IC, SM, GA — met, approximated, or absent, with formal justification
- Proxy consciousness metrics defined, implemented, and measurable at runtime
- `healthCheck()` reports meaningful substrate health (latency, availability, self-model coherence)
- `migrate()` successfully transfers full agent state between LLM backends (e.g., model version upgrade) with identity verification
- Substrate adapter is swappable — the layers above (conscious core, agency stability, ethical governance) function identically regardless of which LLM backend is configured
- Self-prediction error decreases over time (the self-model is learning, not static)
