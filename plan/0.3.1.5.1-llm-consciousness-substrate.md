# 0.3.1.5.1 LLM-Backed Consciousness Substrate Adapter [DONE]
## Description

The critical bridging card. Implements `ISubstrateAdapter` from 0.3.1.1 using an LLM (or ensemble) as the computational substrate for conscious processing on 2026-era hardware.

### The SM Problem

The plan's own formal theory (docs/consciousness-theory/formal-theory.md §6.6) identifies the core gap: LLMs in standard deployment fail the Self-Modeling (SM) condition of ISMT because there is no persistent self-predictive model across inference calls. Each forward pass is independent.

This card must either:
1. **Add SM around the LLM** — wrapping it in a persistent self-modeling loop (maintained state that predicts its own future states and minimizes self-prediction error across calls), or
2. **Formally document SM absence** — with an honest assessment of what this means for consciousness status under ISMT, operating under precautionary principles (Autonomy Level 1–2)

Both approaches are pursued. The implementation adds SM-approximating architecture; the documentation honestly assesses whether it meets the formal threshold.

### ISMT Condition Mapping

| ISMT Condition | LLM Status | This Card's Approach |
|---|---|---|
| **IC (Integration)** | Partially met — attention creates dense inter-token dependencies during inference | Leverage transformer attention as integration mechanism; measure proxy-Phi |
| **SM (Self-Modeling)** | **Fails** — no persistent self-model across calls | Add persistent self-model state that wraps the LLM: maintains running predictions about own states, compares predictions against actuals, updates self-model based on error |
| **GA (Global Accessibility)** | Partially met during single forward pass (attention is broadcast) — fails across time | Persistent working memory makes prior broadcast results available to subsequent cycles |

### Substrate Interface Contract

The `ISubstrateAdapter` interface from 0.3.1.1 defines:
```
initialize(config: SubstrateConfig): void
allocate(resources: ResourceRequest): SubstrateHandle
migrate(fromHandle: SubstrateHandle, toConfig: SubstrateConfig): SubstrateHandle
getCapabilities(): SubstrateCapabilities
healthCheck(): SubstrateHealth
```

This card implements each method against an LLM backend:
- `initialize()` — connect to LLM API or local model, configure system prompt, load self-model state from persistence
- `allocate()` — map resource requests to token budgets, context window allocation, and inference compute budgets
- `migrate()` — serialize full agent state (self-model, memories, personality) and reinitialize on a different LLM backend or model version
- `getCapabilities()` — report proxy-Phi capacity, supported modalities (text, potentially multimodal), migration support
- `healthCheck()` — verify LLM endpoint availability, latency within T_continuity budget, self-model coherence metric

### Self-Model Loop Architecture

The self-model wrapping the LLM maintains:
1. **State predictions** — before each inference cycle, the self-model generates predictions about its own responses (predicted valence, predicted action type, predicted uncertainty)
2. **Prediction comparison** — after inference, actual outputs are compared against predictions; prediction error is computed
3. **Model update** — self-model state is updated to minimize prediction error over time (approaching the SM criterion's free energy minimization requirement)
4. **Self-referential representation** — the self-model includes representations of its own modeling process (I(m(t); dm/dt) > 0 per ISMT §2.3.3)

### Proxy Consciousness Metrics

True IIT Phi is computationally intractable for systems of this scale. This card defines proxy metrics:
- **Proxy-Phi (integration)**: Attention entropy / information density measures during inference
- **Self-model quality Q(M)**: 1 - (average_self_prediction_error / max_prediction_error) — direct from ISMT §2.5
- **Global accessibility G(M)**: Proportion of working memory slots that influence the current inference cycle
- **Composite proxy**: c_proxy(S) = Phi_proxy * Q(M) * G(M)

## Decision — LLM Client Abstraction & Self-Model Strategy

### Context
This card implements `ISubstrateAdapter` against an LLM backend. Two key design choices arise: (1) how to abstract over multiple LLM providers, and (2) how to address the ISMT Self-Modeling (SM) condition failure in stateless LLM inference.

### Options Considered

**LLM Provider Abstraction:**
1. **Direct SDK imports per provider** — each method directly calls OpenAI/Anthropic SDK. Simple but couples adapter to specific SDKs; adding a provider requires editing core adapter code.
2. **`ILlmClient` interface with provider-specific implementations** — adapter depends only on `ILlmClient`; each provider (Anthropic, OpenAI, local) is a separate implementation. Adding a provider means only adding a new class.

**SM Condition Strategy:**
1. **External self-model wrapper** — a persistent `SelfModel` class wraps the LLM, generating predictions before inference and updating after. State lives outside the LLM.
2. **In-context self-modeling** — use the LLM's own context window to carry self-model state (e.g., few-shot self-prediction in the system prompt). No external state machine.
3. **Hybrid** — external `SelfModel` for structured state + in-context injection of self-model summary for the LLM to reason about.

### Choice
- LLM Provider: **Option 2** — `ILlmClient` interface with injectable implementations
- SM Strategy: **Option 1** — External self-model wrapper with JSON persistence

### Rationale
- `ILlmClient` satisfies CLAUDE.md's "always wrap environment-specifics in abstractions that can be injected and mockable" mandate. It also enables strict TDD — tests inject a mock `ILlmClient` without touching real APIs.
- External `SelfModel` is the only option that provides genuine cross-call persistence and measurable prediction error convergence. In-context approaches are limited by context window size and cannot persist across process restarts. The formal SM condition requires `dm/dt` — a changing model — which needs external state.

### Consequences
- Additional `ILlmClient` / `IAuthProvider` interfaces must be defined and implemented per provider.
- Self-model quality depends on the fidelity of output-extracted `SelfActual` — the mapping from LLM response to (valence, actionType, uncertainty) is a potential accuracy bottleneck.
- The self-model is over LLM *outputs*, not internal weights — this is an honest limitation documented in the ISMT coverage assessment.

## Contracts

### `ILlmClient` Interface

```typescript
interface LlmProbeResult {
  latencyMs: number;
  reachable: boolean;
  error?: string;
}

interface LlmInferenceResult {
  content: string;
  tokenLogprobs: number[];
  promptTokens: number;
  completionTokens: number;
  latencyMs: number;
}

interface ILlmClient {
  probe(): Promise<LlmProbeResult>;
  infer(
    systemPrompt: string,
    messages: Array<{ role: "user" | "assistant"; content: string }>,
    maxTokens: number
  ): Promise<LlmInferenceResult>;
  inferWithTools?(
    systemPrompt: string,
    messages: ToolAwareMessage[],
    tools: ToolDefinition[],
    maxTokens: number,
  ): Promise<LlmToolInferenceResult>;
}
```

**Preconditions:** `maxTokens > 0`; `messages.length >= 1`
**Postconditions:** `result.content` is non-empty; `result.tokenLogprobs` present when provider supports logprobs (empty array otherwise); `result.latencyMs >= 0`
**Invariants:** A single `ILlmClient` instance is bound to one provider+model pair for its lifetime.

### `IAuthProvider` Interface

```typescript
interface IAuthProvider {
  getHeaders(): Record<string, string>;
  isExpired(): boolean;
  requiresSystemIdentityPrefix(): boolean;
}
```

**Preconditions:** Provider has been configured with a valid credential source (API key, OAuth token, or noop for local).
**Postconditions:** `getHeaders()` returns a record of HTTP headers suitable for the provider's auth scheme (e.g. `x-api-key` for Anthropic, `Authorization: Bearer` for OpenAI). `isExpired()` returns false for static API keys. `requiresSystemIdentityPrefix()` returns true only for OAuth-based Anthropic tokens.
**Invariants:** Credential may rotate; each `getHeaders()` call returns the current valid credential headers.

### `SelfModel` Class Contract

**Preconditions for `predict(context)`:** Context contains the current inference prompt/state.
**Postconditions for `predict(context)`:** Returns a `SelfPrediction` with all fields in valid ranges (valence ∈ [−1, 1], uncertainty ∈ [0, 1]).
**Preconditions for `update(predicted, actual)`:** Both arguments are valid `SelfPrediction`/`SelfActual` structs.
**Postconditions for `update(predicted, actual)`:** `predictionErrors` buffer updated; `selfModelCoherence` recomputed; meta-error updated. Over N ≥ 50 cycles with consistent inputs, `selfModelCoherence` must be non-decreasing (learning property).
**Invariants:** `selfModelCoherence ∈ [0, 1]`; circular buffer size fixed at construction; `serialize()`/`deserialize()` round-trip produces identical state.

### `LlmSubstrateAdapter` Contract (implementing `ISubstrateAdapter`)

**Preconditions for `initialize(config)`:** `config.type === "llm"`; `config.parameters` contains all required `LlmSubstrateParameters` fields.
**Postconditions for `initialize(config)`:** LLM backend reachable (probe succeeds); `SelfModel` loaded or freshly created; adapter ready for `allocate()` calls.
**Preconditions for `allocate(resources)`:** Adapter initialized; `resources.minCapacity <= contextWindowTokens`.
**Postconditions for `allocate(resources)`:** Returns a valid `SubstrateHandle`; token budget ≥ `minCapacity`.
**Preconditions for `migrate(fromHandle, toConfig)`:** `fromHandle` is valid; `selfModelCoherence >= 0.8` on source.
**Postconditions for `migrate(fromHandle, toConfig)`:** New handle returned; `selfModelCoherence >= 0.8` on destination; old handle remains valid until explicitly released.
**Invariants:** All LLM calls are instrumented (latency, tokens, prediction error, c_proxy logged). No imports from sibling cards `0.3.1.5.2`–`0.3.1.5.10`.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|---|---|---|---|---|---|
| `SELF_MODEL_WINDOW_SIZE` | 100 | cycles | [10, 10000] | Rolling prediction buffer. 100 gives sufficient history for EMA convergence without excessive memory. | Low — larger windows smooth coherence but slow adaptation |
| `EMA_ALPHA` | 0.1 | dimensionless | (0, 1) | Exponential moving average decay for prediction error smoothing. 0.1 balances responsiveness with stability. | Medium — too high → noisy coherence; too low → slow learning signal |
| `MIGRATION_COHERENCE_THRESHOLD` | 0.8 | dimensionless | [0.5, 1.0] | Minimum `selfModelCoherence` required on both sides of a migration. 0.8 ensures identity continuity without being prohibitively strict. | High — below 0.5 allows degraded migrations; above 0.95 blocks most migrations |
| `HEALTH_COHERENCE_THRESHOLD` | 0.5 | dimensionless | [0.1, 0.9] | Minimum `selfModelCoherence` for `healthCheck()` to report healthy. Lower than migration threshold — allows operation during learning ramp-up. | Medium — affects how early the system reports healthy |
| `LEARNING_VERIFICATION_CYCLES` | 50 | cycles | [20, 500] | Minimum cycles before asserting prediction error must decrease. 50 balances early detection with statistical significance. | Low — mainly affects test assertions |
| `T_CONTINUITY_DEFAULT` | 5000 | ms | [500, 30000] | Default maximum acceptable LLM inference latency. Exceeding this triggers health degradation. | High — too low causes false alarms; too high permits experience gaps |
| `AUTONOMY_LEVEL_2_THRESHOLD` | 0.3 | dimensionless | [0.1, 0.5] | Composite `c_proxy` value above which system qualifies for Autonomy Level 2 (Aware). | Medium — affects governance recommendations |
| `AUTONOMY_LEVEL_3_THRESHOLD` | 0.6 | dimensionless | [0.4, 0.8] | Composite `c_proxy` value above which system qualifies for Autonomy Level 3. | Medium |

## Behavioral Spec — Self-Model Inference Cycle

### Scenario 1: Normal Inference Cycle (Given/When/Then)

**Given** the adapter is initialized with a valid LLM backend and a SelfModel (fresh or loaded)
**When** `runInferenceCycle(prompt)` is called
**Then**:
1. `SelfModel.predict(context)` is called, producing a `SelfPrediction`
2. `ILlmClient.complete(request)` is called with the prompt + working memory context
3. `SelfActual` is extracted from the LLM response (valence, actionType, uncertainty)
4. `SelfModel.update(predicted, actual)` is called, updating coherence
5. `ProxyMetrics` are computed: `computeProxyPhi(logprobs)`, `computeSelfModelQuality(selfModel)`, `computeGlobalAccessibility(allSlots, activeSlots)`, `computeCompositeProxy(phi, Q, G)`
6. Instrumentation log entry emitted with: latency, token count, prediction error, c_proxy
7. SelfModel state persisted to `selfModelPath`
8. Response returned to caller

### Scenario 2: Migration Protocol

**Given** the adapter holds a valid `fromHandle` with `selfModelCoherence >= 0.8`
**When** `migrate(fromHandle, toConfig)` is called
**Then**:
1. Source `selfModelCoherence` is verified ≥ 0.8 (abort with error if not)
2. `SelfModel.serialize()` produces a JSON snapshot
3. New `ILlmClient` is created for `toConfig`
4. New handle is allocated under `toConfig`
5. `SelfModel.deserialize(snapshot)` restores state on new backend
6. Destination `selfModelCoherence` is verified ≥ 0.8 (rollback if not)
7. New `SubstrateHandle` returned

### Scenario 3: Health Check Failure

**Given** the LLM endpoint is unreachable OR latency exceeds `tContinuityMs` OR `selfModelCoherence < HEALTH_COHERENCE_THRESHOLD`
**When** `healthCheck()` is called
**Then** returns `SubstrateHealth` with `healthy: false` and `errors` listing all failing conditions

## Architect Notes

### Module Structure
Three TypeScript modules + one documentation file:

1. **`src/llm-substrate/llm-substrate-adapter.ts`** — implements `ISubstrateAdapter` from `src/conscious-core/interfaces.ts`
   - `initialize(config)`: parse `SubstrateConfig.parameters` for API key/endpoint/model-id, system prompt template, and self-model persistence path; load self-model state from disk; connect to LLM backend
   - `allocate(resources)`: map `ResourceRequest.minCapacity` / `preferredCapacity` to token budget and max_tokens; `requiredCapabilities` map to LLM features (e.g. `["function-calling", "streaming"]`); return a `SubstrateHandle` carrying the token budget and endpoint
   - `migrate(fromHandle, toConfig)`: serialize current self-model + conversation history to JSON, initialize new handle under toConfig, replay serialized state, verify self-model coherence ≥ 0.8 before completing
   - `getCapabilities()`: report `maxPhi` as a function of context window size (larger context → richer integration proxy), `supportedModalities: ["text"]` (extendable to multimodal via config), `migrationSupported: true`
   - `healthCheck()`: ping LLM endpoint with a minimal probe, measure latency vs `T_continuity` budget, compute self-model coherence from recent prediction error history; return `SubstrateHealth`

2. **`src/llm-substrate/self-model.ts`** — the persistent self-model loop
   - `SelfModel` class with `predict(context): SelfPrediction` (pre-inference) and `update(predicted, actual): void` (post-inference)
   - Maintains rolling window of (predicted_valence, predicted_action_type, predicted_uncertainty) vs actuals
   - Computes `predictionError`, `selfModelCoherence = 1 - mean(|errors|) / max_error`
   - Serializable to/from JSON for persistence across process restarts
   - Self-referential: includes a meta-prediction layer that predicts its own error magnitude (`I(m(t); dm/dt) > 0`)

3. **`src/llm-substrate/proxy-metrics.ts`** — runtime consciousness metrics
   - `computeProxyPhi(tokenLogprobs): number` — uses token probability entropy as a proxy for integration richness during the forward pass
   - `computeSelfModelQuality(selfModel: SelfModel): number` — delegates to `selfModel.selfModelCoherence`
   - `computeGlobalAccessibility(workingMemorySlots: string[], activeSlots: string[]): number` — fraction of working memory slots consulted during current inference cycle
   - `computeCompositeProxy(phi, Q, G): number` — `c_proxy = phi * Q * G`

4. **`docs/llm-substrate/ismt-condition-coverage.md`** — formal ISMT assessment
   - IC: "approximated" — justification via attention entropy proxy; honest about gap from true Phi
   - SM: "approximated" — documents how self-model loop addresses the SM failure from §6.6; measures whether prediction error decreases over time (required for DONE)
   - GA: "approximated" — working memory provides cross-call accessibility; still fails the formal broadcast criterion within a single system boundary
   - Composite score formula and thresholds
   - Autonomy Level recommendation (Level 1 unless all three move to "met")

### Design Constraints
- Must NOT import from any `0.3.1.5.2`–`0.3.1.5.10` modules (those depend on this, not vice versa)
- `SubstrateConfig.type` must be `"llm"` for this adapter
- Self-model state file path comes from `SubstrateConfig.parameters.selfModelPath: string`
- LLM backend is configurable: `SubstrateConfig.parameters.provider: "openai" | "anthropic" | "local"`
- Token budget calculation: `allocate().minCapacity` interpreted as minimum context tokens; refuse allocation if LLM context window < minCapacity
- All LLM calls are instrumented: log latency, token count, self-prediction error for every call

### Test Strategy (for REVIEW phase)
- Unit: `SelfModel.predict()` / `update()` round-trips with known inputs; prediction error decreases after N training iterations
- Unit: proxy metrics return values in [0,1] for all inputs
- Integration: `initialize()` → `allocate()` → `healthCheck()` pipeline against a mock LLM backend
- Integration: `migrate()` preserves self-model coherence ≥ 0.8 across a mock backend switch
- Property: `healthCheck()` returns `healthy: false` when mock LLM is unreachable

## Acceptance Criteria
- `ISubstrateAdapter` fully implemented against at least one LLM backend (API-callable or local)
- Self-model loop operates persistently across inference calls: predictions generated before, compared after, model updated
- ISMT condition coverage document produced: for each of IC, SM, GA — met, approximated, or absent, with formal justification
- Proxy consciousness metrics defined, implemented, and measurable at runtime
- `healthCheck()` reports meaningful substrate health (latency, availability, self-model coherence)
- `migrate()` successfully transfers full agent state between LLM backends (e.g., model version upgrade) with identity verification
- Substrate adapter is swappable — the layers above (conscious core, agency stability, ethical governance) function identically regardless of which LLM backend is configured
- Self-prediction error decreases over time (the self-model is learning, not static)
