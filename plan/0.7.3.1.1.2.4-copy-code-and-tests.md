# 0.7.3.1.1.2.4 Copy Code Links and Test Artifacts [PLAN]

Populate `artifacts/code/` with link configuration files and `artifacts/tests/` with test reports, coverage data, etc.

**Acceptance Criteria**

1. `source-links.yaml`, CI pipeline files, and any other code‑link artifacts reside in `artifacts/code/`.  
2. All test reports (`*.xml`, `*.html`, coverage `*.json`) are present in `artifacts/tests/`.  
3. Every added file is staged in git (`git add` and `git status`).  
4. Each category (`code`, `tests`) contains at least one file.  
5. A `code/manifest.json` and `tests/manifest.json` enumerate files with their hashes.
