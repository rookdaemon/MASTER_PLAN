# 0.7.3.1.1.1.3.2 Verify Permissions via Script [IMPLEMENT]

Create and run a verification script that checks the CI user’s access rights.

**Acceptance Criteria**
1. A script `scripts/verify-permissions.sh` is added; it uses `stat` to confirm UID/GID read/write bits on each artifact sub‑folder.  
2. The script exits with status `0` only when all directories meet the required permissions.  
3. Script outputs a concise report listing any violations.  
4. The script is executed as part of the CI pipeline step and fails the build if verification fails.  
5. The script file is staged for commit alongside permission changes.

**File to Create**
- **Path:** `scripts/verify-permissions.sh`
- **Type:** Bash script
