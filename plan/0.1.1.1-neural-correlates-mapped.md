# 0.1.1.1 Neural Correlates of Consciousness Fully Mapped [DONE]
## Description

F1.1 — Identify and map the complete set of neural correlates of consciousness (NCCs). This means determining which neural structures, processes, and activity patterns are reliably associated with subjective experience, across all modalities (visual, auditory, somatosensory, emotional, etc.).

This includes:
- Cataloguing known NCCs (e.g. thalamocortical loops, posterior cortical hot zone, claustrum, default mode network)
- Distinguishing NCCs proper from prerequisites and consequences of consciousness
- Mapping correlates across altered states (sleep, anesthesia, psychedelics, meditation, pathological states)
- Resolving open questions (e.g. prefrontal vs. posterior cortex debate)

## Decision

### D1: Deliverable Format — Structured Markdown Catalogue

**Context:** This card produces a research synthesis, not software. The deliverable must serve both human readers and downstream cards (0.1.1.2, 0.1.1.3, 0.1.1.4) that consume its findings programmatically.

**Options Considered:**
1. **Structured Markdown document** — A single `.md` file with a consistent entry schema, organized by sections. Human-readable, version-controllable, parseable by downstream agents.
2. **JSON/YAML knowledge base** — Machine-first format with entries as structured data objects. Better for automated consumption but harder for human authoring and review.
3. **Multiple modality-specific files** — One file per modality/section. Better separation of concerns but fragments the catalogue and complicates cross-referencing.

**Choice:** Option 1 — Structured Markdown document (`docs/consciousness-science/ncc-catalogue.md`).

**Rationale:** The primary consumers are agents working on downstream cards; structured Markdown with a consistent schema is both human-reviewable and machine-parseable. A single file keeps all NCC data co-located for cross-referencing. The entry schema (see Contracts) provides sufficient structure for downstream extraction.

**Consequences:**
- Single source of truth for all NCC data
- Must maintain strict schema consistency across all entries
- Large file, but manageable for a knowledge catalogue

### D2: NCC Taxonomy — Three-Level Type System

**Context:** NCCs vary in scope. A classification system is needed.

**Options Considered:**
1. **Chalmers' content/background/full NCC trichotomy** — Well-established in the literature (Chalmers 2000). Distinguishes correlates of specific contents from enabling conditions from minimal sufficient substrates.
2. **Binary content/state distinction** — Simpler but loses the "full NCC" concept needed for engineering consciousness.

**Choice:** Option 1 — Content NCC / Background NCC / Full NCC trichotomy.

**Rationale:** The three-level system maps directly to what downstream cards need: background NCCs inform substrate requirements (0.1.1.3), content NCCs inform metrics (0.1.1.4), and full NCCs inform computational theory (0.1.1.2).

**Consequences:**
- Every entry must be classified into exactly one type
- Some entries may have debated classifications — these are flagged in open questions

### D3: Prerequisite/NCC-proper/Consequence Distinction Framework

**Context:** A critical methodological challenge is separating true correlates from mere prerequisites (necessary but not part of the NCC) and consequences (caused by consciousness but not constitutive of it).

**Options Considered:**
1. **Tripartite classification with explicit decision criteria** — Each entry tagged with causal status and distinction, supported by a dedicated section explaining the framework and experimental paradigms (after de Graaf et al. 2012).
2. **Flat correlate list without causal classification** — Simpler but fails acceptance criterion #2.

**Choice:** Option 1 — Tripartite classification with decision criteria.

**Rationale:** Directly required by acceptance criteria. The distinction is essential for downstream cards: only NCCs-proper should constrain computational theory.

**Consequences:**
- Requires careful evidence evaluation for each entry
- Some classifications will be provisional — marked as such

## Contracts

### C1: NCC Catalogue Entry Schema

Each NCC entry in the catalogue MUST conform to this schema:

**Preconditions:**
- Entry name is unique within the catalogue
- At least one empirical paradigm or citation supports the entry

**Schema:**
```
### <NCC Name>
- **Type:** content NCC | background NCC | full NCC
- **Modality:** visual | auditory | somatosensory | emotional | cognitive | cross-modal
- **Structures:** [list of brain regions/circuits]
- **Key evidence:** [citations/paradigms]
- **Causal status:** correlational | necessary | sufficient | necessary-and-sufficient
- **Distinction:** NCC-proper | prerequisite | consequence
- **Open questions:** [unresolved issues]
```

**Postconditions:**
- Every field is populated (use "None identified" for open questions if applicable)
- Type is one of: `content NCC`, `background NCC`, `full NCC`
- Causal status is one of: `correlational`, `necessary`, `sufficient`, `necessary-and-sufficient`
- Distinction primary value is one of: `NCC-proper`, `prerequisite`, `consequence`; for genuinely debated entries, use format `<primary> (debated: <alternative> per <theory>)` where primary reflects the current weight of evidence; entries with insufficient evidence to assign a primary value use `uncertain (<explanation>)`

**Invariants:**
- No two entries share the same name
- Every entry references at least one empirical source
- Distinction classification is justified by the framework in Section 5 of the catalogue

### C2: Catalogue Section Structure

The catalogue MUST contain these sections in order:

1. Background NCCs
2. Content NCCs by Modality (visual, auditory, somatosensory, emotional, cognitive)
3. Cross-Modal and Integrative NCCs
4. NCCs Across Altered States
5. Prerequisite / NCC-proper / Consequence Distinction Framework
6. Open Debates and Discriminating Experiments

**Postconditions:**
- Each section contains ≥1 entry or substantive content
- Section 4 covers: sleep, anesthesia, coma/vegetative/minimally-conscious, psychedelics, meditation, epilepsy
- Section 6 includes ≥1 proposed discriminating experiment per open debate

### C3: Interface to Downstream Cards

The catalogue provides these data to consumers:

| Consumer | Required Data | Extraction Method |
|---|---|---|
| 0.1.1.2 Computational Theory | Entries with causal status `necessary` or `sufficient`; distinction = `NCC-proper` | Filter by causal status and distinction fields |
| 0.1.1.3 Substrate Independence | All entries' Structures fields | Extract structures lists |
| 0.1.1.4 Consciousness Metrics | Entries with measurable neural signatures | Filter entries with empirical paradigms in key evidence |
