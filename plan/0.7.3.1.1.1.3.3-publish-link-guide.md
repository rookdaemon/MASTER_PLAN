# 0.7.3.1.1.1.3.3 Publish & Link Guide [PLAN]

Make the usage guide part of the public documentation:

* Commit `docs/traceability/traceability-matrix-usage.md` to the main branch with a descriptive commit message.  
* Add a link to the guide in the repository’s main documentation index (`README.md` or `docs/index.md`).  
* Verify that the link renders correctly on the repository’s GitHub Pages site (if applicable).  
* Tag the commit with `v0.7.3-usage-guide` for traceability.

### Acceptance Criteria
1. Guide file is present in the repository’s `docs/traceability/` folder on the `main` branch.  
2. The main documentation index contains a bullet‑point link labeled “Traceability Matrix Usage Guide”.  
3. The link resolves without 404 errors on both GitHub UI and GitHub Pages (if enabled).  
4. Commit is tagged `v0.7.3-usage-guide`.  
5. A short release note documenting the addition is added to `CHANGELOG.md`.
