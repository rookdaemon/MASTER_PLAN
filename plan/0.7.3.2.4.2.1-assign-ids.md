# 0.7.3.2.4.2.1 Assign IDs [PLAN]

Assign a unique identifier to each collected comment and embed that ID in the comment’s source file.

**Acceptance Criteria**
1. IDs follow the pattern `FB-###` (e.g., FB-001) and are sequential with no gaps.
2. Each comment file is updated to include a front‑matter field `id: FB-###`.
3. A master index (`ids-index.csv`) listing comment file path ↔ ID is generated.
4. The index is committed to version control in the `feedback/` directory.
5. A quick‑look script verifies that every comment file contains a valid ID.
