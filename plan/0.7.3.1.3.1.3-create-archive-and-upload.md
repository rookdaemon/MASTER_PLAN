# 0.7.3.1.3.1.3 Create Archive and Upload [PLAN]

Assemble the tagged source code, verified binaries, configuration files, and deployment scripts into a single version‑controlled archive, upload it to the artifact repository, and record its hash.

### Acceptance Criteria
1. All required files (tagged sources, binaries, configs, scripts) are gathered into a `delivery/` directory.  
2. The directory is compressed into a reproducible archive (e.g., `delivery-vX.Y.Z.tar.gz`).  
3. A SHA‑256 hash file (`delivery-vX.Y.Z.sha256`) is generated for the archive.  
4. The archive is uploaded to the designated artifact repository with a naming convention that includes the version tag.  
5. The archive location and its SHA‑256 hash are documented in the project’s release notes.
