# 0.7.3.1.1.3.1.2 Add CI Validation [PLAN]

Extend the CI pipeline to invoke `scripts/collect-artifact-list.sh` and verify that `artifacts/artifact-list.json` is generated, non‑empty, and contains valid JSON.

**Acceptance Criteria**
1. CI step runs `scripts/collect-artifact-list.sh` and fails if the script exits non‑zero.
2. A subsequent CI step checks that `artifacts/artifact-list.json` exists and its file size > 0.
3. The CI step validates the JSON syntax (e.g., using `jq` with `--exit-status`).
4. CI failure logs include the script’s stderr output and a clear message about which validation failed.
5. The CI configuration is documented in `ci/README.md` with the new steps and sample output.
