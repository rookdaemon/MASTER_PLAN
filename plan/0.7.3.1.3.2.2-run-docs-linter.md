# 0.7.3.1.3.2.2 Run documentation linter [PLAN]

## Description
Execute the project's doc‑linter/formatter on `docs/codebase-map.md` and capture its exit status and any output.

## Acceptance Criteria
1. The command `npm run lint:docs docs/codebase-map.md` (or the configured linter command) is executed.  
2. The process exits with status 0.  
3. No linting errors or warnings are emitted to stdout/stderr.  
4. The full linter output (even if empty) is saved to `artifacts/docs-lint-output.txt`.  
5. The subtask prints the path to the artifact on stdout.
