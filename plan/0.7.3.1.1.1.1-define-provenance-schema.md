# 0.7.3.1.1.1.1 Define Provenance Schema [PLAN]

Create or update `schemas/provenance-schema.json` to formally describe the structure of `provenance.json`.

**Acceptance Criteria**
1. Schema declares an object with a top‑level `deliverables` array.
2. Each array item is an object with required string fields `id` and `type`, and a required array `artifactPaths` of strings.
3. Schema uses JSON‑Schema Draft‑07 (or later) keywords and can be validated with `ajv` or `jsonschema` CLI.
4. The schema file is added to version control with a clear commit message “Add provenance JSON schema”.
5. Documentation in `README.md` links to the schema and explains each field.
