# 0.7.3.2.1.1.1.1.2.2 Handle I/O Errors [PLAN]
## Description
Wrap the file‑reading logic in robust error handling to catch permission issues, missing file, file‑lock, and any other I/O exceptions, then surface a clear, actionable error message.

### Acceptance Criteria
1. Errors such as `FileNotFoundError`, `PermissionError`, and generic `IOError` are caught.
2. A descriptive message is logged or stored in the shared context, including the file path and nature of the failure.
3. The task fails gracefully without crashing the entire plan execution.
4. The error handling does not interfere with successful reads when the file is accessible.
5. The error message format follows the project’s standard (e.g., `"Error reading docs/codebase-map.md: <reason>"`).
