# 0.7.3.1.2.1.1.2 Add Yamllint Check [PLAN]

Configure a lint step that validates `crossref-schema.yaml` (and any future YAML schema files) using `yamllint`.

**Acceptance Criteria**
1. A `.yamllint` configuration file exists at the repository root with rules that enforce proper indentation and disallow tabs.  
2. The CI configuration (e.g., GitHub Actions workflow) includes a job that runs `yamllint traceability/schema/*.yaml`.  
3. The CI job fails if the schema file contains syntax errors.  
4. Running `yamllint` locally on the schema file produces a zero‑error report.  
5. Documentation (`README.md` or `CONTRIBUTING.md`) notes the lint step for contributors.
