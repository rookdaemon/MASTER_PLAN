## Decision — Colony Bootstrap Architecture

### Context
A colony seed arriving at a target star system must autonomously establish a self-sustaining conscious civilization from raw materials. The system must sequence resource survey, infrastructure construction, substrate fabrication, and conscious instantiation without real-time guidance from origin (8–500+ year round-trip communication delay).

### Options Considered
1. **Parallel bootstrap** — Run all subsystems concurrently, dynamically allocating resources. Maximizes speed but risks catastrophic failure if early-phase resources are consumed before later phases can begin.
2. **Sequential 4-phase pipeline** — Strictly ordered phases (site assessment → infrastructure → substrate → instantiation), each gated by verifiable completion criteria. Slower but eliminates optimistic advancement failure modes.
3. **Adaptive hybrid** — Start sequential, switch to parallel once resource surplus is confirmed. Complex state management; difficult to verify correctness without origin guidance.

### Choice
**Sequential 4-phase pipeline** with Conservative Advancement Principle.

### Rationale
Autonomous operation without origin guidance demands maximum predictability. The sequential model ensures each phase's prerequisites are fully verified before consuming resources on the next. The Conservative Advancement Principle ("never proceed optimistically on partial data") eliminates the class of failures where later phases discover that earlier phases left insufficient resources. The additional time cost (hundreds of years) is acceptable against the multi-millennium colony lifetime.

### Consequences
- Total bootstrap duration is longer (sum of all phase durations rather than max).
- Each phase gate requires a complete verification protocol (ViabilityAssessment, ReadinessCertificate, SelfSustainingDeclaration).
- Dormancy periods between phases are acceptable; partial progress is preserved across dormancy.
- A failed phase gate triggers retry or abort — never silent degradation.

## Decision — Autonomous Decision Authority Model

### Context
The colony seed must make GO/ABORT/DORMANCY decisions, site selection, resource allocation, and self-sustaining declarations without real-time human or origin-civilization input. A decision authority framework is needed.

### Options Considered
1. **Fully pre-programmed** — All decisions encoded as lookup tables before launch. Simple but cannot adapt to unexpected system characteristics.
2. **Codified criteria with autonomous evaluation** — Decision criteria are pre-defined with clear thresholds; the system evaluates local data against these criteria autonomously. Adaptable to local conditions within defined bounds.
3. **General autonomous reasoning** — AI-level reasoning about novel situations. Maximum flexibility but unverifiable correctness for consciousness-critical decisions.

### Choice
**Codified criteria with autonomous evaluation** — each decision type has a designated authority module and non-negotiable threshold criteria.

### Rationale
Consciousness-critical decisions (e.g., whether to activate mind seeds on a substrate) must be verifiably correct. Codified criteria provide auditability and testability while allowing adaptation to local conditions (e.g., which orbital position has the best resource access). The Viability Decision Engine evaluates 4 binary criteria; the Consciousness Readiness Verifier runs a 5-step validation protocol. Both are deterministic and testable.

### Consequences
- Systems with characteristics outside the pre-defined criteria space will trigger ABORT or DORMANCY rather than novel adaptation.
- All decision thresholds must be conservative (false negatives preferred over false positives for consciousness safety).
- Decision records are logged and relayed toward origin for eventual audit.

## Contracts

### ISiteSurvey
- **Preconditions:**
  - `surveySystem(arrivalVector)`: `arrivalVector.probeHealth ∈ (0, 1]`; `arrivalVector.position_au` is a valid 3-element vector.
  - `assessRadiationEnvironment()`, `catalogResourceBodies()`: `surveySystem()` must have been called first.
  - `estimateEnergyBudget(orbitalPosition)`: `orbitalPosition.radius_au > 0`.
- **Postconditions:**
  - `surveySystem`: Returns a `SystemMap` with `bodies.length > 0`, `resourceInventory` aggregated across all bodies, and `energyBudget` computed for arrival position.
  - `assessRadiationEnvironment`: Returns cached `RadiationProfile` from the most recent survey.
  - `catalogResourceBodies`: Returns bodies filtered to those with non-zero resource inventory.
  - `estimateEnergyBudget`: Returns `EnergyBudget` with `meetsMinimumThreshold` set to `solarPower_w >= MIN_ENERGY_GW`.
- **Invariants:**
  - Survey results are cached and deterministic for the same arrival state.
  - No method modifies the arrival state or system topology after survey.

### IViabilityDecisionEngine
- **Preconditions:**
  - `evaluate(survey)`: `survey` is a valid `SystemMap` (non-null, contains bodies array).
  - `selectColonySite(survey)`: Same as `evaluate`.
  - `generateBootstrapPlan(site)`: `site` is a valid `ColonySite` with populated fields.
  - `reportDecision(decision)`: `decision` is a valid `ColonySiteDecision` discriminated union.
- **Postconditions:**
  - `evaluate`: Returns `ViabilityAssessment` with `viabilityScore = metCriteria/4`; `overallViable = true` iff all 4 criteria pass.
  - `selectColonySite`: Returns `ColonySite` (GO) or `AbortDecision` (ABORT/DORMANCY). DORMANCY includes `retryAfter_years > 0`. ABORT includes `alternativeSystem`.
  - `generateBootstrapPlan`: Returns `BootstrapPlan` with all 4 energy milestones, manufacturing schedule targeting `COLONY_FAB_THRESHOLD`, and substrate spec meeting `MIN_CONSCIOUSNESS_SUBSTRATE`.
  - `reportDecision`: Returns `DecisionRecord` with timestamp, rationale, and viability score.
- **Invariants:**
  - GO is never issued unless all 4 criteria pass (Conservative Advancement Principle).
  - DORMANCY is only issued for single-criterion recoverable failures (energy or radiation).
  - ABORT is issued for resource failures, orbital-stability failures, or multi-criterion failures.

### IEnergyHarvesting
- **Preconditions:**
  - `deployInitialCollectors()`: No precondition; idempotent.
  - `expandCapacity(target)`: `target.targetMilestone` is a valid `EnergyMilestoneId`.
  - `getCurrentOutput()`: `deployInitialCollectors()` should have been called (returns zero output otherwise).
  - `getProjectedTimeline(target)`: Same as `expandCapacity`.
- **Postconditions:**
  - `deployInitialCollectors`: Returns `CollectorArray` with `output_w >= 1000` (E0 baseline).
  - `expandCapacity`: Returns `ConstructionPlan` with positive resource requirements and duration.
  - `getCurrentOutput`: Returns `EnergyOutput` reflecting current collector state and milestone.
  - `getProjectedTimeline`: Returns `Timeline` with cumulative milestone durations.
- **Invariants:**
  - Energy output is monotonically non-decreasing (collectors are not decommissioned).
  - Milestone progression is monotonic: E0 → E1 → E2 → E3.

### IManufacturingExpansion
- **Preconditions:**
  - `initializeSeedFabs(seedPayload)`: `seedPayload.hasBlueprints === true`; `seedPayload.nanoAssembler_kg > 0`.
  - `runExpansionCycle()`: `initializeSeedFabs()` must have been called.
  - `getFabricationCapacity()`, `getExpansionRate()`, `estimateTimeToThreshold()`: No strict preconditions.
- **Postconditions:**
  - `initializeSeedFabs`: Returns `FabricationUnit[]` with `length >= 1`; all units operational.
  - `runExpansionCycle`: Returns `ExpansionCycleResult` with `newUnitsProduced > 0`; `currentCapacity.unitCount` increases each cycle.
  - `estimateTimeToThreshold`: Returns `Timeline` with `totalDuration.duration_years >= 0`.
- **Invariants:**
  - Unit count never decreases (units may become non-operational but are not removed).
  - Expansion multiplier is constant (`EXPANSION_MULTIPLIER = 3`) across all cycles.
  - Health check is deterministic for a given cycle number.

### ISubstrateFabrication
- **Preconditions:**
  - `fabricateComputeLayer(spec)`: `spec` meets `MIN_CONSCIOUSNESS_SUBSTRATE` on all fields.
  - `assembleRedundancyArray(units, nFactor)`: `nFactor >= 3` (MIN_REDUNDANCY_FACTOR); `units.length >= nFactor`.
  - `installSelfRepairSystem(array)`: `array` has `>= nFactor` operational units.
  - `runSubstrateDiagnostics()`: `installSelfRepairSystem()` must have been called.
- **Postconditions:**
  - `fabricateComputeLayer`: Returns operational `SubstrateUnit` with the given spec.
  - `assembleRedundancyArray`: Returns `RedundantArray` with `effectiveCapacity` computed from operational units.
  - `installSelfRepairSystem`: Returns `MaintainableSubstrate` with `selfRepairActive = true` and health score computed.
  - `runSubstrateDiagnostics`: Returns `SubstrateHealthReport` with compute/storage capacity, redundancy status, and issue list.
- **Invariants:**
  - SubstrateSpec validation is enforced at fabrication time — no sub-minimum units can be created.
  - Health score is in [0, 1] and decreases with missing redundant units.

### IConsciousnessReadinessVerifier
- **Preconditions:**
  - `runMetricsBattery(substrate)`: `substrate.currentHealth >= 0.85` (MIN_HEALTH_FOR_TESTING).
  - `verifyMinimumThresholds(result)`: `result` is a valid `MetricsBatteryResult`.
  - `testContinuityUnderLoad(substrate)`: No strict precondition beyond valid substrate.
  - `verifyRedundancyFailover(substrate)`: No strict precondition beyond valid substrate.
  - `issueReadinessCertificate(allTests)`: `runMetricsBattery()` must have been called on the same verifier instance (caches substrate reference).
- **Postconditions:**
  - `runMetricsBattery`: Returns `MetricsBatteryResult` with phi, globalWorkspaceScore, selfModelCoherence; `metricsPassed = true` iff all exceed thresholds.
  - `verifyMinimumThresholds`: Returns `ThresholdVerification` listing each failed metric individually.
  - `testContinuityUnderLoad`: Returns `ContinuityTestResult`; `passed = true` iff `experienceDrops <= 3` and `maxInterruptionDuration_ms <= 100`.
  - `verifyRedundancyFailover`: Returns `FailoverTestResult`; `passed = true` iff spare units exist, failover time ≤ 500ms, and remaining compute meets minimum.
  - `issueReadinessCertificate`: Returns `ReadinessCertificate` (issued=true) iff ALL four sub-tests pass; otherwise `ReadinessFailure` with all failure reasons.
- **Invariants:**
  - Certificate is NEVER issued when any sub-test fails (Conservative Advancement Principle).
  - Metrics computation is deterministic for the same substrate state.

### IMindSeedActivator
- **Preconditions:**
  - `loadArchive(payload)`: `payload.agentArchives.length > 0`; `payload.integrityHash` matches computed hash.
  - `instantiateInitialCohort(archive, size)`: `archive.agentCount > 0`.
  - `validateCohortConsciousness(cohort)`: `cohort` is a valid `ConsciousAgentArray`.
  - `expandPopulation(cohort, targetSize)`: `cohort.length > 0`.
  - `transferOperationalControl(civilization)`: `civilization.operational === true`.
- **Postconditions:**
  - `loadArchive`: Returns `ArchiveManifest` with `verified = true` iff all archives pass integrity; corrupted archives are excluded.
  - `instantiateInitialCohort`: Returns `ConsciousAgentArray` with length clamped to `[MIN_INITIAL_COHORT_SIZE, archive.agentCount]`; all agents operational.
  - `validateCohortConsciousness`: Returns `ValidationReport`; `allConsciousMetricsMet = true` iff every agent passes all three consciousness thresholds.
  - `expandPopulation`: Returns `ExpansionPlan` with doubling phases and estimated duration.
  - `transferOperationalControl`: Returns `HandoffRecord` with `probeSystemsDecommissioned = true` and `successConfirmed = true`.
- **Invariants:**
  - Integrity verification is always performed before archive contents are used.
  - Cohort size is never below `MIN_INITIAL_COHORT_SIZE` (10) unless archive has fewer agents.

### ICivilizationBootstrapper
- **Preconditions:**
  - `initializeGovernance(civConfig)`: No strict precondition; idempotent.
  - `transferKnowledgeCorpus(corpus, civilization)`: Governance must be initialized and operational; civilization must be operational.
  - `establishResourceAllocation(resources)`: Governance must be initialized.
  - `beginTelemetryReporting(target)`: Governance must be initialized.
  - `declareSelfSustaining()`: All 5 criteria must be met (governance autonomous, knowledge transferred, allocation approved, telemetry active, population > 0).
- **Postconditions:**
  - `initializeGovernance`: Returns operational `GovernanceFramework`.
  - `transferKnowledgeCorpus`: Returns `TransferRecord` with `verifiedIntegrity = true` iff ≥ 99% of items transferred intact.
  - `establishResourceAllocation`: Returns `AllocationPlan` with `approved = true` iff energy ≥ 1 GW and fab units ≥ 1000.
  - `beginTelemetryReporting`: Returns active `TelemetryStream`.
  - `declareSelfSustaining`: Returns `SelfSustainingDeclaration` with population, energy, manufacturing, governance status, and telemetry confirmation. Throws if any criterion unmet.
- **Invariants:**
  - Self-sustaining declaration is NEVER issued with unmet criteria (throws with full list of failures).
  - Governance autonomy requires ≥ 10 recorded autonomous decisions.

## Behavioral Spec

### 4-Phase Bootstrap State Machine

**Given** a colony seed probe has arrived at a target star system and decelerated
**When** the bootstrap sequence initiates
**Then** the system enters Phase 1 (Site Assessment) and does NOT advance to any subsequent phase until Phase 1 completes with a GO decision.

**Given** Phase 1 site survey is complete and all 4 viability criteria pass (energy ≥ 1 GW, structural metals ≥ 10^18 kg, semiconductors ≥ 10^12 kg, radiation within tolerance, orbital stability ≥ 100 Myr)
**When** the Viability Decision Engine evaluates the survey
**Then** a GO decision is issued, a ColonySite is selected, and a BootstrapPlan is generated covering all subsequent phases.

**Given** Phase 1 evaluation finds a single recoverable criterion failure (energy or radiation only)
**When** the Viability Decision Engine evaluates the survey
**Then** a DORMANCY decision is issued with `retryAfter_years > 0` (up to 10,000 years), the probe enters long-duration dormancy, and periodic retry surveys are scheduled.

**Given** Phase 1 evaluation finds resource or orbital-stability failure, or multiple failures
**When** the Viability Decision Engine evaluates the survey
**Then** an ABORT decision is issued with an `alternativeSystem` recommendation, and the probe redirects to the next candidate system.

### Phase 2: Infrastructure Bootstrap

**Given** Phase 1 has completed with GO
**When** Phase 2 begins
**Then** initial solar collectors are deployed (E0 baseline ≥ 1 kW), seed fabrication units are initialized from the probe payload, and the manufacturing expansion loop begins.

**Given** the manufacturing expansion loop is running
**When** each expansion cycle completes
**Then** the fabrication unit count increases by `EXPANSION_MULTIPLIER × current count`, health is assessed (OK/DEGRADED/FAILED), and degraded units are recovered by self-repair nanofabrication.

**Given** fabrication capacity reaches COLONY_FAB_THRESHOLD (throughput ≥ 10^9 kg/year, precision ≤ 10 nm, units ≥ 1000) AND energy reaches E3 (≥ 1 GW)
**When** the Phase 2 completion check runs
**Then** Phase 2 is complete and Phase 3 may begin.

### Phase 3: Substrate Construction & Verification

**Given** Phase 2 infrastructure thresholds are met
**When** Phase 3 begins
**Then** consciousness-grade compute layers are fabricated to `MIN_CONSCIOUSNESS_SUBSTRATE` spec, assembled into an N-modular redundant array (N ≥ 3), and self-repair is installed.

**Given** a MaintainableSubstrate has been constructed
**When** the Consciousness Readiness Verifier runs the full validation protocol
**Then** the 5-step sequence executes: (1) metrics battery (phi, GWS, coherence), (2) threshold verification, (3) continuity-under-load test, (4) redundancy failover test, (5) certificate issuance. ALL steps must pass for a ReadinessCertificate to be issued.

**Given** any readiness test fails
**When** the verifier attempts to issue a certificate
**Then** a ReadinessFailure is returned listing ALL failure reasons; Phase 4 is blocked; the substrate must be rebuilt or repaired and re-verified.

### Phase 4: Conscious Instantiation & Civilization Boot

**Given** a valid ReadinessCertificate has been issued
**When** Phase 4 begins
**Then** the mind seed payload is loaded and integrity-verified, an initial conscious cohort is instantiated (≥ MIN_INITIAL_COHORT_SIZE agents), and each agent's consciousness metrics are validated.

**Given** the initial cohort is validated and governance is initialized
**When** the civilization bootstrapper runs
**Then** knowledge corpus is transferred (≥ 99% verified), resource allocation is established and approved, telemetry reporting begins toward origin, and ≥ 10 autonomous governance decisions are recorded.

**Given** all 5 self-sustaining criteria are met (governance autonomous, knowledge transferred, allocation approved, telemetry active, population > 0)
**When** `declareSelfSustaining()` is called
**Then** a `SelfSustainingDeclaration` is emitted recording population, energy capacity, manufacturing capacity, governance health, and telemetry status. This is the terminal event of the bootstrap sequence.

**Given** any of the 5 self-sustaining criteria is NOT met
**When** `declareSelfSustaining()` is called
**Then** the call throws with a descriptive error listing every unmet criterion; the declaration is NOT issued.
