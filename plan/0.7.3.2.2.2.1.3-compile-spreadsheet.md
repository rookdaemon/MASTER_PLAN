# 0.7.3.2.2.2.1.3 Compile Candidate Fields Spreadsheet [REVIEW]

## Description
Merge the outputs of the documentation review and stakeholder interviews into a single, deduplicated spreadsheet (or markdown table) that satisfies the parent’s acceptance criteria.

## Acceptance Criteria
1. All candidate fields from both sources are merged with duplicates removed.  
2. Each row contains: Field Name, Purpose Description, Source(s) (e.g., “README”, “Owner interview”).  
3. The spreadsheet is saved in both .xlsx and .md formats in the team drive.  
4. A brief “Compilation Report” explains any decisions made (e.g., why certain duplicates were merged).  
5. The final artifact is shared with the team and a link is added to the parent task’s description.

## Execution Spec
**Artifact Paths to Create**
- `team-drive/field-catalog.xlsx` – Excel workbook containing the deduplicated table.
- `team-drive/field-catalog.md` – Markdown table version of the same data.
- `team-drive/compilation-report.md` – Short report describing merge decisions and any outstanding notes.

**Context (inline)**
- **Source A:** `team-drive/documentation-review-fields.md` – Markdown list/table of fields extracted from existing documentation.
- **Source B:** `team-drive/stakeholder-interview-fields.md` – Markdown list/table of fields gathered from stakeholder interviews.
- **Parent Task Description** (for link insertion):  
  *The parent task “Catalog Candidate Fields” expects a link to the compiled spreadsheet and report.*

**Steps for a 7B model**
1. Load the two source files listed above.  
2. Parse each into a list of records with columns: *Field Name*, *Purpose Description*, *Source*.  
3. Normalize field names (case‑insensitive, trim whitespace) and de‑duplicate records:
   - If two records share the same normalized name, keep one record.
   - Merge the *Source* column values, joining with commas.
   - Prefer the *Purpose Description* with the longer text; if equal, keep the first.  
4. Create a combined table with the three columns.  
5. Export the table to `team-drive/field-catalog.xlsx` using a simple CSV‑to‑XLSX conversion (first row as header).  
6. Render the same table to `team-drive/field-catalog.md` using GitHub‑flavored markdown table syntax.  
7. Write `team-drive/compilation-report.md` containing:
   - Header: “# Compilation Report – Candidate Fields”
   - Brief summary of total rows from each source, number of duplicates removed, any notable merge decisions (e.g., “‘UserID’ from README merged with ‘User Id’ from interview”).  
8. Edit the parent task file `plan/0.7.3.2.2.2.1-catalog-fields.md`:
   - Append a line under its description:  
     `Compiled spreadsheet: [field-catalog.xlsx](team-drive/field-catalog.xlsx) | [field-catalog.md](team-drive/field-catalog.md)`  
     `Compilation report: [compilation-report.md](team-drive/compilation-report.md)`

**Acceptance Assertions**
- `team-drive/field-catalog.xlsx` exists and contains a header row exactly matching `["Field Name","Purpose Description","Source(s)"]`.  
- `team-drive/field-catalog.md` exists and renders a markdown table with the same number of data rows as the XLSX file.  
- No duplicate *Field Name* entries exist in either artifact (case‑insensitive check).  
- `team-drive/compilation-report.md` exists and includes at least one bullet point summarizing duplicate handling.  
- The parent task file `plan/0.7.3.2.2.2.1-catalog-fields.md` now contains markdown links pointing to the three artifacts.

**Expected Output Format**
- Excel file (`.xlsx`) with a single worksheet named “Fields” containing the table.  
- Markdown table file (`.md`) with pipe‑separated columns and a header separator line.  
- Compilation report as a markdown document with a level‑1 heading and bullet points.
