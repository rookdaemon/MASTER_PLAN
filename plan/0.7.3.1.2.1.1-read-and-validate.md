# 0.7.3.1.2.1.1 Read and Validate Inventory [PLAN]

Read `inventory.txt` line‑by‑line, trim whitespace, ignore blank lines and comment lines starting with `#`, and ensure each line is a syntactically valid relative or absolute path.

**Acceptance Criteria**
1. The parser strips leading/trailing whitespace and skips empty/comment lines.
2. Each remaining line matches the pattern `^(\/|\.\/|[^\/\s].*)$`; invalid lines cause a fatal error with line number.
3. The parser outputs an in‑memory ordered array `rawEntries` preserving the original ordering.
4. Errors are logged to `logs/parse-inventory-errors.log` with clear messages.
5. The step completes in ≤ 5 seconds for 10 k lines.
