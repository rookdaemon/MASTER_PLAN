# 0.7.3.1.1.1.3.1 Apply CI User Permissions [IMPLEMENT]

Grant the CI user read/write access to the artifact directory hierarchy.

**Acceptance Criteria**
1. Execute `chmod -R u+rw traceability/plan-guardian/artifacts/` (or equivalent) targeting the CI user’s UID/GID.  
2. Ensure no group or other users gain write permission beyond existing policy (e.g., `chmod go-w`).  
3. The command runs successfully on all supported CI platforms (Linux/macOS).  
4. The operation is idempotent – re‑running does not alter correct permissions.  
5. Changes are staged for commit but not yet committed.
