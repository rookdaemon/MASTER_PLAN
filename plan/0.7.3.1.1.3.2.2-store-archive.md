# 0.7.3.1.1.3.2.2 Store Archive Asset [PLAN]

Move the generated archive to its permanent location, add it to Git LFS or a release asset, and commit the change.

### Acceptance Criteria
1. The archive is copied to `archives/traceability/` within the repository.  
2. If the archive size exceeds the LFS threshold, it is tracked with Git LFS; otherwise it is added as a release asset linked to the tag.  
3. A Git commit is created with message:  
   `chore: add traceability archive traceability-<tag>.[zip|tar.gz]`  
   and pushed to the main branch (or relevant release branch).  
4. Release metadata (URL, checksum) is recorded in the repository’s release notes.  
5. CI verifies that the archive is present in the expected location after the commit.
