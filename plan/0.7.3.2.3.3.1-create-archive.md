# 0.7.3.2.3.3.1 Create Distribution Archive [PLAN]

Package the updated source code, regenerated manifest, and accompanying documentation into a reproducible archive (ZIP for Windows, TAR.GZ for Unix).

**Acceptance Criteria**
1. The archive contains the complete `src/`, `docs/`, and `manifest.json` directories/files.
2. Archive naming follows the pattern `project‑v<new‑version>.<zip|tar.gz>`.
3. Archive creation succeeds on both Windows and Linux CI runners without errors.
4. The resulting archive size is recorded in a log file for later verification.
5. The archive file is stored in the `dist/` directory ready for checksum calculation.
