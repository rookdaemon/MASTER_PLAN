# 0.7.3.1.1.3.3.3 Ensure commit idempotency [PLAN]

Make the commit step repeatable: if a commit with the same content and message already exists, do not create a duplicate.

**Acceptance Criteria**
1. After creating the commit, compute the SHA of the new commit.
2. Search the git history (e.g., `git log`) for a previous commit with the same message “Export traceability package” and identical tree hash.
3. If such a commit exists, abort the new commit and report success, indicating the existing commit SHA.
4. If no such commit exists, allow the newly created commit to remain.
5. The task completes with a clear status indicating whether a new commit was made or an existing one was reused.
