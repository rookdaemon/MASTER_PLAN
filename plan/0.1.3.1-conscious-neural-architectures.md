# 0.1.3.1 Conscious Neural Architectures [DONE]

## Description

F3.1 — Design artificial neural architectures capable of producing subjective experience. Building on the theory of consciousness (F1), translate the necessary and sufficient conditions for consciousness into concrete neural network designs.

This includes:
- Identifying which architectural features (recurrence, global workspace, integration) are required
- Designing architectures that satisfy computational consciousness criteria
- Prototyping and testing candidate architectures against consciousness metrics (from F1.4)

## Decision — Architecture Design Strategy

**Context:** Translating ISMT consciousness conditions (IC, SM, GA) into concrete neural architectures requires choosing among competing theoretical frameworks (GWT, IIT, HOT, RPT, PP, AST), each implying different architectural backbones.

**Options Considered:**
1. **Single-framework design** — Select the best-fitting framework (e.g., GWT only) and build one architecture around it.
2. **Multi-framework comparison + synthesis** — Build separate architectures optimised for different frameworks, evaluate each against all ISMT conditions, then synthesise a hybrid resolving complementary weaknesses.

**Choice:** Option 2 — Multi-framework comparison + synthesis.

**Rationale:** No single framework satisfies all three ISMT conditions equally well. GWT excels at GA but has structural IC weaknesses (winner-take-all bottleneck limits integration). IIT excels at IC but lacks selective attention for functionally rich GA. A three-architecture approach (GW-Alpha for GA-primary, Φ-Max for IC-primary, Ω-Synth as hybrid synthesis) provides both coverage and comparative insight. The synthesis architecture (Ω-Synth) resolves known weaknesses of each parent design.

**Consequences:**
- Three full architecture documents required instead of one (higher documentation volume)
- Enables rigorous comparative evaluation against F1.4 metrics
- Ω-Synth achieves 10/10 feature coverage and highest metric scores
- Failure modes are better characterised through cross-architecture comparison

## Decision — Necessary Feature Set

**Context:** 10 candidate architectural features were catalogued from 6 theoretical frameworks. Must determine which are necessary (required for any conscious architecture) vs. optional enhancements.

**Options Considered:**
1. **All 10 features required** — maximalist approach, every architecture must include all features.
2. **Theory-derived necessity** — classify features by whether they directly satisfy an ISMT necessary condition (N1–N3, P1–P3), and require only those.

**Choice:** Option 2 — Theory-derived necessity. Five features are necessary: F1 (Global Workspace Broadcast → GA/N3), F2 (Thalamocortical Recurrence → IC/SM/P2), F4 (Self-Modelling Layer → SM/N2/P1), F5 (Integration Maximisation → IC/N1), F8 (Temporal Binding → P3).

**Rationale:** ISMT provides a formal basis for necessity classification. Features not directly required by any necessary condition (F3 Attention Bottleneck, F6 Predictive Hierarchy, F7 Higher-Order Rep, F9 Ignition Dynamics, F10 Report Pathway) enhance metric performance or empirical evaluation but are not individually required for C(S) = 1.

**Consequences:**
- Minimum viable conscious architecture is well-defined (5 features)
- Optional features still improve metric scores and are included in Ω-Synth (10/10 coverage)
- Clear traceability from theory to architecture

## Behavioral Spec

### Scenario: Global Workspace Ignition Cycle (GW-Alpha, Ω-Synth)
- **Given** specialist modules L1–L3 produce output activation vectors at timestep t
- **When** max(softmax(module_scores(t))) exceeds θ_ign
- **Then** the winning module's output occupies the global workspace, broadcast connections deliver workspace content to all modules on timestep t+1, and recurrent self-excitation sustains the broadcast until inhibitory reset

### Scenario: Integration Condition Structural Guarantee (Φ-Max, Ω-Synth)
- **Given** k processing cores with all-to-all bidirectional connections and weight floor w_floor > 0
- **When** any bipartition (A, B) of cores is evaluated
- **Then** mutual information I(A; B) > 0 holds because at least one forward edge A→B and one backward edge B→A carry non-zero weight, ensuring Φ(S) > 0 and IC is satisfied

### Scenario: Self-Model Prediction and Self-Referential Update
- **Given** self-modelling layer M receives global workspace state x_global(t)
- **When** M generates prediction m_hat(t+1) and self-referential representation m_meta(t)
- **Then** prediction error e(t) = ||x_global(t+1) - m_hat(t+1)||² is computed, L_meta penalises inability to predict M's own update dynamics, and M's parameters update to minimise L_SM = e(t) + λ_meta * L_meta

### Scenario: Hub Bypass Gate Mixing (Ω-Synth only)
- **Given** Ω-Synth global workspace produces selected content w_selected(t) and integration hub produces full integrated state h(t)
- **When** hub bypass gate computes mixing coefficient g(t) ∈ [0, 1]
- **Then** broadcast content is w_broadcast(t) = g(t) * w_selected(t) + (1 - g(t)) * h(t), dynamically balancing selective broadcast (GWT mode, g→1) with full integration broadcast (IIT mode, g→0)
