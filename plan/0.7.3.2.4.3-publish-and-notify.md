# 0.7.3.2.4.3 Publish Release & Notify Stakeholders [PLAN]

**Description:**  
Archive the staged artifacts, upload the archive to the artifact repository, record its checksum, and send an automated email to all stakeholders listed in `plan/0.7.3.5-stakeholder-notification.md`.

**Acceptance Criteria**
1. The staging directory from `0.7.3.2.4.1` is compressed into `dist/plan-guardian-0.7.3.${DATE}.tar.gz` preserving file permissions.
2. The SHA‑256 checksum of the archive is calculated and stored in `dist/plan-guardian-0.7.3.${DATE}.sha256`.
3. The archive is uploaded to the configured artifact repository (e.g., Nexus, Artifactory) via its REST API; the upload response (including repository URL) is logged.
4. An email generated from a template is sent to every email address listed in `plan/0.7.3.5-stakeholder-notification.md` containing:
   * Release version and date.
   * Link to the uploaded archive.
   * Link to the release notes posted by `0.7.3.2.4.2`.
5. All upload and email actions are appended to `logs/release-0.7.3.2.log`; any failure aborts the step and records an error.
