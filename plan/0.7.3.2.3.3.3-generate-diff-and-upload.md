# 0.7.3.2.3.3.3 Generate Diff Report and Upload Assets [PLAN]

Compare the previous manifest with the newly generated one, produce a human‑readable diff, and upload both the archive and diff report to the designated repository location.

**Acceptance Criteria**
1. The old manifest (`manifest‑old.json`) and new manifest (`manifest.json`) are diffed using a JSON‑aware tool, producing `manifest-diff.txt` that lists added, removed, and modified entries.
2. The diff report is stored in `dist/manifest-diff.txt` and included in release notes.
3. Both the archive (`dist/<archive>`) and diff report (`dist/manifest-diff.txt`) are uploaded to the repository (e.g., as GitHub Release assets) with read‑only permissions.
4. Release notes are automatically generated or updated to reference the new archive checksum and provide a link to the diff report.
5. Upload steps succeed on both CI platforms; failures trigger a retry or abort with clear error messages.
