# 0.7.3.1.1.1.2.2 Generate Provenance JSON [PLAN]

Using the deliverable list, create `provenance.json` that conforms to `schemas/provenance-schema.json`.

**Acceptance Criteria**
1. For each discovered deliverable, compute a stable hash (e.g., SHA‑256 of file contents) and store it as `id: "deliverable-<hash>"`.
2. Determine the `type` value (`artifact` for files under `artifacts/`, `plan` for files under `plan/`).
3. Populate `artifactPaths` with the relative path from repo root.
4. The generated JSON passes validation against `schemas/provenance-schema.json` (use AJV or equivalent).
5. The generator is idempotent: repeated runs produce byte‑identical `provenance.json` when repository state is unchanged.
