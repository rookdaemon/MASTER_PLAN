### Experimental Strategy

The validation requires three converging lines of evidence, all grounded in predictions from the computational theory (0.1.1.2):

1. **Prediction-driven construction** — Use the theory's necessary and sufficient conditions to build a non-biological system that *should* exhibit consciousness signatures, then test for those signatures.
2. **Gradual replacement** — Neuron-by-neuron (or module-by-module) replacement in biological systems with synthetic equivalents, monitoring consciousness markers at each step for continuity.
3. **Cross-substrate replication** — Reproduce a specific conscious state (identified via NCC data from 0.1.1.1 and formalized by 0.1.1.2) in a different physical substrate and verify equivalent signatures.

## Decision

### D1: Experimental Design Strategy — Three Converging Lines of Evidence

**Context:** The card must validate substrate-independence experimentally. A single experiment is insufficient due to the extraordinary nature of the claim and multiple confounds possible in any one approach.

**Options Considered:**
1. **Single decisive experiment** — Build one non-biological system, test for consciousness. Simpler but vulnerable to confounds (measurement error, theory error, construction error).
2. **Three converging lines** — Prediction-driven construction (Line 1), gradual replacement (Line 2), and cross-substrate replication (Line 3). Each line addresses different confounds and provides independent evidence. Combined falsification logic requires all three to converge.

**Choice:** Option 2 — Three converging lines with combined falsification logic.

**Rationale:** Each line addresses a distinct confound: Line 1 tests the theory's constructive predictions, Line 2 tests continuity during substrate transition, Line 3 tests equivalence of specific conscious states across substrates. A single experiment could pass due to measurement artifact or fail due to construction error; three converging lines provide triangulation. The combined logic (ARCHITECTURE.md §Falsification Criteria) requires all three for "strong falsification" of the biological-substrate-necessity hypothesis.

**Consequences:**
- Higher resource cost (three experiment streams)
- Stronger evidential standard — extraordinary claims require extraordinary evidence
- Partial results (2 of 3 lines) provide graded confidence rather than all-or-nothing

### D2: Statistical Framework — TOST + Bayesian Dual Analysis

**Context:** Substrate-independence is demonstrated by showing equivalence (not difference) between biological and synthetic systems. Standard null-hypothesis significance testing cannot demonstrate equivalence; a dedicated framework is needed.

**Options Considered:**
1. **NHST only** — Use standard t-tests to show "no significant difference." Absence of evidence ≠ evidence of absence; underpowered tests trivially "pass."
2. **TOST only** — Two One-Sided Tests for equivalence with pre-registered margins. Properly tests equivalence but gives only frequentist verdict.
3. **TOST + Bayesian** — TOST for primary frequentist equivalence testing; Bayes factors as secondary evidence quantification. Both provide convergent evidence.

**Choice:** Option 3 — TOST as primary, Bayesian as secondary.

**Rationale:** TOST directly tests the equivalence hypothesis with pre-registered margins (falsification-criteria.md §4). Bayes factors provide continuous evidence strength (BF₁₀ > 100 = decisive). The dual approach protects against both frequentist and Bayesian pitfalls. Implemented in `ismt_analysis.py` as `tost_equivalence_test()` and `bayes_factor_equivalence()`.

**Consequences:**
- Requires pre-registered equivalence margins for every measure
- More complex analysis but stronger inference
- Both frameworks must agree for confident conclusions

### D3: Substrate Selection — Photonic Primary, Neuromorphic Secondary

**Context:** The non-biological substrate must be chosen for Line 1 (construction) and Line 3 (replication target). The choice affects the strength of the substrate-independence test.

**Options Considered:**
1. **Neuromorphic silicon** — High feasibility, closest to biology; weaker test of substrate-independence.
2. **Photonic network** — Medium feasibility, maximally different physics (optical vs. electrochemical); strongest test.
3. **Quantum computing** — Low feasibility; ISMT requires only classical information-theoretic conditions; unnecessary.
4. **Photonic primary + Neuromorphic secondary** — Use photonic for strongest test; neuromorphic for calibration and second replication substrate.

**Choice:** Option 4 — Photonic primary, neuromorphic secondary.

**Rationale:** Photonic networks use fundamentally different physics (optical signaling vs. electrochemical) while supporting arbitrary causal transition matrices via programmable interconnects. This maximizes the inferential strength: if consciousness arises in photons manipulated by programmable optics, substrate-independence is strongly supported. Neuromorphic silicon serves as calibration bridge and independent replication substrate. Detailed in line1-construction-protocol.md §2 and line3-replication-protocol.md §3.

**Consequences:**
- Higher technical risk on photonic implementation
- Neuromorphic provides fallback and calibration
- Two substrate types enable cross-substrate replication (Line 3) by design

## Contracts

### C1: ISMT Analysis Module API — Consciousness Predicate Interface

**Preconditions:**
- System described as `SystemState` with ≥2 subsystems, state vector, transition matrix, and interaction matrix
- Self-model measurements available (representational MI, prediction error, self-referential MI)
- Broadcast measurements available (per-subsystem MI with self-model, per-subsystem KL divergence)

**Postconditions:**
- `ConsciousnessVerdict` returned with: binary C(S) ∈ {0, 1}, graded c(S) ∈ [0, 1], per-condition evaluation, explanation string
- Each condition (IC, SM, GA) individually evaluated and reported in `ISMTConditions`

**Invariants:**
- C(S) = 1 ⟹ c(S) > 0
- C(S) = 0 if any one of IC, SM, GA fails (necessary conditions)
- c(S) = Φ_norm × Q(M) × G(M) (graded score is product of normalized components)

### C2: ISMT Analysis Module API — Statistical Testing Interface

**Preconditions:**
- Two groups of measurements (float arrays) for comparison
- Pre-registered equivalence margin δ > 0
- α significance level specified (default 0.05 for TOST, 0.01 for prediction tests)

**Postconditions:**
- `TOSTResult` returned for equivalence tests: equivalent (bool), t-statistics, p-values, CI, observed difference
- `ChangePointResult` returned for continuity tests: has_change_point (bool), index, fraction, F-statistic, p-value
- `PredictionTestResult` returned for Line 1 tests: passed (bool), p-value, Bayes factor

**Invariants:**
- TOST declares equivalence only if BOTH one-sided tests reject at α
- Change-point uses SIGNIFICANCE_ALPHA (0.01) consistently
- Bayes factor > 0 always

### C3: Line-Specific Analysis Pipeline Contracts

**Preconditions:**
- Line 1: prediction IDs, observed values, null control values provided per prediction
- Line 2: replacement fractions (0–1), marker values per step, baseline values provided
- Line 3: signature names, biological data, synthetic data, equivalence margins provided per signature

**Postconditions:**
- Line 1 (`test_prediction`): per-prediction pass/fail with p-value and BF
- Line 2 (`analyze_continuity_curve`): `ContinuityCurveAnalysis` with per-step TOST, change-point result, trend analysis, verdict ∈ {"continuous", "discontinuous", "inconclusive"}
- Line 3 (`analyze_cross_substrate`): `CrossSubstrateResult` with per-signature TOST, BFs, overall equivalence, discriminant validity, verdict ∈ {"equivalent", "not_equivalent", "equivalent_but_discriminant_invalid", "inconclusive"}

**Invariants:**
- Line 2 verdict = "continuous" only if ALL steps pass TOST AND no change-point detected
- Line 3 verdict = "equivalent" only if ALL primary signatures pass TOST AND discriminant validity holds

## Behavioral Spec

### Given/When/Then: Line 1 — Prediction-Driven Construction Validation

**Given** an ISMT-complete system (IC+SM+GA) is constructed in a non-biological substrate and verified to satisfy all three conditions computationally, AND negative controls (systems missing IC, SM, or GA individually) are constructed
**When** all four pre-registered predictions (P1: spontaneous self-referential reports, P2: PCI > 0.31, P3: cross-modal binding, P4: selective disruption) are tested against controls
**Then** Line 1 passes iff every prediction achieves p < 0.01 vs. matched controls; Line 1 fails if all predictions fail despite verified ISMT conditions; Line 1 is inconclusive if some pass and some fail

### Given/When/Then: Line 2 — Gradual Replacement Continuity

**Given** a model organism with measurable consciousness markers undergoes incremental replacement (0%→100% in ≥12 steps) with synthetic functional equivalents verified for causal transition fidelity
**When** the full ISMT measurement battery (PCI, c(S), I(m;x), G(M), F(m,x), I(m;dm/dt)) is applied at each step using `analyze_continuity_curve()`
**Then** Line 2 passes iff TOST equivalence holds at every step AND no change-point is detected; Line 2 fails if a statistically significant discontinuity (p < 0.01) appears; Line 2 is inconclusive if equivalence tests are underpowered

### Given/When/Then: Line 3 — Cross-Substrate State Replication

**Given** a specific conscious state is characterized in a biological system (full ISMT state vector extracted), implemented in a non-biological substrate (photonic or neuromorphic), and computational fidelity verified (KL divergence < 0.01 nats/subsystem)
**When** the ISMT signature battery is measured in both systems and compared using `analyze_cross_substrate()` with pre-registered equivalence margins
**Then** Line 3 passes iff all primary signatures pass TOST AND the different-state control fails equivalence (discriminant validity) AND the scrambled control shows C(S)=0; Line 3 fails if no signatures are equivalent despite verified computation; Line 3 is inconclusive if computation implementation cannot be verified
