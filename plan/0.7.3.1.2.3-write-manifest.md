# 0.7.3.1.2.3 Write Manifest [PLAN]

Consume `file‑metadata.csv` and generate `TRACEABILITY-MANIFEST.md` with a TOC, per‑file tables, and a summary section.

**Acceptance Criteria**
1. The manifest is a Markdown file named exactly `TRACEABILITY-MANIFEST.md`.
2. Every file from `inventory.txt` appears once, with a table row showing all required fields.
3. The document begins with a generated Table of Contents linking to each file’s entry.
4. A summary section reports the total number of files and the cumulative size (both in bytes and human‑readable format).
5. The manifest passes a linting step that checks Markdown syntax and that all internal links are valid.
