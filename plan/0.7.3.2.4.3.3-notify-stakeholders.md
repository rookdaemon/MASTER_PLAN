# 0.7.3.2.4.3.3 Send Stakeholder Notification Email [PLAN]

**Description:**  
Generate and send an email to all stakeholders defined in `plan/0.7.3.5-stakeholder-notification.md` containing release details, links to the uploaded archive, and release notes.

**Acceptance Criteria**
1. Stakeholder email addresses are extracted from `plan/0.7.3.5-stakeholder-notification.md`.
2. An email body is rendered from a template (`templates/release-notify.md`) populated with:
   * Release version (`0.7.3`), current date, and archive URL from the upload step.
   * Link to release notes produced by `0.7.3.2.4.2`.
3. Email is sent via the configured SMTP service; delivery success (SMTP 250) is verified.
4. Each sent email is logged with timestamp, recipient, and message-id in `logs/release-0.7.3.2.log`.
5. Failure to send to any recipient aborts the step, logs the error, and marks the task as failed.
