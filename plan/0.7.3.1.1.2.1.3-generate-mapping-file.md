# 0.7.3.1.1.2.1.3 Generate Requirement‑to‑Code Mapping File [PLAN]

Merge the extracted requirement IDs and collected code artifacts into a single machine‑readable traceability matrix and store it under `docs/traceability/req-code-mapping.json`.

### Acceptance Criteria
1. The final file `docs/traceability/req-code-mapping.json` conforms to the schema `{requirement_id: [ {path, type, identifier} ]}`.  
2. All requirement IDs from `req-ids.csv` appear exactly once as top‑level keys; no duplicate keys exist.  
3. The file passes JSON schema validation and is lint‑free.  
4. The mapping file is added to version control and referenced in the project’s README.  
5. The lead developer reviews the generated matrix and formally approves it in the PR.
