# 0.7.3.2.3.1.3 Validate List [PLAN]

## Description
Provide a manual verification step for developers to confirm that `temp/modules-raw.json` contains the complete and correct set of top‑level modules.

## Acceptance Criteria
1. Documentation (in `docs/verification.md`) describes how to open `modules-raw.json` and cross‑check against the repository `src/` tree.
2. A checklist item is added to the Pull Request template prompting the reviewer to confirm the list’s accuracy.
3. The verification step is gated: CI will not proceed to the next stage unless a `validation.ok` flag file is created after manual review.
4. The process logs the reviewer’s GitHub username and timestamp in `temp/modules-raw.validation.log`.
5. Any discrepancy discovered triggers a failure of the subsequent pipeline stage with a clear error message.
