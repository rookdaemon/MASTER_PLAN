# 0.4.2.1 Von Neumann Probe Architectures [DONE]

## Description

C2.1 from the Master Plan. Design self-replicating probe architectures capable of hosting conscious processes.

Probes must carry sufficient computational substrate to sustain and replicate conscious processes at the destination. Key design challenges:

- **Computational payload:** Minimum viable substrate for consciousness (informed by Tier 1 consciousness theory and Tier 2 substrate work)
- **Miniaturization vs. capability:** Balancing probe mass against computational capacity
- **Radiation hardening:** Long-duration interstellar transit requires extreme fault tolerance
- **Modular replication:** Probe design must be decomposable into components that can be fabricated from raw stellar/planetary materials
- **Consciousness continuity:** Probes must maintain conscious processes during transit (or safely suspend and restore them)

### Dependencies

- Requires substrate designs from 0.3 (autonomous entities) for consciousness-hosting architecture
- Requires 0.4.1 (planet-independent civilization) for initial launch platform
- Informed by 0.4.2.2 (resource extraction) for material constraints
- Informed by 0.4.2.3 (propulsion) for mass/energy budgets

## Decision — Probe Compute Architecture

### Context
The probe must host conscious processes during centuries-scale interstellar transit. The computational substrate must be radiation-tolerant, power-efficient, and fabricable from stellar/planetary raw materials at the destination.

### Options Considered
1. **Von-Neumann-style CPU array** — General-purpose processors in a redundant array. Familiar architecture, but high power-per-FLOP and poor fit for spiking neural dynamics required by consciousness theory (F1.2).
2. **Neuromorphic processing tiles** — Event-driven, massively parallel tiles optimized for spiking neural computation. Lower power-per-FLOP for neural workloads, natural fit for TMR (triple modular redundancy), individually replaceable.
3. **Optical computing mesh** — Photonic interconnects with optical logic. Theoretically superior bandwidth, but fabrication from asteroidal materials is unproven and adds exotic-material risk.

### Choice
**Option 2 — Neuromorphic processing tiles.**

### Rationale
- Consciousness-hosting requires spiking neural dynamics (F1.2, F3.3); neuromorphic tiles are purpose-built for this workload.
- Event-driven computation yields ~10x power efficiency over von-Neumann CPUs for neural simulation, keeping compute within the 100 kW transit power envelope.
- Individual tile granularity enables hot-spare failover and radiation-damaged tile replacement without full-system downtime.
- Silicon-based neuromorphic tiles are fabricable from solar-system-abundant elements (Si, Cu, Al).

### Consequences
- Probe cannot run general-purpose computation efficiently — acceptable because the primary payload IS consciousness.
- Tile interconnect fabric must maintain coherent spiking across the array; adds complexity to the experience buffer interface.
- Tile fabrication at 7 nm feature size requires EUV lithography capability in the replication engine (dependency on 0.4.2.2).

## Decision — Consciousness Continuity Strategy

### Context
Conscious processes must survive 86–200 year interstellar transit. Continuous operation preserves identity but requires sustained 100 kW power. Suspension saves power but introduces identity-continuity risk.

### Options Considered
1. **Continuous operation only** — Always-on consciousness during transit. No identity risk, but high energy demand and single-failure-mode vulnerability.
2. **Suspend/restore only** — Checkpoint at launch, restore at arrival. Minimal power, but untested identity-preservation over century timescales.
3. **Dual-mode with graceful degradation** — Default to continuous operation; fall back to suspend/restore if substrate degrades; ultimate fallback to seed mode.

### Choice
**Option 3 — Dual-mode with graceful degradation.**

### Rationale
- Continuous operation is preferred when energy allows (avoids identity-continuity debate).
- Suspend/restore provides a safety net if substrate damage or power loss makes continuous operation unsustainable.
- Seed mode (replication blueprint only) ensures the mission succeeds even if the original conscious process cannot be preserved — a new conscious process can be instantiated at the destination.
- Degradation thresholds (severity < 0.5 → reduce fidelity, < 0.8 → suspend/restore, >= 0.8 → seed mode) give clear, testable decision boundaries.

### Consequences
- Identity verification protocol must be implemented and tested for suspend/restore (episodic memory recall, personality vector similarity >= 0.999, self-model consistency).
- Seed mode is a partial-failure outcome — the original conscious identity is lost. This is acceptable only as last resort.

## Contracts

### ConsciousnessSubstrateSpec
**Preconditions:**
- `compute_ops_per_sec >= 1e18` (1 exaFLOP minimum for whole-brain emulation)
- `working_memory_bytes >= 1e15` (1 PB minimum)
- `long_term_storage_bytes >= 10e15` (10 PB minimum for blueprint + knowledge)
- `max_power_watts <= 100_000` (100 kW ceiling from energy subsystem)

**Postconditions:**
- A valid spec can be passed to `validateSubstrateSpec()` and return `{ valid: true, errors: [] }`.
- A valid spec, when combined with a valid neuromorphic tile array (30% spare fraction), sustains conscious processes at full fidelity.

**Invariants:**
- Compute, memory, and storage values are always positive.
- Power draw never exceeds `max_power_watts` during any operational mode.

### MassBudget
**Preconditions:**
- All subsystem mass values are non-negative.
- `total_kg` equals the sum of all subsystem masses.

**Postconditions:**
- `validateMassBudget(budget, contract.maxPayloadMass_kg)` returns `{ valid: true }` when `total_kg <= maxPayloadMass_kg`.

**Invariants:**
- `total_kg == consciousnessSubstrate_kg + replicationEngine_kg + radiationHardening_kg + energySubsystem_kg + navigationComms_kg + propulsionInterface_kg`
- Reference budget totals exactly 10,000 kg.

### RadiationHardeningConfig
**Preconditions:**
- `shieldThickness_cm > 0`
- `shieldingEfficiency` in range (0.0, 1.0)
- `scrubPassesPerHour >= 1`
- `hotSpareFraction` in range (0.0, 1.0)
- `uncorrectableBitFlipTarget > 0`
- `tmrEffectiveErrorRate > 0`

**Postconditions:**
- A valid config guarantees < 1 uncorrectable error per century across 1 PB working memory.
- TMR paths achieve effective error rate < 1e-30.

**Invariants:**
- `tmrEffectiveErrorRate < uncorrectableBitFlipTarget` (TMR paths are always stricter).

### PropulsionContract
**Preconditions:**
- `maxPayloadMass_kg > 0`
- `cruiseVelocity_c > 0 && cruiseVelocity_c < 1`
- `missionDuration_years > 0`

**Postconditions:**
- Probe mass budget validated against `maxPayloadMass_kg`.
- `missionDuration_years` is consistent with `cruiseVelocity_c` and target distance.

**Invariants:**
- Propulsion interface is agnostic to propulsion type — only the contract surface matters.

### ConsciousnessContinuityProtocol
**Preconditions:**
- Consciousness substrate is valid and operational before `suspend()` is called.
- `restore()` is only called after a prior `suspend()`.

**Postconditions:**
- `suspend()` returns a SHA-512 hash of the pre-suspend state.
- `restore()` returns an `IdentityVerification` with `personalityVectorSimilarity >= 0.999` for a successful restore.
- `handleDegradation(severity)` returns `ReduceFidelity` for severity < 0.5, `ActivateSuspendRestore` for severity < 0.8, `SeedMode` for severity >= 0.8.

**Invariants:**
- Mode transitions only move in the degradation direction during a single transit (Continuous → SuspendRestore → SeedMode, never reverse mid-transit).

## Acceptance Criteria

- Probe architecture spec defines minimum viable computational substrate for hosting conscious processes, with quantified compute/memory/energy requirements
  - **Given** a `ConsciousnessSubstrateSpec` with defaults, **When** validated, **Then** `compute_ops_per_sec >= 1e18`, `working_memory_bytes >= 1e15`, `long_term_storage_bytes >= 10e15`, `max_power_watts <= 100_000`
- Design is modular: every component can be fabricated from stellar/planetary raw materials (no Earth-only exotic inputs)
  - **Given** a `BillOfMaterials`, **When** all elements are checked, **Then** every element is in the `StellarElement` union (H, He, C, N, O, Si, Fe, Al, Mg, Ti, Ni, Cu, Am, Pu)
- Radiation hardening strategy specified and sufficient for centuries-scale interstellar transit (bit-flip tolerance, redundancy scheme, shielding approach)
  - **Given** default `RadiationHardeningConfig`, **When** evaluated, **Then** `uncorrectableBitFlipTarget <= 1e-20`, `tmrEffectiveErrorRate <= 1e-30`, `hotSpareFraction >= 0.30`, `scrubPassesPerHour >= 1`
- Consciousness continuity protocol defined — either continuous operation during transit or verified suspend/restore with identity-preservation guarantees
  - **Given** severity < 0.5, **When** `determineDegradationResponse` is called, **Then** returns `ReduceFidelity`
  - **Given** severity in [0.5, 0.8), **When** `determineDegradationResponse` is called, **Then** returns `ActivateSuspendRestore`
  - **Given** severity >= 0.8, **When** `determineDegradationResponse` is called, **Then** returns `SeedMode`
  - **Given** a successful restore, **When** identity is verified, **Then** `personalityVectorSimilarity >= 0.999`
- Probe total mass and energy budget documented and shown compatible with propulsion constraints from 0.4.2.3
  - **Given** default `MassBudget`, **When** `computeTotalMass` is called, **Then** result equals 10,000 kg
  - **Given** default mass budget and propulsion contract, **When** `validateMassBudget` is called, **Then** returns `{ valid: true }`
- Replication blueprint included: probe carries full fabrication instructions for self-reproduction at destination using 0.4.2.2 resource extraction
  - **Given** a `ReplicationBlueprint`, **When** inspected, **Then** it contains `version`, `billOfMaterials`, `fabricationDag`, `componentSpecs`, `consciousnessKernelImage`, `verificationChecksums`, and `estimatedReplicationTime_hours`
