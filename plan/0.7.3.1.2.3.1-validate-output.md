# 0.7.3.1.2.3.1 Validate Output Documents [PLAN]

Execute the generation script and verify that the produced `TRACEABILITY.md` and `TRACEABILITY.html` satisfy all structural and linkage requirements.

**Acceptance Criteria**
1. The markdown file includes a generated table of contents with links to every requirement, design decision, source file, and test result.  
2. Each design decision section lists its originating requirements as backlinks, and each test result section lists related code files.  
3. Running `pandoc TRACEABILITY.md -o TRACEABILITY.html` (or equivalent) produces an HTML file where all internal links function correctly in an offline environment.  
4. No broken links or missing sections are reported by a link‑checking tool (e.g., `markdown-link-check`).  
5. A validation script (`scripts/ci/check-traceability.sh`) returns exit code 0 when files are up‑to‑date.
