# 0.5.4 Cosmological-Scale Resilience [DONE]

## Description

E1.4 — Ensure that the distributed conscious civilization is resilient against cosmological-scale threats (supernovae, gamma-ray bursts, galactic collisions, rogue black holes).

Key challenges:
- Threat modeling at galactic scales and timescales
- Redundancy and diversity across stellar neighborhoods
- Early warning and response systems for cosmological events
- Ensuring no single catastrophe can eliminate all conscious presence

## Decision

### Context
A distributed conscious civilization must survive cosmological-scale threats. The design must model threats, define spatial distribution requirements, validate colony independence, and compute extinction probability — all as a testable TypeScript module.

### Options Considered
1. **Monte Carlo simulation** — Full stochastic simulation of threat events against colony distributions. Pros: handles spatial correlations naturally. Cons: slow, non-deterministic, hard to test deterministically.
2. **Analytic probability model with independence assumption** — Closed-form P(extinction) = P(single)^N with conservative safety margins. Pros: deterministic, fast, testable, conservative. Cons: overestimates independence; mitigated by 10× safety margin on rates.

### Choice
Option 2 — Analytic probability model with independence assumption.

### Rationale
Deterministic computation is easier to validate and test. The 10× safety margin on all threat rates compensates for the independence assumption. The `DistributionModel.simulate()` method signature reserves `nEvents` for future Monte Carlo extension without breaking the API.

### Consequences
- All tests are deterministic and fast.
- Spatial correlation effects are conservatively bounded rather than precisely modeled.
- Future Monte Carlo integration possible without interface changes.

## Contracts

### ThreatClass
- **Preconditions:** `id` is unique within catalog; `lethalRadiusLy ≥ 0`; `warningWindowSeconds ≥ 0`; `eventRatePerMillennium ≥ 0`; beamed threats must define `beamHalfAngleDeg > 0`.
- **Postconditions:** Immutable data; used as input to probability model.
- **Invariants:** `directionality === "beamed"` ↔ `beamHalfAngleDeg` is defined and > 0.

### Colony
- **Preconditions:** `id` is unique; `positionGalacticLy` is a 3-tuple of finite numbers; `independenceScore` ∈ [0, 1].
- **Postconditions:** Immutable data; used as input to distribution validation.
- **Invariants:** `knowledgeStoreComplete` and `localManufacturing` must both be `true` for a colony to pass validation.

### DistributionModel
- **Preconditions:** `colonies` array is non-empty; `threats` defaults to `THREAT_CATALOG`.
- **Postconditions:** `validate()` returns `ValidationResult` with `valid: true` iff all distribution requirements are met. `simulate()` returns `ExtinctionProbability` with `meetsTarget: true` iff `value < 10⁻²⁰`.
- **Invariants:** `colonies` and `threats` are readonly after construction.

### validateDistribution
- **Preconditions:** Input is an array of `Colony`.
- **Postconditions:** Returns `ValidationResult`. `valid === true` ↔ `errors.length === 0`. Checks: colony count ≥ 100, pairwise separation ≥ 100 ly, all knowledge stores complete, all manufacturing autonomous.
- **Invariants:** Warnings (e.g. partial independence) do not affect `valid` status.

### AlertProtocol
- **Preconditions:** `threatClass` references a valid `ThreatClass.id`; `detectionMethod` is non-empty.
- **Postconditions:** Immutable data describing per-threat detection and response strategy.
- **Invariants:** Threats with zero warning window (`T1b`, `T2a`, `T2b`) must use `"local-autonomous"` propagation and `maxResponseLatencyYears === 0`.

## Threshold Registry

| Name | Value | Unit | Valid Range | Rationale | Sensitivity |
|------|-------|------|-------------|-----------|-------------|
| minColonies | 100 | count | [50, ∞) | N≥100 achieves P(extinction)<10⁻²⁰ even with 10× safety margin | Reducing to 50 still meets target; below 10 fails |
| minSeparationLy | 100 | light-years | [50, 500] | Must exceed largest isotropic lethal radius (50 ly for T1a) with margin | Below 50 ly allows correlated supernova kills |
| minGalacticArms | 3 | count | [2, 6] | Prevents single-arm GRB or merger disruption from killing >1% | 2 arms marginal for GRB coverage |
| maxFractionDestroyedPerEvent | 0.01 | fraction | (0, 0.05] | 1% cap ensures no single event is civilization-threatening | 5% still survivable but reduces redundancy |
| extinctionProbabilityTarget | 1e-20 | per millennium | (0, 1e-10] | "Astronomically improbable" threshold per AC | Relaxing to 1e-10 allows fewer colonies |
| safetyMarginMultiplier | 10 | dimensionless | [3, 100] | Compensates for rate uncertainties and independence assumption | 3× marginal; 100× overly conservative |
| galaxyRadiusLy | 50000 | light-years | [40000, 60000] | Milky Way disk radius for volume normalization | ±20% changes P(single) by ~±50% |
| galaxyThicknessLy | 1000 | light-years | [500, 2000] | Milky Way thin disk thickness for volume normalization | ±50% changes P(single) by ~±50% |

## Behavioral Spec

### Scenario: Valid distribution passes all checks
```
Given a distribution of 100 colonies with pairwise separation ≥ 150 ly, all with complete knowledge stores and manufacturing
When validateDistribution is called
Then result.valid is true and result.errors is empty
```

### Scenario: Insufficient colony count rejected
```
Given a distribution of 10 colonies (below the 100 minimum)
When validateDistribution is called
Then result.valid is false and errors include "Insufficient colonies"
```

### Scenario: Extinction probability meets target for compliant distribution
```
Given 100 well-separated colonies and the canonical THREAT_CATALOG with 10× safety margin
When simulateExtinctionProbability is called
Then result.meetsTarget is true and result.value < 1e-20
```

### Scenario: Small colony count fails extinction target
```
Given only 2 colonies
When simulateExtinctionProbability is called
Then result.meetsTarget is false
```

### Scenario: Zero-warning threats require local-autonomous response
```
Given ALERT_PROTOCOLS for T2a and T2b (GRBs)
Then propagationStrategy is "local-autonomous" and maxResponseLatencyYears is 0
```

## Acceptance Criteria
- Threat catalog documented covering: supernovae (including core-collapse and Type Ia), gamma-ray bursts (long and short), galactic collisions, rogue stellar objects, black hole encounters, and interstellar medium hazards
- Minimum viable distribution density defined: minimum number of independent stellar systems required to reduce simultaneous extinction probability to < 10⁻¹⁰ per millennium
- Early warning system architecture specified: detection horizons, signal propagation strategies, and response latency requirements for each threat class
- Spatial redundancy strategy defined such that no single cosmological event within any plausible radius can eliminate more than 1% of conscious presence
- Isolation protocol defined: colony independence requirements so that no shared infrastructure creates correlated failure modes
- Extinction probability reduced to astronomically improbable levels (< 10⁻²⁰ per millennium) through validated spatial distribution model
