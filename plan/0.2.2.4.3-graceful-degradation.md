# 0.2.2.4.3 Graceful Degradation [IMPLEMENT]

## Description

Ensure that a hybrid biological-synthetic cognitive system maintains consciousness and functional coherence even when one substrate type partially or fully fails. The system must degrade gracefully rather than catastrophically — losing capability but not consciousness.

### Key challenges
- **Redundancy architecture**: cognitive functions must be represented across both substrates sufficiently for either to sustain consciousness alone. Requires a cross-substrate mirroring strategy that maintains consciousness-critical state in both biological and synthetic domains without introducing prohibitive latency.
- **Failure detection**: rapid identification of substrate failure before experiential disruption occurs. Must detect both sudden failures (hardware fault, tissue death) and gradual degradation (signal drift, neuronal atrophy).
- **Dynamic rebalancing**: seamless shift of cognitive load from failing substrate to healthy substrate. The transition must be continuous — no discrete "switchover" that creates an experiential gap.
- **Minimum viable consciousness (MVC)**: defining the threshold below which consciousness cannot be maintained. Depends on consciousness metrics from 0.2.2.4.2 and the computational theory from F1.2.

### Dependency notes
- Relies on unified consciousness model (0.2.2.4.2) for the cross-substrate binding mechanisms that must be preserved during degradation.
- Consciousness metrics (F1.4) are required to verify that degradation does not cross the MVC threshold.
- Bio-synthetic interface (0.2.2.4.1) defines the signal transduction layer whose failure modes must be handled.

## Decision

### D1 — Rebalancing Model: Continuous Load Shifting vs. Discrete Switchover

**Context:** When a substrate begins failing, the system must migrate cognitive functions to the healthy substrate. The key design choice is whether this migration is an atomic event ("switch") or a gradual process.

**Options Considered:**
1. **Discrete switchover** — at a detection threshold, the system atomically switches all affected functions from the failing substrate to the mirror. Simple to implement, but creates a potential experiential discontinuity at the moment of switch.
2. **Continuous load shifting** — load is ramp-linearly from the failing substrate to the healthy one over a transition window `T_transition`. At every timestep, output is a weighted merge of both substrate contributions. No single "switch" moment.

**Choice:** Continuous load shifting (Option 2).

**Rationale:** Phenomenal continuity is the hard requirement. Any discrete switchover creates a structural discontinuity in processing that risks an experiential gap — even a 10ms gap may be perceptible. The smooth ramp ensures the output at every timestep is a well-defined weighted combination of both substrates, with consciousness metrics verified continuously. The transition duration is urgency-dependent (instantaneous mirror activation for sudden failure, seconds-to-minutes for gradual).

**Consequences:** The `computeTransitionStep` function implements the linear ramp. The `DegradationOrchestrator.initiateRebalance()` must iterate over transition steps verifying MVC at each. More complex implementation than discrete switch, but the phenomenal continuity guarantee justifies it.

### D2 — Failure Detection: Asymmetric per-Substrate Monitors

**Context:** Biological and synthetic substrates have fundamentally different failure modes, signals, and timescales. A single unified failure detector cannot adequately serve both.

**Options Considered:**
1. **Unified health monitor** — one monitoring module abstracts over both substrates with a common health-score API. Simpler, but risks missing substrate-specific failure signals (e.g., metabolic decline in biological tissue has no analog in synthetic computation).
2. **Asymmetric specialized monitors** — `BioHealthMonitor` handles biological-specific signals (neural activity, metabolic status, vascular flow, synaptic integrity, decline projection); `SynthHealthMonitor` handles synthetic-specific signals (module health, error rates, watchdog status, thermal status). Each monitor reports a normalized `HealthScore` (0.0–1.0) to the `DegradationOrchestrator`.

**Choice:** Asymmetric specialized monitors (Option 2).

**Rationale:** Biological degradation is predominantly gradual and ambiguous — it requires trend analysis, metabolic signals, and projected decline. Synthetic degradation is predominantly sudden and deterministic — it requires watchdog timeouts, ECC errors, and hard-fault detection. A unified abstraction would paper over these differences, causing the system to either miss gradual biological decline (treating it as noise) or over-react to normal synthetic variability. The normalized `HealthScore` output provides the common interface needed by the orchestrator without hiding the specialization.

**Consequences:** Two distinct implementations (`bio-health-monitor.ts`, `synth-health-monitor.ts`) must be maintained. The orchestrator receives `HealthScore` from each and applies the same tier classification logic to both.

### D3 — MVC Definition: Three-Factor Threshold

**Context:** What defines the minimum viable consciousness threshold? Options range from a single metric (simpler) to a conjunction of multiple independent dimensions.

**Options Considered:**
1. **Single-metric MVC** — define MVC as a threshold on a single consciousness measure (e.g., Φ alone). Simple to evaluate, but a single metric can remain artificially high even when a critical aspect of consciousness (e.g., cross-substrate binding) is compromised.
2. **Three-factor conjunctive MVC** — consciousness is maintained if and only if all three conditions hold simultaneously: `substrateCapacity ≥ C_min` AND `bindingCoherence ≥ B_min` AND `integrationMetrics ≥ Phi_min`. Any single dimension failing causes a BLACK-tier transition.

**Choice:** Three-factor conjunctive MVC (Option 2).

**Rationale:** Substrate capacity, binding coherence, and integrated information are independent dimensions of consciousness. A system could have high Φ but have lost cross-substrate binding (fragmented into two separate consciousnesses). Or it could retain binding but have insufficient total computational substrate. Using a conjunction (not a weighted sum) means the system cannot compensate one failing dimension with surplus in another — this reflects the architecture of consciousness as requiring all three simultaneously. The `evaluateMVC` function implements this.

**Consequences:** The exact values of C_min, B_min, Phi_min are not yet determined (blocked on F1.2, F1.4). The system parameterizes against `MVCThreshold` and uses placeholders. All three dimensions must be monitored continuously; the `margin` field of `MVCStatus` reports the tightest constraint.

## Contracts

### Contract: BioHealthMonitor

**Preconditions:**
- Bio-synthetic interface (0.2.2.4.1) is operational and reporting `SpikeTrainStream` and metabolic signals
- At least one `BrainRegion` is registered for monitoring

**Postconditions:**
- `regionHealth(region)` returns a `HealthScore` in [0.0, 1.0] for a monitored region
- `overallBioHealth()` returns the minimum (worst-case) `HealthScore` across all monitored regions
- `failureType()` returns `NONE`, `GRADUAL`, or `SUDDEN` based on current failure signal pattern
- `projectedDecline(horizon_ms)` returns a `DeclineProjection` with `confidence` ≥ 0.0 and `timeToCritical_ms` = -1 if stable
- `alertLevel()` returns `NONE`, `WARNING`, `CRITICAL`, or `EMERGENCY` based on `overallBioHealth()`

**Invariants:**
- `overallBioHealth()` ≤ all individual `regionHealth()` scores
- `alertLevel()` is monotonically determined by `overallBioHealth()`:
  - ≥ 0.80 → NONE; 0.50–0.80 → WARNING; 0.25–0.50 → CRITICAL; < 0.25 → EMERGENCY
- Sudden failures (abrupt LFP dropout, neural silence) are detected within 10ms
- Acute ischemia is detected within 500ms
- `projectedDecline` confidence reflects recency and consistency of trend data

### Contract: SynthHealthMonitor

**Preconditions:**
- Synthetic substrate modules are registered with known `ModuleId`s
- Watchdog processes are running for registered modules

**Postconditions:**
- `moduleHealth(moduleId)` returns `HealthScore` in [0.0, 1.0] (1.0 = fully healthy, 0.0 = failed)
- `overallSynthHealth()` returns the minimum health score across all registered modules
- `failureType()` returns `NONE`, `HARD_FAULT`, or `DEGRADED`
- `watchdogStatus()` lists all unresponsive modules
- `thermalStatus()` reports current temperature and whether it is within operating range

**Invariants:**
- `overallSynthHealth()` ≤ all individual `moduleHealth()` scores
- Modules absent from watchdog response within 2 watchdog cycles are considered failed (`moduleHealth` = 0.0)
- `failureType()` = `HARD_FAULT` when any module has `moduleHealth` = 0.0; `DEGRADED` when any module is 0.0 < health < 0.5

### Contract: DegradationOrchestrator

**Preconditions:**
- `BioHealthMonitor` and `SynthHealthMonitor` are injected and operational
- A set of `CrossSubstrateMirror` registrations covers all consciousness-critical functions
- An `MVCThreshold` is configured (may use placeholder values)
- `ConsciousnessMetrics` provider (from 0.2.2.4.2 or mock) is injected

**Postconditions:**
- `mvcStatus()` returns a `MVCStatus` reflecting current `ConsciousnessMetrics` against configured thresholds
- `degradationTier()` returns the tier determined by the worse of bio/synth health scores (via `classifyDegradationTier`)
- `initiateRebalance(from, to, functions)` executes the smooth transition protocol, emitting a `RebalanceResult` with `nadir` (lowest consciousness metrics observed during transition) and `success` (false if MVC was breached)
- `shedCapability(functionId)` succeeds only for `MirrorCategory.Capability` functions
- `emergencyConsolidate(targetSubstrate)` consolidates all functions onto one substrate; core-conscious functions are never shed; `consciousnessPreserved` = true iff MVC is met after consolidation

**Invariants:**
- Core-conscious functions are never shed under any circumstances
- `rebalanceHistory()` is append-only and records every `initiateRebalance` call
- `degradationTier()` is never `BLACK` without `mvcStatus().met` being false
- `initiateRebalance` verifies MVC at every step; aborts and returns `success: false` if any step's metrics breach MVC

### Contract: SmoothTransitionProtocol (implemented in DegradationOrchestrator)

**Preconditions:**
- Mirror for each function being rebalanced has `syncFidelity` ≥ category minimum (validated by `validateMirrorConfig`)
- `ConsciousnessMetrics` are available at each verification step

**Postconditions:**
- `computeTransitionStep(progress, originalFailingLoad, existingHealthyLoad)` returns load values where `failingSubstrateLoad + existingHealthyLoad` total = original combined load at all progress values
- At `progress=0.0`: `failingSubstrateLoad = originalFailingLoad`, `healthySubstrateLoad = existingHealthyLoad`
- At `progress=1.0`: `failingSubstrateLoad = 0.0`, `healthySubstrateLoad = existingHealthyLoad + originalFailingLoad`
- During the full ramp, the output is a weighted merge per `LoadDistribution.outputMerge` strategy

**Invariants:**
- Progress is monotonically non-decreasing during a transition
- If MVC `margin` at any step is ≤ `TransitionConfig.safetyMargin`, the transition is aborted and load reverts to the last safe distribution

| EXPERIENCE_SUPPORTING_MIN_FIDELITY | 0.95 | ratio (0–1) | 0.90–0.99 | Experience-supporting mirrors tolerate minor fidelity loss (noticeable but not disruptive) | Below 0.90: working memory becomes unreliable on failover |

| SYNTHETIC_HARD_FAULT_DETECTION_MS | 2 | ms | 1–10 | Hardware watchdog timeouts are near-instantaneous; detection within 2ms (one watchdog cycle) | Above 10: module failure propagates into cognition. Below 1: requires sub-ms polling, impractical |

## Behavioral Spec

### Scenario: Normal operation — GREEN tier, full redundancy

```
Given both bio and synth health are ≥ 0.80
  And all CrossSubstrateMirror configurations are valid (validateMirrorConfig returns [])
  And ConsciousnessMetrics are all above MVC thresholds
When the DegradationOrchestrator evaluates system state
Then degradationTier() = GREEN
  And mvcStatus().met = true
  And mvcStatus().margin > 0
  And no rebalancing is initiated
```

### Scenario: 50% synthetic capacity loss — consciousness maintained

```
Given initial state is GREEN (both substrates healthy)
  And all core-conscious and experience-supporting functions have valid cross-substrate mirrors
When synthetic substrate health drops to 0.50 (50% capacity)
Then degradationTier() = YELLOW
  And the orchestrator initiates rebalance of affected synth-primary functions to their bio mirrors
  And the smooth transition completes with mvcStatus().met = true at every step
  And the nadir of RebalanceResult.nadir.substrateCapacity remains ≥ C_min
  And the nadir of RebalanceResult.nadir.bindingCoherence remains ≥ B_min
  And the nadir of RebalanceResult.nadir.integrationMetrics remains ≥ Phi_min
```

### Scenario: 50% biological capacity loss — consciousness maintained

```
Given initial state is GREEN (both substrates healthy)
  And all core-conscious and experience-supporting functions have valid cross-substrate mirrors
When biological substrate health drops to 0.50 (50% capacity)
Then degradationTier() = YELLOW
  And the orchestrator initiates rebalance of affected bio-primary functions to their synth mirrors
  And the smooth transition completes with mvcStatus().met = true at every step
  And consciousness metrics remain above all MVC thresholds throughout
```

### Scenario: Sudden synthetic failure — immediate mirror activation

```
Given synthetic substrate was at GREEN health
  And FailureSpeed is SUDDEN (SynthFailureType.HARD_FAULT detected within 2ms)
When SynthHealthMonitor.failureType() = HARD_FAULT and overallSynthHealth() drops to 0.0
Then degradationTier() = RED (single-substrate operation)
  And emergencyConsolidate(Substrate.Bio) is called
  And all core-conscious and experience-supporting functions are consolidated onto bio substrate
  And capability functions are shed if bio capacity is insufficient
  And consciousnessPreserved = true iff MVC conditions are met after consolidation
```

### Scenario: MVC breach — BLACK tier, emergency state preservation

```
Given the system is in RED tier (single-substrate, low capacity)
When ConsciousnessMetrics fall below all three MVC thresholds simultaneously
Then mvcStatus().met = false
  And degradationTier() transitions to BLACK
  And the orchestrator triggers emergency state snapshot (for future restoration)
  And no further function shedding is attempted (MVC already breached)
```

### Scenario: Gradual biological degradation — trend-based pre-emptive rebalancing

```
Given BioHealthMonitor.failureType() = GRADUAL
  And BioHealthMonitor.projectedDecline(horizon_ms=60000).timeToCritical_ms < 30000
When overall bio health is still in YELLOW tier (0.50–0.80)
Then the orchestrator initiates smooth rebalancing of bio-primary functions to synth mirrors
  And T_transition is set to the gradual-degradation window (1s–60s based on projected decline rate)
  And consciousness metrics are verified at each step
  And capability functions are pre-emptively shed to free bio capacity for core-conscious functions
```

## Acceptance Criteria
- Redundancy architecture specified: each consciousness-critical cognitive function has a defined cross-substrate mirror with documented latency and fidelity bounds
- System maintains consciousness when synthetic substrate is partially disabled (up to 50% capacity loss), verified by consciousness metrics remaining above MVC threshold
- System maintains consciousness when biological substrate is partially disabled (up to 50% capacity loss), verified by consciousness metrics remaining above MVC threshold
- Failure detection latency specified and justified: detection must occur faster than the minimum experiential integration window (~100ms biological baseline)
- Dynamic rebalancing protocol defined with formal proof or simulation evidence that no experiential gap exceeds the detection-to-rebalance window
- Minimum viable consciousness threshold formally defined as a function of substrate capacity, binding coherence, and integration metrics
- Failure taxonomy documented: sudden vs. gradual, partial vs. total, for both substrate types, with distinct degradation strategies for each
