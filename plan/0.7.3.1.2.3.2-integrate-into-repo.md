# 0.7.3.1.2.3.2 Integrate Traceability into Repository [PLAN]

Commit the validated traceability artifacts to the repository, update documentation references, and ensure CI checks them on each push.

**Acceptance Criteria**
1. `TRACEABILITY.md` and `TRACEABILITY.html` are placed under `traceability/plan-guardian/` in the repo.  
2. The project `README.md` includes a section with links to both files and a brief description of the traceability package.  
3. The CI pipeline runs `scripts/ci/check-traceability.sh`; the step passes when the committed files match the script’s output.  
4. A pull‑request template is updated to remind contributors to run the generation script after any change that affects requirements, design decisions, code, or tests.  
5. The commit history includes a clear commit message (`ci: add generated traceability artifacts`) and the changes are tagged with a version bump (e.g., `v1.2.0‑traceability`).
