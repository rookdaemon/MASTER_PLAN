# 0.7.3.2.3.3.2 Calculate SHA‑256 Checksum [PLAN]

Generate a SHA‑256 hash of the newly created archive and ensure it matches the checksum recorded in the repository.

**Acceptance Criteria**
1. A SHA‑256 checksum is computed for the archive file and written to `dist/<archive>.sha256`.
2. The checksum file follows the format `<checksum>  <filename>`.
3. The checksum is compared against any pre‑existing checksum entry; mismatches cause the task to fail.
4. The checksum value is logged and made available to downstream upload steps.
5. The checksum file is committed to version control in the `dist/` folder (or stored as a release asset).
