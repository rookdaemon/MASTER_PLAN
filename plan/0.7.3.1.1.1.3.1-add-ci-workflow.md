# 0.7.3.1.1.1.3.1 Add CI Workflow Integration [PLAN]

Create or update the GitHub Actions CI workflow to run the provenance generator and validate its output.

**Acceptance Criteria**
1. `.github/workflows/ci.yml` contains a step that executes `npm run provenance`.
2. The step runs a validation script (`npm run provenance:validate`) that checks `provenance.json` against the defined schema; the job fails on validation errors.
3. CI logs contain the line “Provenance validation passed” when validation succeeds.
4. The workflow is triggered on pull‑request and push events to the main branch.
5. The new workflow passes on a clean repository (no schema violations).
