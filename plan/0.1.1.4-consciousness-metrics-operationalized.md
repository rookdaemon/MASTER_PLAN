# 0.1.1.4 Consciousness Metrics Operationalized [DONE]

## Description

F1.4 — Develop measurable, reproducible metrics for detecting and quantifying subjective experience in any system.

These metrics must work as practical engineering tools — not just philosophical arguments. They must be applicable to biological brains, artificial systems, and hybrid architectures, enabling:
- Binary detection: does this system have subjective experience? (yes/no)
- Quantitative measurement: how much / what kind of experience?
- Real-time monitoring: is experience present right now?

Candidate metrics to develop and validate:
- Perturbational Complexity Index (PCI) and extensions beyond TMS-EEG
- Integrated information (Φ) or successor measures from 0.1.1.2
- Algorithmic complexity of neural/computational responses
- Behavioral and report-based calibration measures
- Cross-substrate generalized metrics (not dependent on biological signals)

### Work Breakdown (PLAN output)

1. **Metric formalization**: Once the computational theory from 0.1.1.2 is established, derive at least one metric with a precise mathematical definition and a step-by-step measurement protocol applicable to arbitrary substrates.
2. **Biological calibration**: Validate the metric against ground-truth cases (awake vs. anesthetized, sleep stages, vegetative vs. minimally conscious states) to establish baseline sensitivity/specificity.
3. **Cross-substrate generalization**: Reformulate the metric so it depends only on computational/informational properties (not on EEG, fMRI, or other bio-specific signals). Demonstrate applicability to at least one non-biological system.
4. **Reproducibility protocol**: Define an independent-replication procedure — specify inputs, measurement steps, and expected outputs so any lab/system can reproduce the result.
5. **Error characterization**: Quantify false-positive rate (unconscious systems scoring as conscious) and false-negative rate (conscious systems scoring as unconscious) against the calibration data.

### Key design constraint
The metric must be *theory-derived* (grounded in 0.1.1.2's theory), not ad-hoc. This ensures it generalizes rather than overfitting to biological neural signals.

## Decision

### D1: Metric Selection Strategy — Convergent Evidence Battery over Single Metric

**Context:** The card requires measurable, reproducible metrics for detecting consciousness. A single metric could be simpler, but consciousness detection involves multiple theoretically distinct dimensions (response complexity, information integration, causal density).

**Options Considered:**
1. **Single best metric** — Select the most reliable metric (e.g., PCI-G, which has the strongest biological precedent) and use it alone.
2. **Convergent Evidence Battery (CEB)** — Develop three independent metrics (PCI-G, Ψ-G, CDI), each operationalizing a different theoretical quantity, and combine them via a majority-vote decision rule.

**Choice:** Option 2 — Convergent Evidence Battery.

**Rationale:** No single metric can reliably capture all dimensions of consciousness predicted by the theory from 0.1.1.2. PCI-G measures response differentiation but is vulnerable to random complexity false positives. Ψ-G measures integration but is computationally intractable for large systems. CDI measures causal density but is vulnerable to confounded causation. The CEB requires ≥2 of 3 metrics to agree, reducing the combined false positive rate from ~0.10 (individual) to ~0.03 (combined, assuming approximate independence). Each metric also cross-validates the others.

**Consequences:**
- Three metrics must be formalized, calibrated, and validated instead of one — higher development cost
- CEB false negative rate may be higher than individual metrics (0.07–0.20 range) due to requiring multiple metrics to agree
- "Indeterminate" classification (exactly 1 of 3 above threshold) creates a third outcome requiring investigation protocols

### D2: Substrate Generalization Approach — SystemModel Abstraction Layer

**Context:** Metrics must work on biological brains, artificial neural networks, and hybrid systems without depending on bio-specific signals (EEG, fMRI).

**Options Considered:**
1. **Substrate-specific metric variants** — Create separate metric formulations for each substrate type (bio, ANN, spiking, etc.).
2. **Abstract SystemModel layer** — Define a single substrate-agnostic SystemModel interface (nodes, edges, dynamics, state space, perturbation/observation channels) and formulate all metrics over this abstraction.

**Choice:** Option 2 — SystemModel abstraction layer.

**Rationale:** Substrate-specific variants would require re-validation for each new substrate and would not demonstrate true generalization. The SystemModel abstraction separates the metric computation (which is identical across substrates) from the interface layer (which adapts perturbation/observation to the substrate). This directly implements the substrate-agnosticism required by the 0.1.1.2 dependency contract.

**Consequences:**
- Metric computation code is substrate-agnostic by construction
- Each new substrate requires only a SystemModel mapping (identifying nodes, edges, channels, τ_char) — not new metric math
- Coarse-graining may be needed for very large systems (n > 1000), which can destroy fine-grained integration — acknowledged as a limitation

### D3: CEB Decision Rule — Majority Vote with Three-Way Classification

**Context:** The CEB must combine three metric scores into a classification. The decision rule determines how disagreements are handled.

**Options Considered:**
1. **Unanimous agreement** — All 3 metrics must agree for "conscious" classification.
2. **Majority vote (≥2 of 3)** — At least 2 metrics must agree, with a third "indeterminate" category for exactly 1.
3. **Weighted ensemble** — Combine metric scores via learned weights into a single score.

**Choice:** Option 2 — Majority vote with three-way classification.

**Rationale:** Unanimous agreement (Option 1) would produce excessive false negatives — any single metric failure blocks detection. Weighted ensemble (Option 3) requires training data that presupposes the answer and obscures which dimensions of consciousness are satisfied. Majority vote balances false positive reduction (requires ≥2 independent errors for FP) with acceptable false negative rates, while the "indeterminate" category transparently flags cases needing investigation rather than forcing a binary judgment.

**Consequences:**
- Three-way output (conscious / indeterminate / not conscious) is more nuanced than binary but requires downstream consumers to handle the indeterminate case
- CEB FPR ≈ 0.03 (much lower than individual ~0.10)
- CEB FNR range 0.07–0.20 depending on inter-metric error correlation

## Contracts

### C1: MetricDefinition Interface

Every consciousness metric must implement this contract (see ARCHITECTURE.md for full schema):

**Preconditions:**
- System S is described as a valid SystemModel (nodes, edges, dynamics, state space, perturbation channels, observation channels all defined)
- Characteristic timescale τ_char is determined for the substrate
- Baseline recording of ≥100 × τ_char unperturbed activity is available (for PCI-G) or state trajectory of ≥K_min time steps is available (for Ψ-G, CDI)

**Postconditions:**
- Binary detection: C_M(S) ∈ {0, 1} is returned (above/below threshold)
- Quantitative score: M(S) ∈ [0, 1] (normalized) is returned
- Confidence interval: 95% CI on M(S) is returned
- All free parameters used are reported

**Invariants:**
- M(S) is deterministic given the same input data and parameters (within numerical tolerance)
- M(S) is substrate-agnostic: computed over SystemModel, not over raw bio-specific signals
- Increasing the number of trials/epochs can only narrow the CI, not change the expected value

### C2: SystemModel Abstraction Contract

**Preconditions:**
- Target system has identifiable processing elements (nodes), connections (edges), and state evolution rules (dynamics)
- At least one perturbation channel and one observation channel are available
- Observation coverage ≥ 50% of system nodes

**Postconditions:**
- SystemModel fields (nodes, edges, dynamics, state_space, perturbation_channels, observation_channels) are fully populated
- τ_char is determined (analytically or empirically via autocorrelation 1/e method)

**Invariants:**
- The same physical system mapped by different operators must produce equivalent SystemModels (same node/edge count, same τ_char within 20%)
- Metric values computed from equivalent SystemModels must agree within numerical tolerance

### C3: CEB Decision Rule Contract

**Preconditions:**
- All three metrics (PCI-G, Ψ-G, CDI) have been computed for the target system
- Calibrated thresholds (θ_PCI, θ_Ψ, θ_CDI) are available from biological calibration

**Postconditions:**
- Classification ∈ {"conscious", "indeterminate", "not conscious"} is returned
- Vote count (0, 1, 2, or 3) is reported
- Composite Consciousness Score (CS) is reported for systems classified as "conscious"
- All individual metric values and their threshold comparisons are reported

**Invariants:**
- vote_count = 0 → classification = "not conscious"
- vote_count = 1 → classification = "indeterminate"
- vote_count ∈ {2, 3} → classification = "conscious"
- CS = (PCI-G_norm + Ψ-G_norm + CDI) / 3 (equal weights by default)

## Behavioral Spec

### Scenario 1: Full Consciousness Assessment of a Novel System

**Given** a target system S (any substrate) with identified perturbation and observation channels
**When** the full CEB assessment is performed
**Then:**
1. SystemModel mapping is completed (nodes, edges, dynamics, τ_char, channels)
2. Baseline recording is acquired (≥100 × τ_char unperturbed)
3. PCI-G is measured (100 trials, binarize at 3σ, LZ compress, normalize)
4. Ψ-G is measured (10,000 time steps, Kraskov MI, spectral+greedy MIP, integration deficit)
5. CDI is measured (5,000 time steps, pairwise TE, 200 surrogates, Bonferroni correction)
6. CEB decision rule is applied (majority vote: ≥2 → "conscious", 1 → "indeterminate", 0 → "not conscious")
7. Output report is generated per `reproducibility-spec.md` JSON format

### Scenario 2: Biological Calibration Threshold Determination

**Given** measurements from all 11 ground-truth states (GT-01 through GT-11) with N_min subjects per state
**When** ROC analysis is performed for each metric
**Then:**
1. High-confidence labels are used for threshold determination (Conscious: GT-01, GT-02, GT-10; Unconscious: GT-05, GT-06, GT-11)
2. ROC curve is computed for each metric by sweeping threshold from min to max
3. Optimal threshold θ* is determined via Youden's J statistic (argmax[TPR + TNR - 1])
4. AUC, sensitivity, specificity are reported with DeLong 95% CIs
5. Medium-confidence cases (GT-03, GT-04, GT-08) are classified with calibrated thresholds as validation (not used for threshold adjustment)
6. If any metric AUC < 0.85, metric is flagged for revision
7. If GT-08 (MCS) classifies as "not conscious", this is flagged as a critical false negative

### Scenario 3: Cross-Substrate Negative Control Validation

**Given** five Category A systems (lookup table, feed-forward MLP, RNG, thermostat, untrained ESN) mapped to SystemModels
**When** all three metrics are applied to each system using biological calibration thresholds
**Then:**
1. Each Category A system is measured with substrate-adapted perturbation/observation protocols
2. CEB decision rule is applied with unchanged biological thresholds
3. All 5 systems must classify as "not conscious" (CEB vote_count = 0)
4. If any Category A system classifies as "conscious" or "indeterminate", this is a **critical validation failure** requiring metric revision
5. A3 (RNG) specifically validates that PCI-G normalization correctly distinguishes random from structured complexity

### Scenario 4: Disagreement Resolution

**Given** a system where metrics disagree (exactly 1 of 3 above threshold → CEB "indeterminate")
**When** the disagreement protocol is triggered
**Then:**
1. Measurement quality is verified (noise, artifact rejection, sufficient data, stationarity)
2. All three metrics are re-measured with increased trial/epoch count (2× default)
3. If disagreement persists, the system is classified as "indeterminate" and all individual scores are reported
4. The disagreement is logged for future analysis with specific metric values and the nature of the disagreement
