# 0.7.3.2.4.1.2 Validate Comments Integrity [PLAN]

Run a sanity‑check on the exported comment file to ensure completeness and uniqueness.

**Acceptance Criteria**
1. A validation script parses `feedback/comments.json` and confirms the count matches the platform‑reported total.  
2. The script detects and reports any duplicate `reviewer_id` + `timestamp` + `text` triples.  
3. Any missing required fields cause the script to fail with a clear error message.  
4. On successful validation the script outputs “All comments verified – N entries”.  
5. Validation results are saved to `feedback/comments_validation.log`.
