# 0.7.3.1.3.1.2 Build and Verify Binaries [PLAN]

Build all deliverable binaries from the newly‑tagged source code, generate checksums, and confirm that they match expected outputs.

### Acceptance Criteria
1. Build scripts execute successfully against the tagged source without manual intervention.  
2. All binaries are produced in the designated `dist/` directory.  
3. SHA‑256 checksums are generated for each binary and stored in a `checksums.txt` file.  
4. A verification step re‑computes the checksums and confirms they match the recorded values.  
5. Build artefacts are stored in a temporary staging area ready for packaging.
