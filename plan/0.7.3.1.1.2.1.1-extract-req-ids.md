# 0.7.3.1.1.2.1.1 Extract Requirement IDs [PLAN]

Identify and export every high‑level requirement identifier from the master requirements document into a flat list (e.g., CSV column or JSON array) that will serve as the authoritative source for downstream mapping.

### Acceptance Criteria
1. All requirement IDs present in `docs/requirements/master-requirements.md` are captured; a sanity check confirms 100 % coverage.  
2. The output file `docs/traceability/req-ids.csv` contains one ID per line with no empty rows.  
3. The file is validated against a simple schema (single column header `requirement_id`).  
4. The extraction script runs without errors on a fresh checkout and is committed to the repo.  
5. The lead developer reviews and signs off the list in the PR comments.
