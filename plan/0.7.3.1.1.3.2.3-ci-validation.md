# 0.7.3.1.1.3.2.3 Add CI validation for generated index table [PLAN]

**Description**  
Create a CI step that verifies the presence and integrity of `artifacts/INDEX_TABLE.md` after the script runs.

**Acceptance Criteria**
1. The CI pipeline includes a job that checks `artifacts/INDEX_TABLE.md` exists and is non‑empty.
2. The job parses the markdown table and confirms that every **Relative Path** entry points to an existing file in the repository; missing files cause the job to fail.
3. The job also ensures the table is sorted correctly (alphabetically by Category then Artifact Name); out‑of‑order rows cause failure.
4. Failure messages are explicit, indicating the exact problem (missing file, empty table, or sorting issue).
5. The CI configuration is added to `.github/workflows/generate-index.yml` (or equivalent) with a clear step name.
