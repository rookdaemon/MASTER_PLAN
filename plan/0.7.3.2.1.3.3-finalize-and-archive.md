# 0.7.3.2.1.3.3 Finalize and Archive Approval Package [PLAN]

Update the security manifest, obtain panel sign‑off, and archive the complete approval package.

**Acceptance Criteria**
1. `manifest/v2.0.json` is generated, incorporating all agreed‑upon changes, and versioned correctly.
2. The panel signs the approval document, producing `review-output/approval-signoff.pdf`.
3. A compressed archive (`review-output/approval-package.zip`) contains the findings report, signed manifest, and meeting recording.
4. The archive is committed to the repository under `review-output/` with a release tag `v2.0-approval`.
5. A post‑approval checklist is completed and stored as `review-output/post‑approval-checklist.md`.
