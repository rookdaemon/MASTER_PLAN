# 0.7.3.2.2.1.1 Draft Criteria Matrix [PLAN]

Produce an initial criteria matrix that enumerates required expertise areas (traceability, security, compliance), minimum experience levels, independence constraints, and a scoring rubric.

**Acceptance Criteria**
1. A markdown file `REVIEWER_CRITERIA.md` is created in the `docs/peer-review/` folder.
2. The matrix lists at least three domains (traceability, security, compliance) with specific skill items.
3. Minimum experience (≥ 5 years) is defined for each domain.
4. Independence rules (e.g., no conflict of interest with the code author) are clearly articulated.
5. A 0‑5 scoring rubric for each criterion is included.
