# 0.7.3.2.2.2 Create Archive & Generate Checksum [PLAN]

**Description:**  
Package the staged artifacts into a zip file named `traceability-0.7.3.1-<YYYYMMDD>.zip`, compute its SHA‑256 checksum, and store both files in `deliverables/traceability/`.

**Acceptance Criteria**
1. The zip archive contains exactly the files listed in the collection report, preserving directory structure.
2. Archive name follows the required pattern, with `<YYYYMMDD>` matching the current date (`20260409`).
3. A checksum file `traceability-0.7.3.1-<YYYYMMDD>.zip.sha256` is generated and its value matches a re‑computed checksum.
4. Both files are present in `deliverables/traceability/` and are readable.
5. A short log `archive-creation-log.md` documents the command used, file sizes, and checksum verification outcome.
