# 0.7.3.2.2.2.2.1 Draft Initial Schema [PLAN]
## Description
Create a first‑pass schema (JSON Schema format) that lists all candidate fields identified in the catalog, assigning provisional data types and marking each as required or optional based on initial business analysis.

## Acceptance Criteria
1. A machine‑readable JSON Schema file (`schema-draft.json`) is produced.  
2. Every candidate field from the catalog appears in the schema with a placeholder type.  
3. Fields are preliminarily flagged as required or optional.  
4. The draft schema can be loaded by a standard JSON‑Schema validator without syntax errors.  
5. Documentation (markdown table) summarizing the draft fields, types, and required flags is included.
