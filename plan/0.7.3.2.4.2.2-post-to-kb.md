# 0.7.3.2.4.2.2 Post Release Notes to Knowledge Base [PLAN]

**Description:**  
Programmatically POST the generated release‑notes markdown to the central knowledge‑base via its REST API. Capture the full API response, including the permanent URL, and store it in `logs/release-0.7.3.2.log`.

**Acceptance Criteria**
1. An HTTP POST request is made to the KB endpoint with the markdown content as payload.
2. The response JSON is saved to `logs/release-0.7.3.2.log`.
3. The permanent URL from the response is extracted and recorded in the same log file.
4. If the POST fails, a non‑zero exit code is returned and an error entry is appended to the log.
5. The log file is created (or overwritten) atomically to avoid partial writes.
