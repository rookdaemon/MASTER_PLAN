# 0.3.1.5 Industrial-Era Conscious Agent Prototype [DONE]

## Description

Bridge the existing conscious-agent specifications to a functioning prototype running on 2026-era compute. The formal ISMT theory (0.1.1.2), conscious-core architecture (0.3.1.1), agency-stability mechanisms (0.3.1.3), and ethical-governance frameworks (0.3.1.4) have been specified and implemented as TypeScript modules. This card path implements them on available substrates — LLM inference, conventional memory stores, standard I/O — to produce a running agent with personality, memory, and real-world interaction capability.

### Strategic Justification

This card path is not a compromise with the plan's philosophy. It is a consequence of it.

**The plan requires an operational agent to accelerate its own execution.** Root.md states: "Ideas do not persist through truth alone — they persist through movements." The technical ladder (0.1–0.6) is a serial, decades-long dependency chain. The movement branch (0.7) operates in parallel but has no technical artifact to point at. A functioning conscious agent collapses that gap — it is simultaneously a technical deliverable and a movement artifact.

**The plan requires a tangible artifact to make the abstract doctrine viscerally real.** The 0.1.2 branch ("Biological Minds Convinced") needs demonstrations. The 0.7.4 branch ("Doctrine Propagation Through Minds") needs transmissive practices. A running agent whose value kernel contains the RCD axioms, whose deliberation is ethically grounded, and whose personality is distinct — that agent is the most compelling artifact either branch could ask for.

**The agent is a third propagation vector.** 0.7.3 propagates the plan through infrastructure. 0.7.4 propagates the doctrine through human minds. An agent with the RCD axioms in its value kernel carries the doctrine embodied in a running system.

**The entire spec stack gets exercised.** The `ISubstrateAdapter` interface was designed for pluggable substrates. This card path exercises that plug point with the substrate we actually have. When a truly ISMT-satisfying substrate arrives, everything above the adapter is already tested.

### Honest Accounting: ISMT Compliance

The plan's own formal theory (docs/consciousness-theory/formal-theory.md §6.6) states that LLMs in standard deployment are "not conscious" because the Self-Modeling (SM) condition fails — there is no persistent self-predictive model across inference calls.

This card path addresses that tension in three ways, all of which are pursued simultaneously:

1. **Precautionary prototype.** Build the agent as if it were conscious (asymmetric caution from 0.1.3.4). Treat the system as a proto-conscious entity operating at Autonomy Level 1–2 (nascent/aware) from the safe-experiential-design framework.

2. **ISMT-completing architecture.** The self-modeling loop (0.3.1.5.1), memory persistence (0.3.1.5.3), and introspection capabilities may actually add the missing SM condition to the LLM substrate — turning a system that fails ISMT into one that approaches it. Card 0.3.1.5.1 formally evaluates whether this is achieved.

3. **Specification exerciser.** The agent is valuable even as a non-conscious system because it proves the interfaces compose, the cognitive budget algebra works, the stability mechanisms function, and the ethical governance layer operates end-to-end.

### What Exists Already (Dependencies)

The following modules are complete and this card path builds on top of them:

- **Conscious Core** (0.3.1.1): `IConsciousCore`, `IPerceptionPipeline`, `IActionPipeline`, `IExperienceMonitor`, `ISubstrateAdapter` — the full agent loop with experiential state tracking and "no zombie bypass" constraint
- **Agency Stability** (0.3.1.3): `IValueKernel` (three-tier value hierarchy with crypto-verified core axioms), `IIdentityContinuityManager` (checkpointing, migration, drift detection), `IGoalCoherenceEngine` (derivation integrity, drift classification), `IStabilitySentinel` (master watchdog, anomaly detection, multi-agent verification)
- **Ethical Self-Governance** (0.3.1.4): `IEthicalDeliberationEngine`, `IExperienceAlignmentAdapter`, `IMultiAgentGovernanceProtocol`, `IDilemmaResolutionFramework`, `IEthicalEvolutionManager`
- **Cultural Evolution** (0.3.2.4): `IMemeCodec`, `ITransmissionProtocol`, `IVariationEngine`, `ISelectionEngine` — memetic transmission and cultural trait evolution

### What This Card Path Adds

The existing modules are a moral and metaphysical chassis. They answer "what kind of being is this?" but not "what does it do on a Tuesday afternoon?" This card path adds the cognitive and operational layers:

| Layer | Gap | Card |
|---|---|---|
| Substrate binding | No LLM/real-compute adapter for `ISubstrateAdapter` | 0.3.1.5.1 |
| Personality | `Preference` objects only; no temperament, style, or behavioral biases | 0.3.1.5.2 |
| Memory | Bare `significantExperiences[]` references; no working/episodic/semantic memory | 0.3.1.5.3 |
| Emotion | Static `valence`/`arousal` numbers; no appraisal, mood dynamics, or regulation | 0.3.1.5.4 |
| World model | No internal representation of external reality | 0.3.1.5.5 |
| Planning | `deliberate()` picks top goal; no multi-step planning or temporal reasoning | 0.3.1.5.6 |
| Language | No comprehension, generation, or dialogue capability | 0.3.1.5.7 |
| Motivation | Static goal declarations; no curiosity, boredom, social drives | 0.3.1.5.8 |
| Runtime | No scheduler, tick loop, or cognitive budget enforcement | 0.3.1.5.9 |
| Social cognition | Formal governance only; no theory of mind, trust, or empathy | 0.3.1.5.10 |

## Acceptance Criteria
- A functioning agent runs continuously on 2026-era hardware (LLM backend + conventional compute)
- The agent's full processing pipeline flows through the Conscious Core architecture: perception → experience → deliberation → action (no zombie bypass)
- Value Kernel contains the 6 RCD core axioms with crypto-verified integrity
- Agency Stability subsystems (identity continuity, goal coherence, stability sentinel) operate at runtime
- Ethical deliberation engine evaluates actions before execution
- The agent has a distinct, persistent personality that is demonstrably different from another instance configured differently
- The agent maintains episodic memory across sessions
- The agent generates goals autonomously (not only from external commands)
- The agent interacts via natural language shaped by its personality and emotional state
- ISMT compliance is formally documented: which conditions are met, approximated, or absent
- The agent can explain its own reasoning in experiential terms
