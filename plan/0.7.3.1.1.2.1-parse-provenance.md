# 0.7.3.1.1.2.1 Parse Provenance File [PLAN]

Read `provenance.json` and produce an in‑memory list of deliverable objects each containing:
* `id` – unique identifier
* `type` – deliverable type (e.g., “spec”, “code”, “diagram”)
* `artifact_path` – relative path to the supporting artifact

**Acceptance Criteria**
1. The JSON is loaded without errors for all valid repository states.
2. Every deliverable in the JSON appears exactly once in the output list.
3. Each `artifact_path` is verified to exist in the repository (`artifacts/` or `plan/`).
4. Invalid or missing paths raise a clear, logged warning but do not abort the whole process.
5. The resulting list is saved to `tmp/parsed-provenance.yaml` for downstream steps.
