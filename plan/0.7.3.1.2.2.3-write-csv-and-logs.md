# 0.7.3.1.2.2.3 Write CSV and Logs [PLAN]

Consume the metadata objects from the previous step, write them to `file-metadata.csv`, and emit a summary log line.

**Acceptance Criteria**
1. CSV header matches exactly: `path,sha256,md5,size_bytes,created_iso,modified_iso`.
2. All rows are correctly escaped for commas and newline characters in file paths.
3. After writing, a log line is emitted: `Processed X files, Y bytes in Z seconds`.
4. The CSV file is created atomically (write to temp then rename) to avoid partial writes.
5. The implementation returns a promise that resolves only after the file is closed and the log line is flushed.
