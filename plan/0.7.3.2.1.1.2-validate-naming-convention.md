# 0.7.3.2.1.1.2 Validate Naming Convention [PLAN]

Verify that each artifact identified in Subtask 0.7.3.2.1.1.1 follows the naming convention defined in the Traceability Standards.

**Acceptance Criteria**
1. A regex pattern for each artifact type (e.g., `req-matrix_v\d+\.\d+\.xlsx`) is defined in the plan.
2. An automated check runs against the file names recorded in `found‑artifacts.json`.
3. All files match their respective pattern; mismatches are reported with file path and expected pattern.
4. The check aborts the workflow if any file violates the naming convention.
5. A brief validation log (`naming‑validation.log`) is generated and stored alongside the summary table.
