# 0.7.3.2.1.1 Verify Artifact Presence [PLAN]

Ensure that the traceability package produced in **0.7.3.1** contains every required artifact type:

* Requirements matrix  
* Test‑case mapping  
* Risk traceability matrix  
* Compliance checklists  

**Acceptance Criteria**
1. Each of the four artifact types is present in the package folder.
2. Artifact filenames match the naming convention defined in the Traceability Standards (e.g., `req-matrix_v{major}.{minor}.xlsx`).
3. No extra, unrelated files are included in the top‑level package directory.
4. A summary table listing artifact type → file path is created and saved as `artifact‑presence‑summary.md`.
5. The summary table is reviewed and approved by the Lead Engineer.
