# 0.7.3.1.1.1.2 Collect Artifacts Hashes [PLAN]

Identify every code artifact and configuration file generated by prior tasks, compute its SHA‑256 hash, and record metadata needed for provenance.

**Acceptance Criteria**
1. A script `scripts/collect_artifacts.py` that scans `artifacts/` (recursively), calculates SHA‑256 for each file, and outputs JSON objects with `id` (relative path), `type` (`code` or `config`), `source` (file path), `created_at` (git commit date), `owner` (from front‑matter if present, else `plan‑guardian`), and `hash`.
2. The script excludes generated provenance files themselves to avoid circularity.
3. Result saved to `temp/artifact_hashes.json`.
4. Unit test `tests/test_collect_artifacts.py` confirms that at least two known artifacts (e.g., `artifacts/provenance/provenance.json` after generation and a sample config `artifacts/config/example.yaml`) are correctly hashed.
5. README section in `docs/provenance.md` describes “Artifact Hashing Process”.
