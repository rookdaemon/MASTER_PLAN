# 0.7.3.2.2.1 Define Reviewer Criteria [PLAN]

Create a concrete set of selection criteria that specifies the required expertise, independence, and security/compliance knowledge for reviewers.

**Acceptance Criteria**
1. A documented criteria matrix covering traceability, security, and compliance domains is stored in the repo.
2. Minimum experience level (e.g., 5 years) and independence requirements are explicitly listed.
3. Criteria are reviewed and approved by the project governance board.
4. The criteria document includes a scoring rubric for evaluating candidates.
5. The final criteria are referenced in the reviewer roster document.
