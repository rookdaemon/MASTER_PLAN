# 0.7.3.4.1.1.1 Create Interview Guide [PLAN]

Draft a comprehensive interview guide covering the delivery experience, focusing on pain points, successes, and improvement ideas.

**Acceptance Criteria**
1. At least 10 questions are written, covering process, tools, communication, and outcomes.  
2. The guide is reviewed by the project manager and one stakeholder representative.  
3. All feedback is incorporated and the final version is approved in writing.  
4. The guide is stored in the project knowledge‑base folder `knowledge-base/interviews/`.  
5. The guide file is named `interview-guide.md` and version‑controlled in the repository.
