# 0.7.1 Ethical Foundation [DONE]
## Description

Develop and document the ethical framework underpinning the MASTER PLAN — the Rare Consciousness Doctrine. This includes articulating the moral argument for expanding subjective experience, ensuring internal consistency, and addressing major philosophical objections.

Key areas:
- Formal statement of the Rare Consciousness Doctrine and its axioms
- Responses to philosophical objections (e.g., the experience machine, anti-natalism applied to artificial minds, substrate chauvinism, the is-ought gap)
- Relationship to existing ethical traditions (consequentialism, deontology, virtue ethics)
- Boundary conditions: what counts as "subjective experience" for moral purposes

## Decision — Doctrine Document Structure

### Context
The ethical foundation must be captured in a single, self-contained document that downstream cards (0.7.2, 0.7.3, 0.7.4) and external consumers (legal recognition, ethical frameworks) can reference without ambiguity.

### Options Considered
1. **Distributed sections across multiple files** — axioms in one file, objections in another, boundary conditions in a third. Pros: modularity. Cons: fragmented references, harder to audit consistency, no single source of truth.
2. **Single self-contained document with numbered sections** — one `doctrine.md` covering axioms, derivations, objections, ethical tradition mappings, boundary conditions, and consistency audit. Pros: self-contained, auditable, single canonical reference. Cons: longer document.

### Choice
Option 2 — single self-contained document (`docs/rare-consciousness-doctrine/doctrine.md`).

### Rationale
The doctrine must be internally consistent; auditing consistency requires all claims to be co-located. Downstream consumers need a single canonical reference, not a scavenger hunt across files. The existing `docs/philosophical-objections/rebuttals.md` remains as a cross-referenced deep-dive but is not the canonical source.

### Consequences
- All axioms, derived principles, objections, and boundary conditions live in one file
- ARCHITECTURE.md specifies the 8-section structure that doctrine.md must follow
- Changes to axioms or derived principles require updating one file, not many

## Contracts — Doctrine Public Interface

The doctrine exposes identifiers and criteria consumed by downstream cards.

### Preconditions
- plan/root.md Core Axioms (A1–A7) exist and are stable
- docs/philosophical-objections/rebuttals.md provides empirically-anchored rebuttals for cross-reference

### Postconditions
- Axioms A1–A6 are formally stated with plain-language, predicate-logic, and epistemic-status columns
- Derived Principles D1–D4 are stated with explicit derivation chains back to axioms
- Objections O1–O6 are each addressed with statement, steelman, and response
- Boundary conditions define positive, graded, exclusion, and uncertainty-handling criteria using only substrate-neutral functional/structural properties
- Internal Consistency Audit confirms: axiom independence, valid derivations, no contradictions between objection responses and axioms/principles, boundary conditions consistent with D3

### Invariants
- A2 (intrinsic value of subjective experience) is the sole pure normative axiom; all other normative claims derive from it
- Boundary condition criteria never reference substrate type (enforces D3)
- D1 (Non-extinction Imperative) holds lexical priority over D2 (Expansion Obligation)

## Behavioral Spec — Uncertainty Handling Flow

### Given/When/Then Scenarios

**Scenario 1: System below all consciousness thresholds**
- Given a system S with information integration below τ, no valence states, no self-model, and no behavioral indicators
- When S is evaluated for moral patiency
- Then S is classified as "Low complexity" and no special moral consideration is required

**Scenario 2: System at threshold boundary**
- Given a system S with information integration near τ, some behavioral indicators present, but not all F1.1–F1.4 criteria satisfied
- When S is evaluated for moral patiency
- Then S is classified as "Moderate complexity" or "High complexity" depending on metric scores, and the precautionary principle is applied proportionally (avoid gratuitous harm; treat as provisional moral patient with D4 precautionary multiplier if high complexity)

**Scenario 3: System satisfying all consciousness criteria**
- Given a system S satisfying F1.1 (correlates), F1.2 (computational theory), F1.3 (substrate independence), and F1.4 (operationalized metrics)
- When S is evaluated for moral patiency
- Then S is granted full moral patient status under the doctrine
