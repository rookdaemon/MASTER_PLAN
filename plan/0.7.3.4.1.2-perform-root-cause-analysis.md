# 0.7.3.4.1.2 Perform Root‑Cause Analysis [PLAN]

Analyze the observations gathered from interviews and audit data to identify underlying causes of any issues.

**Acceptance Criteria**
1. Select an analysis technique (e.g., 5 Whys, Fishbone) and document the choice.
2. Conduct the analysis for each identified issue, producing a cause‑effect diagram.
3. Validate each root cause with at least two independent data points or stakeholder confirmations.
4. Prioritize root causes using a defined impact/effort matrix.
5. Deliver a concise root‑cause analysis document (≤ 5 pages) to the project lead.
