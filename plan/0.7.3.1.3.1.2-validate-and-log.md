# 0.7.3.1.3.1.2 Validate and Log [PLAN]

## Description
Validate that the captured `git status` output contains only the expected modified file `docs/codebase-map.md`, then write the output to `artifacts/git-status.txt`.

## Acceptance Criteria
1. The saved `git status` output is parsed and confirms exactly one line matching `M docs/codebase-map.md` (or equivalent).  
2. No other modified, added, deleted, untracked, or staged files are present in the output.  
3. If the validation fails, the plan records a clear error message and aborts.  
4. When validation passes, the full `git status --porcelain` output is written to `artifacts/git-status.txt`.  
5. The artifact file is created with UTF‑8 encoding and ends with a newline.
