# 0.7.3.2.4.2.3 Assemble Final Summary [PLAN]

Combine the sign‑off table, release notes, and open‑items enumeration into a single markdown document, insert the final version number and release date, and place the file at `docs/final_review_summary.md`.

**Acceptance Criteria**
1. Document contains three sections: Sign‑off Table, Release Notes, Open Items.  
2. Final version number and release date are sourced from the manifest and inserted in the header.  
3. All open items are listed with assigned owners and resolution deadlines (or “None” if no open items).  
4. The assembled file is saved as `docs/final_review_summary.md`.  
5. The document passes a final markdown validation script and is referenced in the top‑level manifest.
